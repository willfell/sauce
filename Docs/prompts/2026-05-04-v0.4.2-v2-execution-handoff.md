## Cross-cutting gotchas (carried from prior cycles + new this cycle)

> [!warning] Read BEFORE starting
> 
> **Gotcha 1: Bracket-escape gap on runTemplaterTemplate folder/filename.** RESOLVED at v0.4.2 S1 — the split-field schema makes literal text never reach `moment.format()`. Manifests still using legacy `folder` / `filename` fields are silently ignored. Both consumer manifests (to-do + meetings) migrated to split-field at v1 S2 staging. v2 evolves them further to date-suffix filename pattern.
>
> **Gotcha 2: Node-harness install + Obsidian's plugin auto-rescan.** `platform/test/run-install.js` writes via `fs` (Node), bypassing Obsidian's vault adapter. Obsidian's CustomJS plugin watches the vault adapter for change events to auto-rescan its jsFolder. After a Node-harness install adds new files OR rewrites class bodies, the user MUST Cmd+R Obsidian to force plugin re-init. **APPLIES at T4.1 — SpaceNavButtons class body changes in S3 T3.1.**
>
> **Gotcha 3: T-FIX1 mkdir module_directory at install time** (v0.3.0 codified). Every blueprint with `module_directory` set gets that directory mkdir'd between `applyPreInstall` and `files[]` loop. Mechanisms exempt.
>
> **Gotcha 4: Force re-process via installed.json edit (CLAUDE.md ask-before-acting).** `install.js:111` short-circuits per-item install when `subscribed.version === installed.version`. Versions are bumped this cycle so re-process fires automatically; no manual installed.json edits expected.
>
> **Gotcha 5: User-override preserved on folder_templates + nav-button registry overwrites.** `applyTemplaterFolderTemplates` preserves user-customized template paths; `applyNavButtons` REPLACES the contributions[<source>] array on each install. v0.4.2 S2 changes the meetings folder layout to date-routed; existing folder-template registrations apply RECURSIVELY (Templater feature) so no folder-template change needed. Existing flat `beacon/meetings/hubs/` registration covers `beacon/meetings/hubs/2026/05-May/...` automatically.
>
> **Gotcha 6: Mid-cycle redesign methodology.** When smokes surface a misalignment with the user's mental model, ACCEPT the redesign rather than ship the wrong primitive. Workflow: pause SDD → invoke `de:brainstorming` → present revised design → get approval → invoke `de:writing-plans` → resume SDD. v0.4.2 v2 used this flow; if a similar surprise surfaces during S4 smokes, do the same.
>
> **Gotcha 7 (NEW v0.4.2): Mobile-aware visual design as platform value.** Every renderer (CustomJS class) MUST be designed for both desktop AND mobile from now on. Adaptive layouts, touch-friendly targets, no hover-only affordances. The installer remains desktop-only (landmine #8 unchanged); the visual contract is what changes. Carry forward to all future blueprint CustomJS class authoring.
>
> **Gotcha 8 (NEW v0.4.2): Filename pattern `<Name>-YYYY-MM-DD` invariant.** All date-routed daily artifacts use date-suffix; hyphen-only separator; no spaces. Future blueprints with date-routed artifacts MUST follow this convention. Date-extraction in CustomJS classes uses regex `\d{4}-\d{2}-\d{2}` against filenames (handles both prefix-style legacy + suffix-style v0.4.2+).
