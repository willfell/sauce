## What this cycle is + why

Headspace Phase 3 e2e validation clicked through the 7 entity-create flows shipped in v0.46.0. **6 of 7 functionally worked** (with one cross-cutting cosmetic bug); **1 of 7 (project) was broken** beyond cosmetics. The four real defects:

| # | Severity | Symptom | Fix posture |
|---|---|---|---|
| **BUG-1** | HIGH | Every entity-create button label rendered with literal Lucide kebab name prefix (`users-plus New Meeting`, `user-plus + New Person`, etc.). Cause: `entity-create.js render()` passes `spec.icon` (a name string) to `AccentButton`, which does `btn.innerHTML = opts.icon + ...` — names get inserted as text. | NEW `icons@0.1.0` mechanism extracts the Lucide-name → SVG translation. Two in-cycle consumers (entity-create + nav-buttons) ratify Lego principle 5 ≥2-rule. |
| **BUG-2** | HIGH | Newly created `Alice Testperson.md` had no `type: person` in frontmatter. Cause: `people/manifest.json` `frontmatter_template` doesn't set `type`. Every other entity-create blueprint sets `type: <bp>` correctly. | One-line manifest edit + `validator@0.1.2 → 0.2.0` Layer 2 rule that forecloses the omission across all future blueprints. |
| **BUG-3** | CRITICAL | Project atlas body empty after creation. Cause: `project/manifest.json` declares `body_template: "Template, Project.md"` (bare basename); installer's `substituteLenient` is a no-op on token-less values; runtime `_readBody` calls `app.vault.getAbstractFileByPath("Template, Project.md")` which resolves against vault root → returns `""`. | Installer auto-prepends `{{templates_path}}/` on bare basenames + JSON-schema tightens `body_template` to basename-only (rejects path-shaped values forever). |
| **BUG-4** | CRITICAL | `extra_files[].body_template` has the identical regression — Project Map.md + `<slug>-board.md` both created with empty bodies. | Same fix as BUG-3 (applies via the same `_resolveBodyTemplatePath` helper on `extra_files[].body_template` too). |

**Cycle slot:** **v0.47.0 MINOR** (renamed from initial v0.46.3 PATCH per Q1.1 — every prior new-mechanism cycle in sauce was MINOR: v0.32.0 platform-claude, v0.40.0 scratch, v0.41.0 convenience, v0.46.0 entity-create). Bundling a new `icons@0.1.0` extraction into v0.47.0 MINOR rather than a PATCH keeps the convention.

**Lego principle 5 check:** the icons mechanism extraction is principled because **three existing icon-resolution patterns already exist today**: (a) entity-create's broken string-passthrough, (b) AccentButton's raw-SVG contract (17 callers pass SVG; entity-create incorrectly passes names), (c) nav-buttons' hardcoded 15-entry `ICONS` map. Two in-cycle consumers migrate (entity-create + nav-buttons); AccentButton stays raw-SVG (a future callsite-simplification cycle widens its API).

## Escalation gates — STOP and report, do not auto-iterate

Per the design's risk register + the brainstorm-handoff's escalation rules:

- **S1 Tier 2 `setIcon` fallback misbehaves headlessly under preflight.** Tier 1 covers every kebab name any current manifest uses, so preflight should pass on Tier 1 alone. If preflight goes red because Tier 2 throws, vendor any missing name into Tier 1 OR document Tier 2 as runtime-only and assert Tier 1 coverage. **Don't silently break preflight.**
- **S3 reveals a nav-buttons consumer using private behavior Icons doesn't reproduce.** Revert S3 and ship S1+S2+S5/S7/S8 as-is. Icons mechanism still gets its ≥2-consumer claim from a future cycle.
- **S5 convention-shift catches an `extra_files[].body_template` with `/` in an existing manifest.** No v0.47.0 manifest uses this shape, but if a future manifest does, do not silently strip — surface the value, ask for re-scope.
- **S7.1 surfaces that the validator has no Layer 2 surface at all.** The rule lands in `install.js`'s pre-install manifest validation step instead — both paths are acceptable. Don't invent a Layer 2 architecture if none exists.
- **S11 surfaces a 5th bug not in the catalogue.** **STOP.** Do not auto-iterate. Capture in a follow-up handoff and re-scope.
- **Any consumer of `nav-buttons` ICONS map relies on private behavior that Icons doesn't reproduce.** If S3 surfaces a regression in a hub render, revert S3, ship S1+S2+S5/S7/S8 as-is.
