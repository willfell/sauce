## Cross-cutting gotchas (recent surprises codified — relevant subset for v0.24.0)

> [!warning] Carry-forward from v0.6.0+
> 1. **API-contract quoting** (v0.6.0) — when implementing the skill body, QUOTE design Sections 3 + 7 literally. Don't paraphrase. Drift here costs an entire CF.
> 2. **Cmd+R reload after new CustomJS class** (gotcha 2) — codified smoke prerequisite for any cycle introducing new classes. v0.24.0 introduces 3.
> 3. **Filename pattern: `<Name>-YYYY-MM-DD`** (gotcha 8) — date as suffix, hyphen-only, no spaces. Cowork dailies follow this.
> 4. **In-cycle re-process bump rule** (landmine #16) — every CF that touches blueprint content needs a PATCH bump (cowork 0.1.0 → 0.1.1 → ...) with 3-file lockstep edits.
> 5. **dv-shim composition discipline** (v0.17.0 lesson a) — when CoworkHubCards composes other renderers (e.g., delegates to BeaconCards via per-section sub-containers), the dv-shim MUST proxy `pages` + `current` + `el` minimum. Single-method shims silently break sub-renderers.
> 6. **YAML date auto-parsing** (v0.16.0 lesson a) — frontmatter values matching `YYYY-MM-DD` or `YYYY-MM` auto-parse to Date|Luxon objects. Quote on creation; tolerant parse on read. Cowork summaries use `created: YYYY-MM-DD` and may need `_toMoment(val)` helper if cards filter by date.
> 7. **Trailing-whitespace lint** (v0.20.0 caseTW1) — every blueprint template content file is auto-linted. Authoring prompts + templates with trailing spaces will fail the harness; trim before commit.
