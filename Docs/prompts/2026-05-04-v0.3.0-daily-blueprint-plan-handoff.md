## Locked decisions (do NOT re-negotiate without user approval)

> [!success] Architecture (from design + post-v0.2.0 conventions)
> - **`daily` blueprint shipped alone in v0.3.0.** No sibling blueprints (meetings, to-do). Dashboard tolerates missing siblings (renders empty sections; never errors).
> - **Verbatim accuris daily pattern.** Path `beacon/daily/YYYY/MM-MMMM/YYYY-MM-DD-DayOfWeek.md`; frontmatter `{ created, tags: [<vault_identity_tag>, daily, YYYY/MM/DD], cssclasses: [wide] }`; body two dataviewjs blocks (`SpaceNavButtons` + `SpaceDailyDashboard`) + `## Notes`.
> - **Entry-point UX is the core Daily Notes plugin** — built-in "Open today's daily note" command + Calendar community plugin for browse-by-day. NO custom slash command, NO custom Templater hotkey, NO smart-Today nav-button. Mirrors how the user already uses accuris.
> - **Allowlist expands by ONE path** — `.obsidian/daily-notes.json`. Calendar plugin's data.json is NOT added (Calendar reads daily-notes.json directly; its own data.json is only weekly-notes + display prefs).
> - **NEW manifest field `core_plugin_settings[]`** declaring core-plugin data files to write. Schema: `[{ id, settings }]`. Substitution applies to settings values (per-blueprint variables overlay carries `{{module_directory}}` + `{{templates_path}}`).
> - **NEW helper `applyCorePluginSettings`** mirroring v0.1.3 helper posture (additive merge, backup-on-edit to `<target>.beacon-backup`, malformed-JSON guard, failure-loud history). Wired in `installItem` after `applySlashCommanderBindings`.
> - **`SpaceDailyDashboard` ported with hardcoded paths** to `beacon/meetings` + `beacon/to-do`. Future per-consumer override deferred until needed.
> - **Workshop does NOT subscribe to daily.** Same posture as project + boards — barebones-only dogfood. No personal content in workshop.
> - **Deploy mechanism:** v0.1.2 stub (already in place at md5 `a39257da...`).
