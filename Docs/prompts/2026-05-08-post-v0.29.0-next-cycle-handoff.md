### Option A — vault baseline rollup (multi-session work-list)

> **Recommended first.** Land a clean baseline before adding more functionality on top.

Start the post-v0.29.0 cleanup work tracked at `Docs/plans/2026-05-08-vault-baseline-rollup.md`. Use `sauce audit` reports as the work-list. **Multi-session; not a workshop release cycle** — no version bump, no tag, no harness changes. Each session edits vault files in place (or via Claude making edits directly) and appends a delta entry to the rollup doc.

**Done criterion:** all 3 vaults (accuris-sauce / ero-sauce / headspace-sauce) pass `sauce audit --quiet` with exit 0 (or every untracked dir is intentional + accepted as user-owned residue, which would be allowlisted in a future `audit-allowlist.json` if v0.29.1 ships that mechanism).

**Per-session protocol** (per `Docs/plans/2026-05-08-vault-baseline-rollup.md`):

1. **Snapshot first** — copy target vault to `<vault>.pre-cleanup-<YYYYMMDD-HHmmss>/`. Mirrors v0.28.0 `.pre-migration-<ts>/` discipline.
2. **Run `sauce audit <vault> --output-file <vault>/ranch/audits/<YYYY-MM-DD-HHmmss>-audit.md`** to capture starting state.
3. **Walk the audit report top-down**, fix violations one by one. Claude makes file edits directly. Each fix is committed (if vault is git-managed) or simply persisted to disk + Obsidian Sync.
4. **Re-run audit** at session end. Capture closing report.
5. **Append delta-summary to `Docs/plans/2026-05-08-vault-baseline-rollup.md`** (NEW per-session entry with: vault, before-violation-count, after-violation-count, what-was-fixed, what-remains).

After Option A reaches "done" criterion across 3 vaults, **then** pick Option B or Option C as the next workshop release cycle.
