## Phase 1 — FIX-LATER-NOTE triage

Walk each of the 9 FIX-LATER-NOTEs from the result doc. The triage decision for each is below; apply the must-fix items BEFORE running the headspace test.

### a — `_loadSpec` error masking (MUST FIX before headspace test)

**What:** `_loadSpec` returns `null` for both "file missing" and "malformed JSON" — the user sees silence on a parse error. `_promptText` has a missing type-guard on the optional-branch path (line ~174 of `EntityCreate.js`).

**Fix (3-line):** In `platform/mechanisms/entity-create/EntityCreate.js`, `_loadSpec`'s catch block should `console.error("entity-create: malformed registry JSON — " + e.message)` + `new Notice("entity-create: registry JSON parse error — check console", 8000)` before returning `null`. For `_promptText`: add a `typeof val !== 'string'` guard before the empty-string check on the optional branch.

After the fix: bump `entity-create@0.1.0 → 0.1.1` in `platform/mechanisms/entity-create/manifest.json` AND in `platform/manifest.json`'s mechanisms catalogue. Run `npm run release:preflight` — must exit 0. Commit:

```
feat(entity-create): v0.46.1-fix-a — _loadSpec parse-error Notice + _promptText type-guard
```

### b — S1 polish / M-items (DEFER)

Items M1–M8 are cosmetic or low-risk graceful-fallback patterns with no functional impact. No action this session. Flag as deferred in the result doc if desired, but no code change.

### c — S2 regex edge cases (DEFER)

## Phase 3 — End-to-end entity-create test flows

Open Obsidian and navigate to the headspace vault. Test each flow that headspace subscribes to. The acceptance criteria below apply per flow.

### Flow 1 — meetings / New Meeting

**Entry point:** `spice/meetings/Meeting Hub.md` — click the `+ New Meeting` AccentButton.

**Input:** Provide a safe filename for the title (e.g., `Test Sync 2026-05-15`).

**Expected outputs:**
- File created at `spice/meetings/notes/YYYY/MM-MMMM/<title>-YYYY-MM-DD.md` (where YYYY-MM-DD is today's date).
- Frontmatter: `date: <today>`, `type: meeting`, `summary: ""`, `attendees: []`, `tags: [meeting]`, `cssclasses: [wide]`.
- Body: SpaceNavButtons dataviewjs block visible.
- **Path C note (NOT a bug):** No Templater attendees prompt fires. The Templater Meeting template is for manual "Insert template" use only. Acknowledge this as documented posture.

**Idempotency check:** Click the button again with the SAME title. Expected: opens the existing file; does NOT overwrite it.

**Reject safe-filename:** Enter a title containing `/` (e.g., `Bad/Name`). Expected: Notice fires, no file created.

### Flow 2 — people / New Person

**Entry point:** `spice/people/People.md` — click the `+ New Person` AccentButton.

**Input:** Provide a person name (e.g., `Alice Testperson`).

**Expected outputs:**
- File created at `spice/people/Alice Testperson.md`.
- Frontmatter: `type: person` (plus any manifest-declared fields).
- Body: PersonNavButtons dataviewjs block; mention-list view present.

**Idempotency check:** Re-enter the same name — existing file opens, no overwrite.

**Reject safe-filename:** Enter `Bob/Invalid`. Expected: Notice fires, no file created.

### Flow 3 — project / New Project

**Entry point:** `spice/projects/Projects.md` — click the `New Project` AccentButton.

**Input:** Provide a project name (e.g., `Test Project Alpha`).

**Expected slug derivation:** slug auto-derives to `test-project-alpha` (slugify + lowercase).

**Expected outputs:**
- Atlas at `spice/projects/test-project-alpha/Test Project Alpha.md` with manifest's `frontmatter_template` fields (status: idea, status_changed_at: today, workstreams: [], etc.).
- Map sidecar at `spice/projects/test-project-alpha/Project Map.md`.
- Board sidecar at `spice/projects/test-project-alpha/test-project-alpha-board.md`.
- Three-file creation is the `extra_files[]` mechanism — verify all three appear in the vault.

**Idempotency check:** Re-enter the same name. Expected: atlas file opens; sidecars not re-created.

### Flow 4 — scratch / New Scratch

**Entry point:** Today's Scratch Day Hub (`spice/scratch/YYYY/MM-MMMM/YYYY-MM-DD.md`) — click the `+ New Scratch` AccentButton.

**Expected outputs:**
- File created at `spice/scratch/YYYY/MM-MMMM/YYYY-MM-DD/Scratch-YYYY-MM-DD-HH-mm.md`.
- Frontmatter includes `day` field populated from `{{current_file.frontmatter.day}}` substitution.
- Body: inline_body content (customjs-guard block or equivalent per manifest's `inline_body` value).

**Path C note (NOT a bug):** No Templater auto-move fires. Path C acknowledged.

**Idempotency check:** Two scratches in the same minute should produce different HH-mm suffixed files (normal behavior — not strictly idempotent in the traditional sense; verify the minute counter works).

### Flow 5 — finance / New Budget

**Entry point:** `spice/finance/Finance.md` (or the finance hub) — click `New Budget`.

**Input:** Month prompt. Enter `2026-05` (month format).

**Expected outputs:**
- File created at `spice/finance/budgets/2026-05/Budget-2026-05.md`.
- Frontmatter: `type: budget`, `month: 2026-05` (or per manifest's `frontmatter_template`).
- Body: inline_body content.

**Idempotency:** Re-enter `2026-05`. Expected: existing file opens.

### Flow 6 — finance / New Paycheck

**Entry point:** Finance hub — click `New Paycheck`.

**Inputs (3 prompts in order):**
1. `start_date` — enter a date (e.g., `2026-05-01`).
2. `end_date` — enter a date >= start_date (e.g., `2026-05-15`). Entering a date BEFORE start_date should fire a Notice (gte:start_date validate predicate).
3. `amount` — enter a positive number (e.g., `3200`). Entering a negative number should fire a Notice (min:0 validate predicate).

**Expected outputs:**
- File created with expected path (per manifest — likely `spice/finance/paychecks/<start_date>--<end_date>.md`).
- Frontmatter: `paycheck_amount: 3200` — emitted as an **unquoted numeric YAML scalar** (not `"3200"`). This is the `|number` substitution pipe. Verify the frontmatter raw text in the file.

**Idempotency:** Re-enter same dates. Expected: existing file opens.

### Flow 7 — finance / New Invoice

**Entry point:** Finance hub — click `New Invoice`.

**Input:** Month prompt (e.g., `2026-05`).

**Expected outputs (3 files via `extra_files[]`):**
- Atlas at `spice/finance/invoices/2026-05/Invoice-2026-05.md`.
- Time-Log sidecar at `spice/finance/invoices/2026-05/Time-Log-2026-05.md` (or per manifest's `extra_files[]` path).
- Board sidecar (kanban sub-folder) at `spice/finance/invoices/2026-05/board/` or equivalent.

All three files should exist. Verify paths against the manifest's `extra_files[]` entries in `platform/blueprints/finance/manifest.json`.

**Idempotency:** Re-enter `2026-05`. Expected: atlas file opens; sidecars not re-created (idempotent).
