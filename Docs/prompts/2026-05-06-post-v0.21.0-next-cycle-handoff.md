> - **Two backup-suffix conventions (v0.19.0 lesson e)** — `.bak` for file-content overwrite (vendored themes), `.beacon-backup` for plugin-data additive merge.
> - **Vendored theme is mechanism-owned (landmine #15)** — never hand-edit `.obsidian/themes/<Name>/`.
> - **Test isolation via inline synthetic bodies (v0.20.0 lesson a)** — when a harness test reads a blueprint asset for fixture content, prefer an inline synthetic body over `fs.readFileSync` of a real path.
> - **Cross-doc line-citation drift is a real maintenance class (v0.20.0 lesson c)** — any cycle that touches a file cited as `<filename>:<lineno>` in any doc should grep all docs for that pattern.

> - **`process.exit` pipe truncation (v0.21.0 lesson b)** — for any new CLI tool that may be piped: use `process.exitCode = N` + return; never `process.exit(N)` mid-async-work.
> - **Async `spawn` over `spawnSync` when capturing child stdout (v0.21.0 lesson c)** — `close` event fires after stdout EOF only on async.
> - **Canonical helper posture stack (v0.21.0 lesson d)** — atomic write + backup-on-overwrite + path-traversal validator (when ids flow from external sources). Adopt up-front, not at quality-review time.
> - **Defer-then-validate-then-load wizard architecture (v0.21.0 lesson e)** — when a config field gates downstream prompts, gate the load behind the field's prompt+validator.
