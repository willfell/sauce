## Step 3 — what wave 2 ships

> [!abstract] Scope — v0.33.0
> Three blueprints adopt `claude_surface[]`. Each ships canonical slash command + SKILL.md bodies authored in `platform/blueprints/<bp>/{commands,skills}/` and declared in the blueprint manifest. On install, consumer-side `.claude/commands/<bp>.md` is overwritten with the canonical (this is the cohesion fix — the original drift the project was created to solve).

### Per-blueprint deliverables

| Blueprint | Version bump | Slash command | SKILL.md bodies | claude_md_row contributions |
|---|---|---|---|---|
| `project` | `1.4.1 → 1.5.0` MINOR | `/project` | `new-project/SKILL.md` (other sub-skills if natural) | resolvers + skills-index + directory-map row for `spice/projects/` |
| `daily` | `0.2.6 → 0.3.0` MINOR | `/daily` | `morning/SKILL.md` + `eod/SKILL.md` (or whatever the natural sub-skills are) | resolvers + skills-index + directory-map row for `spice/daily/` |
| `meetings` | `0.3.1 → 0.4.0` MINOR | `/meetings` | `new-meeting/SKILL.md` + similar | resolvers + skills-index + directory-map row for `spice/meetings/` |

### What "canonical body" means

For each `/project`, `/daily`, `/meetings`:

1. **Locate the consumer-side hand-maintained body.** The original Phase-1 audit found stale bodies in `accuris-sauce` at `spice/projects/sauce-claude-cohesion/tasks/Phase 1 Inventory Report/Phase 1 Inventory Report.md`. Use that as the authoritative starting point for what the body SHOULD describe.
2. **Update it to current blueprint layout.** Project: `spice/projects/<slug>/` (NOT `boards/planning/<slug>/`). Templates: `ranch/templates/Template, Project.md` etc. Scripts: `ranch/scripts/project/*.js`. Use whatever `platform/blueprints/project/manifest.json` actually ships.
3. **Author in workshop, NOT consumer.** Final canonical lives at `platform/blueprints/project/commands/project.md` + `platform/blueprints/project/skills/<name>/SKILL.md`.
4. **Reference real CLI verbs only.** Apply the lesson from S6: any `sauce <verb>` in the body must match `platform/cli/sauce-cli.js` VERBS table. The `CS-SUB-3` regression guard in `run-claude-surface.js` catches drift for `platform-claude`; consider extending it to walk all materialized command/skill bodies regardless of mechanism.
5. **Include the version comment.** Each command + SKILL body opens with `<!-- @claude-surface:version <X.Y.Z> -->` (matches the blueprint's manifest version). This lets `/audit` emit `stale_but_valid` findings when a deployed body falls behind.

Fix in `platform/mechanisms/platform-claude/claude-md-renderer.js` (move `## Title` outside marker OR include in both branches). Touches the renderer; wave 2 will regen CLAUDE.md many times.
- **S2 ranch/ allowlist breadth** — currently any subscriber can write anywhere under `ranch/`. Narrow to `ranch/claude/` OR document the collision-coordination expectation. Touches `platform/install.js:aggregateClaudeSurface`.
- **CS-SUB-3 generalization** — today only checks platform-claude bodies for valid CLI verbs. Generalize to walk ALL materialized command/skill bodies. Would automatically catch wave-2 verb drift.

### Triage but don't necessarily fix in wave 2
- **S4 table-cell escaping** — `|` and newlines in `claude_md_row` cells break the rendered table. Document the constraint OR add escaping in renderer's `rowFn`.
- **S4 multiple-marker-pair detection** — duplicate `BEGIN ... END` in CLAUDE.md silently orphans the second pair (first wins).
- **S2 unknown_kind silent skip** — `kind: "comand"` (typo) drops the entry without warning. Add `claude_surface_unknown_kind` warning event.
- **S2 dest path normalization** — `dest: ".claude//commands/foo.md"` may bypass allowlist.
- **S5 redundant null check** + magic-string `.local/` root constants.
- **S7 stale_but_valid + consumer_edit_at_risk mutually-exclusive** — intentional; undocumented.

### Pre-existing inconsistencies (NOT in v0.32.0 scope, surface during wave 2)
- **Workshop subscription drift:** `ranch/platform-subscription.json` pins `validator@0.1.1` + `nav-buttons@2.5.3`, but catalogue lists `0.1.2` + `2.6.0`. Self-install emits skip events. Consider reconciling early in wave 2 since wave 2 will run more self-installs.
