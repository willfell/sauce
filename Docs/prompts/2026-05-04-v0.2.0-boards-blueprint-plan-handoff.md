### 3. Manifest schema additions — tighter than the design

Design proposed an `external_plugins[]` field (Open call 3). v0.1.3 ALREADY shipped this field with a more polished shape (`{ id, required, reason }` + read-only check against `.obsidian/community-plugins.json`). The boards manifest should declare:

```json
    "id": "obsidian-kanban",
    "required": true,
    "reason": "Renders boards/To-Do-Board.md as an interactive kanban; falls back to plain markdown without the plugin."
  }
]
```
Verify the actual community-plugin id (`obsidian-kanban` is a guess; check Obsidian's community plugin browser before locking).

### 4. Open Calls 1–3 from the design — still need user resolution

The design has three open design calls (card-notes path, button id namespacing, external-plugin declaration mechanism). #3 is now closed by v0.1.3 (use the canonical `external_plugins[]` shape). #1 + #2 still need the user to pick.
