## Candidate cycles (pick ONE)

> [!example]- v0.4.2 runTemplaterTemplate schema robustness **(Strong candidate)**
> **What:** address the bracket-escape gap surfaced TWICE now (v0.4.0 FIX-1 substitute-bracket; v0.3.2 FIX-2 author-discipline-bracket). Two design paths:
>
> **(a) Heuristic extension to substituteForMomentFormat** — scan input for unbracketed letter-sequences and bracket-wrap them. Brittle: hard to distinguish "this is a literal that needs escaping" from "this is intentional moment format tokens." Probably rejected at design.
>
> **(b) Explicit two-field schema (Recommended for the cycle)** — replace the single `folder` + single `filename` action fields with two pairs:
> - `folder_prefix: "<literal-path>"` (NOT moment-formatted; substituteLenient applied)
> - `folder_date_pattern: "<moment-format>"` (moment-formatted at click-time)
> - `filename_prefix: "<literal>"` + `filename_date_pattern: "<moment-format>"` + `filename_suffix: "<literal>"` (or a similar split)
>
> Mirrors Obsidian Daily Notes plugin's clean separation. Renderer composes `${folder_prefix}/${moment.format(folder_date_pattern)}` etc. Author-discipline burden moves from "remember to bracket-escape" to "split your fields correctly" — clearer + harder to get wrong.

> **Estimated scope:** 1-2 stage cycle. nav-buttons mechanism 2.1.0 → 2.2.0 (BREAKING for action shape). validateAndResolve updated. Renderer updated. R1 harness case rewritten. v0.4.0's to-do + v0.3.2's meetings manifests both migrated to new schema. Workshop bump 0.8.1 → 0.9.0 MINOR (mechanism break).

> **Why "settle now" thinking favors this:** the bug surfaced twice; third occurrence justifies the schema bump per the existing doctrine. The current bracket-escape discipline is documented but NOT enforced — every future blueprint author must remember it. The split-field schema makes the bug architecturally impossible.

## Cross-cutting gotchas codified during recent cycles (READ BEFORE STARTING)

> [!warning] Gotchas surfaced 2026-05-03 / 2026-05-04 — load-bearing for any cycle touching runTemplaterTemplate or CustomJS classes
>
> **Gotcha 1: moment.format token corruption on literal text.** The renderer's `window.moment().format(<folder>)` interprets ANY letter in the format string as a potential moment format token. Substituted `{{...}}` values are auto-bracket-wrapped by `substituteForMomentFormat` (v0.4.0 FIX-1). Non-substituted literals are NOT — author must manually `[bracket-escape]`. Example trap: `folder: "{{module_directory}}/hubs"` → moment.format → `beacon/meetings/11ub1` because `h`/`s` are tokens. Fix: `{{module_directory}}/[hubs]`. Carry as v0.4.2 systemic-fix candidate.
>
> **Gotcha 2: Node-harness install + Obsidian's plugin auto-rescan.** `platform/test/run-install.js` writes via `fs` (Node), bypassing Obsidian's vault adapter. Obsidian's plugins (CustomJS, Templater, etc.) watch the vault adapter for change events to auto-rescan their jsFolder/templates_folder/etc. After a Node-harness install adds new files (CustomJS classes, templates), the user MUST Cmd+R Obsidian to force plugin re-init + re-registration. Smoke checklists for any cycle adding new CustomJS classes should explicitly call out the Cmd+R step.
>
> **Gotcha 3: T-FIX1 mkdir module_directory at install time** (v0.3.0 codified). Every blueprint with `module_directory` set gets that directory mkdir'd between `applyPreInstall` and `files[]` loop. Mechanisms exempt (no module_directory). Future blueprints whose `files[]` doesn't write under `{{module_directory}}/...` (siblings of daily — to-do, meetings, journal, retros, etc.) inherit this guarantee for free.
>
> **Gotcha 4: Force re-process via installed.json edit (CLAUDE.md ask-before-acting).** `install.js:111` short-circuits per-item install when `subscribed.version === installed.version`. Tests/cycles that need to exercise behavior INSIDE `installItem()` (e.g., to verify a fix landed) must EITHER triple-bump versions (item manifest + workshop manifest entry + consumer subscription — landmine #10) OR drop the entry from `<consumer>/Docs/Meta/platform-installed.json:installed[]` (CLAUDE.md ask-before-acting; user-approved). Both paths used in v0.4.0 FIX-1 + v0.3.2 FIX-2.
>
> **Gotcha 5: User-override preserved on folder_templates + nav-button registry overwrites.** `applyTemplaterFolderTemplates` (v0.4.0) preserves user-customized template paths if folder matches. `applyNavButtons` REPLACES the contributions[<source>] array on each install (not append). When migrating layout (e.g., v0.3.2's notes/ + hubs/ subfolders), legacy `folder_templates` entries from prior installs may persist as user-overrides — cosmetic clutter but functionally inert.
>
> **Gotcha 6: Mid-cycle redesign methodology.** When smokes surface a misalignment with the user's mental model, ACCEPT the redesign rather than ship the wrong primitive. Workflow: pause subagent-driven-development → invoke `de:brainstorming` → present revised design → get approval → invoke `de:writing-plans` → resume `de:subagent-driven-development`. v0.3.2 used this flow successfully (v1 one-click-meeting → v2 hub-centric). Stage-close `AskUserQuestion` checkpoints are the right gate; smokes are the right surface for redesign signals.
