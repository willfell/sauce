## Cross-cutting gotchas codified during recent cycles (READ BEFORE STARTING)

> [!warning] Gotchas surfaced 2026-05-03 / 2026-05-04 — load-bearing for any cycle touching runTemplaterTemplate, CustomJS classes, invoke_command, new renderer action_types, or BeaconCards

> **Gotcha 1: Bracket-escape gap on runTemplaterTemplate folder/filename.** RESOLVED at v0.4.2 S1 — split-field schema makes literal text never reach `moment.format()`. Manifests still using legacy `folder` / `filename` fields are silently ignored (BREAKING). Future blueprints inherit the safety for free (journal@0.1.0 v0.13.0 confirmed; date-aware RTT dispatcher v0.14.0 inherited the safety transparently).
>
> **Gotcha 2: Node-harness install + Obsidian's plugin auto-rescan.** `platform/test/run-install.js` writes via `fs` (Node), bypassing Obsidian's vault adapter. Obsidian's CustomJS plugin watches the vault adapter for change events to auto-rescan its jsFolder. After a Node-harness install adds new files OR rewrites class bodies, the user MUST Cmd+R Obsidian to force plugin re-init. Smoke checklists for any cycle adding OR rewriting CustomJS classes should explicitly call out the Cmd+R step. **v0.13.0 + v0.14.0 reinforced:** even one-line ICONS additions to `space-nav-buttons.js` need a Cmd+R; even mechanism patches with no new files but body-only changes need a Cmd+R for the existing class to reload.
>
> **Gotcha 3: T-FIX1 mkdir module_directory at install time** (v0.3.0 codified). Every blueprint with `module_directory` set gets that directory mkdir'd between `applyPreInstall` and `files[]` loop. Mechanisms exempt. **v0.13.0 reinforced:** journal blueprint has no install-time files under its module_directory (date-routed children land at runtime), but `beacon/journal/` still gets mkdir'd. **v0.14.0 unchanged** — pure mechanism patch, no module_directory surface.
>
> **Gotcha 4: Force re-process via installed.json edit (CLAUDE.md ask-before-acting).** `install.js:111` short-circuits per-item install when `subscribed.version === installed.version`. Tests/cycles that need to exercise behavior INSIDE `installItem()` must triple-bump versions OR drop the entry from `<consumer>/Docs/Meta/platform-installed.json:installed[]` (CLAUDE.md ask-before-acting; user-approved).
>
> **Gotcha 5: User-override preserved on folder_templates + nav-button registry overwrites.** `applyTemplaterFolderTemplates` preserves user-customized template paths. `applyNavButtons` REPLACES the contributions[<source>] array on each install. Existing `folder_templates` entries from prior installs may persist as user-overrides — cosmetic clutter but functionally inert.

Workflow: pause SDD → invoke `de:brainstorming` → present revised design → get approval → invoke `de:writing-plans` → resume SDD. Used four times historically (v0.3.2 v1→v2; v0.4.2 v1→v2; v0.5.0 inline S2.x sub-stages; v0.12.0 S3.4.1 tasks-as-bullets revert). v0.13.0 + v0.14.0 did NOT need a mid-cycle redesign (both clean single-stage cycles). Pattern is reproducible.
>
> **Gotcha 7: Mobile-aware visual design as platform value (v0.4.2; reinforced v0.5.0 + v0.11.0 + v0.12.0 + v0.13.0 + v0.14.0).** Every renderer (CustomJS class) MUST be designed for both desktop AND mobile. BeaconCards 0.2.x (chip-collapse on small screens; row-layout adapts; subtitle two-line-wrap word-break preserved) is the canonical pattern reference. > **Gotcha 8: Filename pattern `<Name>-YYYY-MM-DD` invariant (v0.4.2; reinforced v0.13.0 + v0.14.0).** All date-routed daily artifacts use date-suffix; hyphen-only separator; no spaces. Date-extraction in CustomJS classes uses regex `\d{4}-\d{2}-\d{2}`. **NOTE:** project artifacts are slug-routed (NOT date-routed) — gotcha 8 does NOT apply to project files. **v0.14.0 reinforced:** the regex `/(\d{4}-\d{2}-\d{2})/` was reused VERBATIM across the click-time RTT dispatcher (L300-358 of `space-nav-buttons.js`) — the same regex from L107-122 daily-nav top-row applied to a different code path with ZERO divergence. Future renderer features that need to extract a date from a filename should reuse the same regex verbatim — not reinvent.
