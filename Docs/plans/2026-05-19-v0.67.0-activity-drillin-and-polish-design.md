# v0.67.0 — Activity Drill-In + Polish — Design

> Brainstorm origin: 2026-05-19 conversation, triggered by user smoke of the v0.66.x Activity panel at `/Users/willfellhoelter/notes/sauce/accuris-sauce/spice/daily/2026/05-May/Tuesday-2026-05-19.md`. Screenshot in conversation thread.
>
> **Status:** design, awaiting user spec-review. No code touched.

## 1. Context

The v0.66.x arc (3 releases in one evening: MINOR + 2 PATCHes) shipped Activity Dashboard Cohesion — flat-grouped panel layout, per-card meta line (time · type pill · breadcrumb), project + trip rollup, smart title resolver. v0.66.x closed with 7 FLNs captured. User smoke at accuris-sauce surfaced visible gaps that this cycle resolves.

Concrete issues observed in the screenshot:

1. **Every card shows `12:00 AM`** regardless of underlying `created_at`. FLN-v66-7 — Dataview parses ISO frontmatter into Luxon DateTime objects; `window.moment(luxonObj)` silently produces an invalid value that `.format("h:mm A")` renders as `12:00 AM`. Visible on every Activity card, every day.
2. **`7 children touched` is static text.** The rollup count is informative but there's no way to peek at WHICH 7 child notes were edited. Obvious next-click is missing.
3. **Activity icon (`zap` lightning bolt) feels off-tone** relative to the panel's review-y character.
4. **Activity section defaults to collapsed** (v0.66.1 decision) but the user found it adds friction — they want it open when there's content to see.
5. **Long single-color left accent line** loses visual impact when the section holds multiple blueprint groups.
6. **No surface for open todos.** A meeting or scratch with unchecked tasks looks identical to one without — easy to forget follow-ups.
7. **"children touched" wording is jargon-y.** Reads better as "notes touched".

Bundled together: a single MINOR cycle that closes one regression (FLN-v66-7), adds one new feature (drill-in), and applies several small polish items. All scoped to the daily-note dashboard.

## 2. Decisions captured from brainstorm

| Axis | Decision | Reason |
|---|---|---|

```js
_formatTime(tsRaw) {
  if (!tsRaw) return null;
  try {
    if (typeof tsRaw.toFormat === "function") return tsRaw.toFormat("h:mm a");  // Luxon
    const m = window.moment(tsRaw);
    if (m && m.isValid()) return m.format("h:mm A");
  } catch (_) {}
  return null;
}
```

`_renderActivityMeta` now calls `this._formatTime(tsRaw)` and renders only when non-null.

### 5f. Drill-in expansion (NEW behavior on rollup cards)

Inside `_renderActivityMeta`, the breadcrumb element becomes a click target.

Mechanics:
- Breadcrumb HTML becomes `· N notes touched ▸` (chevron-right SVG appended; rotates 90° on expand via CSS).
- A sibling `<div class="sauce-drill-in" hidden>` is appended inside `parentEl`. CSS `display: block` (when not `[hidden]`) makes it wrap below the inline meta items.
- Click handler on the breadcrumb toggles `[hidden]` + `data-expanded` on the breadcrumb.

New helper `_renderDrillInList(parentEl, children, rootPath)`:
- Sorts children by `file.mtime` DESC (uses Luxon `.toISO()` for sort key — mirrors `pickLatest` pattern in activity-feed).
- Caps at 12 visible. Overflow shows as `+N more` (non-interactive label for v0.67.0).
- Each row: `<a href="#" class="sauce-drill-row">` with `<span class="sauce-drill-name">` left + `<span class="sauce-drill-time">` right.
- Relative path computation: take `child.file.path`, strip the leading `<rootPath-dirname>/` prefix, strip the trailing `.md` extension. Example: `child.file.path = "spice/projects/k8s-mig/tasks/auth.md"` with `rootPath = "spice/projects/k8s-mig/Project.md"` → dirname `spice/projects/k8s-mig/`, strip → `tasks/auth.md`, strip ext → `tasks/auth`. Edge case: if `child.file.path` does not start with the rootPath dirname (shouldn't happen for properly-bucketed rollups but defensive), fall through to `child.file.name` without extension.
- Click handler: `ev.preventDefault(); ev.stopPropagation(); app.workspace.openLinkText(child.file.path, "")`. Obsidian `app` global available in dataviewjs context (existing pattern in nav-buttons + accent-button).

### 5g. Universal open-todo badge (NEW behavior on Meetings + Activity cards)

New helper `_renderTodoBadge(p, parentEl)`:

```js
_renderTodoBadge(p, parentEl) {
  const tasks = p?.file?.tasks;
  if (!Array.isArray(tasks)) return;
  const open = tasks.filter(t => t && !t.completed).length;
  if (open <= 0) return;
  const pill = parentEl.createEl("span");
  pill.className = "sauce-todo-pill";
  pill.title = `${open} open task${open === 1 ? "" : "s"}`;
  pill.innerHTML = `<span class="sauce-todo-icon">${icons.square}</span><span class="sauce-todo-count">${open}</span>`;
}
```

Wired in TWO places:
- **Meetings panel** (line 209-220): add `meta: (p, el) => this._renderTodoBadge(p, el)` to the `BeaconCards.render` call.
- **Activity `_renderActivityMeta`**: call `this._renderTodoBadge(p, parentEl)` after the type pill, before the breadcrumb. Visual order: `time · pill · ☐ N · N notes touched ▸`.

### 5h. Segmented accent line (Activity only)

- Extend `_getActivityCount` → returns `{total, byBlueprint: {project: 1, scratch: 5, ...}}`.
- New helper `_buildAccentSegments(byBlueprint)` returns a `linear-gradient(to bottom, ...)` string:

```js
_buildAccentSegments(byBlueprint) {
  const entries = Object.entries(byBlueprint).filter(([, n]) => n > 0);
  if (entries.length === 0) return null;
  const total = entries.reduce((s, [, n]) => s + n, 0);
  const colors = this._BLUEPRINT_PILL_COLORS || {};
  const stops = [];
  let cursor = 0;
  for (const [type, count] of entries) {
    const color = colors[type] || "var(--color-base-50)";
    const start = (cursor * 100).toFixed(2) + "%";
    cursor += count / total;
    const end = (cursor * 100).toFixed(2) + "%";
    stops.push(`${color} ${start}, ${color} ${end}`);
  }
  return `linear-gradient(to bottom, ${stops.join(", ")})`;
}
```

- `_renderSection` accepts a new optional `accentSegments` string param; when present, sets `section.dataset.segmented = "true"` and `section.style.setProperty("--sauce-accent-segments", accentSegments)`.
- Activity call site passes `accentSegments: this._buildAccentSegments(activityByBlueprint)`. If `_buildAccentSegments` returns `null`, `_renderSection` falls through to the single-color path (defensive).
- Tasks + Meetings unchanged — single-blueprint sections retain the existing `border-left` rule.

## 10. Risk register

| Risk | Severity | Mitigation |
|---|---|---|
| Drill-in click intercepts card click meant for "open hub" | Medium | `ev.stopPropagation()` on breadcrumb click; card's title click target unaffected. |
| Todo badge renders on cards with completed-but-present tasks | Low | `tasks.filter(t => !t.completed)` excludes correctly; assert covers. |
| `_rollUpChildrenPages` references Dataview page objects GC'd between render and click | Low | Page objects held in closure for lifetime of rendered DOM; Dataview keeps index alive in session. |
| Segmented gradient renders weirdly when one group dominates (1 + 50) | Low | Sub-1% segment still visible as a few px; mitigation `min-segment-px` enforcement deferred to follow-up if observed. |
| `app.workspace.openLinkText` unavailable in customJS context | Low | Obsidian `app` global is accessible; existing nav-buttons + accent-button code uses the same pattern. |
| Subscription-lockstep foot-gun (FLN-v66-6) | Medium | S7 commits subscription BEFORE running installer; pre-flight `git status` check before `scripts/sauce update --force`. |
| Activity icon swap breaks an external consumer | Very Low | `icons.zap` only consumer is line 226 in this file; no external customJS reference. |
| Query-semantics divergence between `_getActivityCount` and `ActivityFeed.render` | Medium | The segmented bar is sized from the `byBlueprint` counts returned by `_getActivityCount` (called pre-render). `ActivityFeed.render` does its own query. If the two diverge, the segmented bar won't match the displayed groups. Mitigation: keep `_getActivityCount` mirroring `ActivityFeed._query` semantics (existing invariant in the file's docstring); extend it conservatively (groupBy=blueprint must include rollup coalescing if it ever affects counts). S6 includes a regression assert that pre-flight `byBlueprint` matches a post-render group count where feasible. |
