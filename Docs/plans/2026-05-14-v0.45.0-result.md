# v0.45.0 — cowork self-contained + UX polish wave 2 (CLOSED 2026-05-14)

## Summary

After v0.44.0 landed in `accuris-sauce`, the user surfaced six follow-on issues that clustered into one architectural concern and five UX-consistency concerns. v0.45.0 fixes all six in one MINOR.

**Architectural change:** cowork is now a self-contained system. NEW path convention `spice/cowork/daily/YYYY/MM-MMMM/YYYY-MM-DD.md` mirrors the daily blueprint's folder shape but lives inside cowork's own module-directory (landmine #11 invariant). The Daily Hub cards-helper retargets from `dv.pages('"spice/daily"')` → `dv.pages('"spice/cowork/daily"')`. The cowork orchestrators will be rewired to write into this new path family in a future scheduling/aggregation cycle; v0.45.0 ships the shape + the manual creation flow.

**UX-consistency changes:** CoworkHubNav rewrites from BeaconCards 4-card row → AccentButton centered flex row matching ScratchDayActions aesthetic; drops "you are here"; the current-hub button is OMITTED so each hub renders 3 buttons pointing at the others. Universal nav-pattern (SpaceNavButtons → --- → CoworkHubNav → --- → content) lands on every cowork content artifact (5 hubs + 3 note templates = 8 files). Weekly/Monthly/Daily Note templates compute friendly H1 titles via Templater (`Week 20 · May 11–17, 2026` / `May 2026` / `Thursday, May 14, 2026`). The Timeframes row on `Cowork.md` grows 5 → 6 cards: `Daily Hub | Today | Weekly Hub | This Week | Monthly Hub | This Month`.

Net: `cowork@0.6.0 → 0.7.0` MINOR + workshop `0.44.0 → 0.45.0` MINOR. nav-buttons unchanged at 2.6.2. Tag `v0.45.0` annotated + pushed; release.yml preflight → bump-tap chain auto-bumps `Formula/sauce.rb` in `willfell/homebrew-sauce` on tag push.

## Plan vs reality

### Plan-ordering correction validated

The v0.44.0 lesson — "catalogue lockstep is prerequisite for the preflight gate" — was baked into this plan's S8 (catalogue + harness + preflight together, NOT S9). Result: S8 was the green-or-stop gate, S9 was a trivial release commit. No mid-cycle plan patches needed this time.

### S5 substitution-token vs literal-path inconsistency

Prior Weekly/Monthly Note templates used `{{vault_identity_tag}}` and `{{views_path}}` Templater substitution tokens; new templates use literal values per the plan body. Both forms substitute identically post-install (installer's `substituteLenient` is a no-op for already-literal paths). Same FLN-1 from v0.44.0; logged again here as FLN.

### S8 sub-assert overage

Plan target: ~+24 sub-asserts. Actual: roughly **+40** (run-cowork-smoke +30, run-helper-cases +8, run-audit +2, run-claude-surface stale-bump only, run-integration-smoke +1). The agent's judgment was sound: per-case bodies in run-cowork-smoke carry 4–6 internal asserts; the navPatternFiles loop covers 8 files × 2 asserts = 16 alone. All asserts load-bearing.

### S8 audit rule-code adaptation

The plan's CW-16 wrong-type assertion specified rule code `required_frontmatter.type`, but the rule-runner emits `required_frontmatter.type.equals` for an `equals` mismatch. The implementer updated the assertion to match the actual rule code — the assert still catches the type-mismatch defect, just under the correct rule name.

## FIX-LATER-NOTEs

1. **FLN-1 (carried from v0.44.0): substitution-token vs literal-path inconsistency.** Daily/Weekly/Monthly Note templates use literal `ranch/views/customjs-guard` paths and hardcoded `cowork-*` tags rather than `{{views_path}}` + `{{vault_identity_tag}}` substitution tokens. Functionally equivalent post-install (installer substitutes lenient). Cosmetic only. Candidate for a small housekeeping cycle.

2. **FLN-2 (pre-existing, carried from v0.42.0+): subscription pins project@1.5.0 but catalogue has project@1.6.1.** Caused `sauce reinstall --vault $PWD` to exit 1 with one skip during S9 dogfood. Not in scope for v0.45.0; same FLN flagged in v0.42.0/v0.43.0/v0.44.0 narratives.

3. **FLN-3: Cowork orchestrators' SKILL.md bodies still reference the daily blueprint's paths** (`ensure-daily-note`, `patch-daily-callouts`). v0.45.0 ships the new `spice/cowork/daily/` path family + the manual creation flow, but the orchestrator skill bodies remain pointed at `spice/daily/`. Rewiring is the scheduling/aggregation cycle's scope. Orchestrators will need to choose between writing into cowork-owned files exclusively vs. patching the main daily note (or both, with cowork-owned as the primary write target and main daily as a mirrored output).

4. **FLN-4: AccentButton mobile layout edge cases.** ScratchDayActions uses `flex: true` for AccentButton which makes buttons fill row width; on narrow viewports with 3 buttons (CoworkHubNav) or 6 buttons (Timeframes), `flex-wrap: wrap` should reflow them but no manual mobile smoke was performed this cycle. Revisit in a small patch if it nags in use.

5. **FLN-5: v0.44.0 `assertCoworkV044Shape` asserts neutralized in S8.** Two v0.44.0 asserts became stale after the v0.45.0 CoworkHubNav rewrite (the function previously asserted "you are here" presence + the BeaconCards layout). Implementer neutralized them in place (kept the function, commented out the stale asserts). Future cycles: consider promoting these to a single "schema version" assert that captures the cowork manifest version + asserts shape invariants conditionally.
