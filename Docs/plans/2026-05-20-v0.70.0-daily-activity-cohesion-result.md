## Lessons

- **Framed renderer bypasses BeaconCards for inner rows.** First surface to do so since cards@0.2.x shipped. Pattern: when the row layout doesn't match the cards primitive, render directly and pass the same `getTitle` + `metaBuilder` contract. Worth remembering for future surfaces (weekly/monthly hubs may want it too).
- **Drill-in walk preserved by row-shape design.** `_renderActivityMeta`'s `parentElement.parentElement.parentElement` walk lands on `.sauce-group-row` because we made it a column-flex parent with title+meta inside an inner `.sauce-group-row-line`. The walk stays unchanged across both BeaconCards and framed surfaces. If a future surface uses a different row shape, the walk will need a `cardRoot` accessor opt.
- **Bucket merge → ordering → render is a clean three-pass pipeline.** Synthetic bucket keys flow through `groupOrder` / `colorByType` / `defaultClosed` exactly like a real type. No special-case branching in those passes.
- **Buried test-pin maintenance is a multi-stage hazard.** S2's activity-feed manifest bump cascaded: AF-1d (caught by S1's plan), AF-V065 (caught by S2 implementer), FA6-MANIFEST-activity-feed (caught by S3 implementer running the OTHER harness). Future cycles bumping a mechanism's manifest should grep for `"<name>"` + `"<old-version>"` across `platform/test/run-*.js` BEFORE running preflight. Same lesson on the daily side: FA6-MANIFEST-daily and FA4-MANIFEST-daily both pinned 0.9.0 + 0.5.0 respectively; S5 implementer fixed both.
- **FA4 version-floor comparison is broken for double-digit minor versions.** The assertion uses `m.version >= floors[bp]` with raw string lex compare; `"0.10.0" >= "0.5.0"` is `false` lexicographically. S5 worked around it by bumping the floor; the underlying lex-comparison-is-not-semver bug is pre-existing and out-of-scope for this cycle. Worth a follow-up cycle to swap in a proper semver-compare helper.
