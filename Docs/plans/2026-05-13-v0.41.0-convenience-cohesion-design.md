## Intent

Bundle three consumer-default behaviors under the `convenience` mechanism:

1. **Reader-view default** — every newly opened note starts in reading view (not editor).
2. **Cmd+T → today's daily** — new-tab hotkey opens today's daily instead of an empty tab.
3. **Tasks-plugin emojis → Lucide icons** — replaces 📅 ⏫ 🔼 🔽 ⏳ 🛫 ✅ 🔁 with monochrome SVG glyphs via a CSS snippet shipped + auto-enabled.

**Out of scope (deferred to a future cycle):** v0.39.0 cleanup. Initial design folded a revert of orphaned `project@1.6.0` manifest fields. Mid-cycle execution surfaced that v0.39.0 actually shipped 15+ commits including full `products@0.1.0` + `teams@0.1.0` blueprint scaffolds (templates, helpers, hub notes, slash commands, SKILL.md files) + 11 passing test-harness cases (PROD-1..5, TEAM-1..6) that exercise those scaffolds. A partial revert turned the harness red on PROJ-2 + PROJ-3. The properly-scoped close belongs in a future cycle that either finishes v0.39.0 (create missing `project-action-buttons.js` + wire subscription) or rolls it back wholesale. v0.41.0 leaves the partial state untouched — workshop subscription doesn't subscribe to products/teams, so scaffolds are inert; `project@1.6.0` catalogue pin is dormant because subscription pins `project@1.5.0`.

## Deliverable 1 — Reader-view default

- `platform/manifest.json` `app_settings` += `"defaultViewMode": "preview"`, `"livePreview": false`.
- Existing `applyAppSettings` helper writes both keys to `<vault>/.obsidian/app.json` (platform-as-overrider semantics already in place since v0.26.1).
- Zero installer changes.
- Effect: every freshly opened note starts in reading view; `Cmd+E` flips to edit.

## Deliverable 2 — Cmd+T → today's daily

- `convenience` manifest `hotkeys[]` += `{ "command_id": "daily-notes:goto-today", "modifiers": ["Mod"], "key": "t" }`.
- Built-in command. Inherits new-tab semantics from already-shipped `alwaysOpenInNewTab: true`.
- Zero new code; pure manifest delta routed through existing `applyHotkeys` helper.

## Deliverable 3 — Tasks-plugin emojis → Lucide icons

**Source:** `/Users/willfell/Documents/obsidian/sync/old-vaults/accuris/.obsidian/snippets/tasks-icons.css` (verified working in user's prior accuris vault).

**Mechanics:**
- Hides emoji text on `.task-list-item .task-extras span[data-task-priority]` + `.task-due` / `.task-start` / `.task-scheduled` / `.task-done` / `.task-recurring` via `font-size: 0`.
- Restores size on `::before` pseudo-elements + injects Lucide SVGs as `-webkit-mask-image` data-URIs (monochrome, inherits `currentColor`).
- Works on default Tasks-plugin emoji format — no `taskFormat` setting change needed.

**Ship path:** vendored at `platform/mechanisms/convenience/assets/snippets/sauce-tasks-icons.css`; installer copies → `<vault>/.obsidian/snippets/sauce-tasks-icons.css`; auto-enabled in `<vault>/.obsidian/appearance.json` `enabledCssSnippets[]`.

**New manifest schema (convenience@0.2.0):**

```json
{
  "snippets": [
    { "source": "assets/snippets/sauce-tasks-icons.css",
      "name":   "sauce-tasks-icons" }
  ],
  "appearance": {
    "enabledCssSnippets": ["sauce-tasks-icons"]
  }
}
```

**Installer wiring (simplified after scouting):**
- ONE new helper: `applySnippets` — copies declared CSS assets to `<vault>/.obsidian/snippets/<name>.css`. Mirrors `applyVendoredThemes` posture (sha256-compare overwrite-with-backup; `.sauce-backup` suffix; failure-loud history).
- Existing `applyAppearance` helper (since v0.19.0) ALREADY supports `enabledCssSnippets` additive union — registration piggybacks on the existing `appearance` block at convenience manifest. No new registration helper.

**Helper count:** 13 → 14 (one new helper: `applySnippets`).

## Risks + mitigations

| Risk | Mitigation |
| --- | --- |
| Tasks plugin not installed yet → CSS selectors don't match | Snippet harmless when plugin absent; appearance.json registration is plugin-independent |
| `defaultViewMode: "preview"` annoys editing-first users on fresh notes | Documented in convenience manifest description; `Cmd+E` flips per-note; can be overridden by removing from `platform/manifest.json` app_settings |
| `daily-notes:goto-today` opens in current tab if today's daily already focused | Acceptable — Cmd+T pressed when on today's daily is a no-op-by-design; alwaysOpenInNewTab covers the first-open case |
| `.obsidian/appearance.json` additive-merge collision with user's other enabled snippets | Helper merges sauce-snippets-list ∪ existing; never replaces; backup-on-edit per landmine #12 mechanics |
| v0.39.0 cleanup deletes work-in-progress for ProjectActionButtons | Work is partial + never installed (1.5.0 pinned in subscription); future cycle can re-introduce cleanly. Git history preserves the 1.6.0 manifest for reference |
