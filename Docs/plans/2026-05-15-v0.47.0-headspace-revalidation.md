## Surfaced during S11 — queued for v0.48.0

**Project task-creation flow does not prompt for workstream (and "+ Add Task"/"+ Add Task Board" helper buttons are absent from the project atlas).**

- **Pre-existing v0.46.0 regression**, not caused by v0.47.0. v0.47.0 install did not touch `ranch/scripts/project/project-nav-buttons.js` (source bytes unchanged) and did not touch `.obsidian/plugins/templater-obsidian/data.json` (last modified May 13 at headspace; untouched by S9 reinstall).
- **Root cause:** v0.46.0 stripped `_createTaskNote` + `_createTaskBoard` (~184 LOC) from `ProjectNavButtons`. The replacement plan relied on the Kanban plugin's "+ Add a card" UI + `Template, Kanban Card.md`'s `<%* %>` workstream picker. Kanban plugin uses Obsidian's CORE template paste, which does NOT execute Templater syntax. Project's manifest declares zero `templater_folder_templates[]`, so Templater's `trigger_on_file_creation` doesn't fire on `spice/projects/<slug>/tasks/` files.
- **Decision:** per user, **close v0.47.0 cycle as planned** (the 4 documented bugs are fixed; this is a separate cycle's problem). Queue as FLN-i in the result doc for a v0.48.0 brainstorm + design.
- **v0.48.0 candidate design options:** (a) extend `entity-create` with a `pick-from-frontmatter` prompt type for a declarative `+ New Task` entity-create entry; (b) restore `_createTaskNote`/`_createTaskBoard` in `ProjectNavButtons` (revert v0.46.0's broader strip); (c) register Templater folder template for project tasks (needs glob/regex verification).
