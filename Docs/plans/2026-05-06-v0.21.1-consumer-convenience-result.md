## Lessons (NEW v0.21.1)

> [!tip] What this cycle taught
> 1. **Parallel-subagent dispatch with literal-contract prompts works cleanly when insertion points are far apart.** Both subagents landed clean first-pass. The API-contract quoting (v0.6.0) plus literal "insert immediately AFTER X function; don't touch any other line" eliminated collision risk. Pattern reusable: when N independent helpers share posture and implementation files are 800+ lines apart, parallel dispatch saves wall-time vs sequential.
> 2. **Broadened prereq gate is a defensible per-helper choice.** Styling helpers (v0.19.0) honor `_externalPluginsSatisfied`'s `required:true` semantic; `applyCommunityPluginData` (v0.21.1) broadens to ANY declared `external_plugins[]` id. Divergence is intentional and documented; future helpers should pick the semantic that fits their failure mode.
> 3. **First-wins on hotkeys is non-negotiable for consumer trust.** Never modify a binding the user has already set. The `skipped_existing` history entry is the canonical signal. No override knob and there shouldn't be one.
> 4. **Cmd+= is the literal physical key for "Cmd++"** on US keyboards — the unmodified key is `=`; users say "Cmd++" because that's what's printed on the keycap with shift. The hotkeys.json shape stores the unmodified key. Documented in how.md to prevent future authoring confusion.
> 5. **Reserved-headroom-UNUSED precedent now 7 data points** (v0.13.0 + v0.14.0 + v0.18.0 + v0.18.1 + v0.18.2 + v0.20.0 + v0.21.1). Pure-additive helper-pair / pure docs / pure adoption / scoped-bug-fix cycles reliably land clean. v0.21.0 (5 CFs) was the orchestration outlier.
