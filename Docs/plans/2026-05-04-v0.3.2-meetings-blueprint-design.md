# Design — `meetings` blueprint (v0.3.2)

> [!abstract] Goal
> Ship the `meetings` blueprint at `beacon/meetings/YYYY/MM-MMMM/YYYY-MM-DD <title>.md` with title + attendees prompts at click time. Lights up SpaceDailyDashboard's "Today's Meetings" panel (currently silent because `beacon/meetings/` doesn't exist). Pure additive blueprint on top of v0.4.0's runTemplaterTemplate + applyTemplaterFolderTemplates foundations — no installer changes, no schema additions, no mechanism bumps.

> [!info] Why this design exists
> v0.4.0 closed with the foundations explicitly designed for date-routed runtime-creation blueprints. Meetings is the second consumer (after to-do) and the test case for filename-with-prompted-title via `tp.file.rename`. The dashboard's `getMeetings` query (`p.file.name.startsWith(today)` + `p.summary` frontmatter read) was written during v0.3.0 with this blueprint shape in mind. Shipping meetings finishes the daily-dashboard's two-panel design (tasks panel from to-do + meetings panel from this cycle).

## Decisions locked during brainstorming (2026-05-04)

> [!success]+ Locked decisions
>
> 1. **1-stage cycle, additive-only patch.** No installer changes. No schema additions. No mechanism bumps. No harness changes. Pure blueprint authoring on top of v0.4.0 foundations. Workshop bump 0.8.0 → 0.8.1 (PATCH). Tag `v0.8.1` at close.
>
> 2. **Workshop version bump approved: 0.8.0 → 0.8.1 (PATCH).** User-confirmed during brainstorming.
>
> 3. **Schema: accuris-style with title + attendees prompts.** Verbatim port of accuris's `Template, Meeting.md` body shape with three surgical changes (paths + vault_identity_tag) PLUS one new addition (title-prompt-and-rename block at the top of the `<%* %>` block). Frontmatter: `date`, `type: meeting`, `summary: ""` (dashboard reads), `attendees: [[Wikilinks]]`, `tags: [{{vault_identity_tag}}, meeting, person/<Names>, YYYY/MM/DD]`. Sections: Attendees / Agenda / Notes / Action Items.
>
> 4. **Title-prompt mechanic at click time.** Template body uses `tp.system.prompt` for title; `tp.file.rename` switches the file from the deterministic intermediate `YYYY-MM-DD-Meeting` to `YYYY-MM-DD <title>`. If user cancels prompt (empty input), file stays at intermediate name (recoverable via manual rename). Multiple meetings per day fully supported — each click creates a fresh meeting with its own title.
>
> 5. **Layout: date-routed mirroring to-do + daily.** `beacon/meetings/YYYY/MM-MMMM/YYYY-MM-DD <title>.md`. Dashboard query is filename-based (`startsWith(today)`), not path-based, so the nested layout works.
>
> 6. **Bracket-escape `Meeting` literal in filename.** Action shape declares `filename: "YYYY-MM-DD-[Meeting]"` so moment.format treats `Meeting` as literal. Mirrors to-do's `[ToDo]` precedent. v0.4.0 FIX-1's `substituteForMomentFormat` covers the `{{module_directory}}` portion of `folder` automatically.
>
> 7. **Entry UX: nav-button only.** Single `+ Meeting` button using `runTemplaterTemplate`, order 120 (boards=100, to-do=110, meetings=120). NO Templater hotkey, NO Slash Commander binding. The button renders everywhere SpaceNavButtons mounts (same scope as boards/to-do).
>
> 8. **Folder-template backstop.** `templater_folder_templates: [{folder: "{{module_directory}}", template: "{{templates_path}}/Meeting.md"}]` so manual file creation in `beacon/meetings/` auto-applies the template (prompts fire just like nav-button).
>
> 9. **MeetingsHub aggregation surface DEFERRED.** Accuris has a `Template, MeetingsHub.md` + `NewMeetingButton` + `MeetingsHubCards` CustomJS classes for cross-meeting browsing. SpaceDailyDashboard already covers today's meetings; the hub adds value only for historical querying. Defer to a future cycle if needed.
>
> 10. **`daily_note` wikilink frontmatter field DEFERRED.** Accuris meeting notes have a `daily_note: "[[YYYY-MM-DD-Day]]"` field linking back. Today's daily note is reachable via Calendar/dashboard; the link is convenience, not load-bearing. Defer.
>
> 11. **Per-meeting `company`/`client` fields DEFERRED.** Accuris convention but not platform-load-bearing. User can add manually if relevant.
>
> 12. **Filename-collision recovery DEFERRED.** Two meetings same day same title → second rename throws. Real-world frequency is low; surface as Templater error if it actually happens.
>
> 13. **No harness changes.** No new installer mechanics → no new helper-cases. Workshop self-install + barebones install are the gates. Helper-cases stays at 220 sub-asserts.
>
> 14. **Pacing: subagent dispatch on T1.1 (manifest + template authoring) + T1.7 (result writeup); master-driven mechanical T1.2-T1.5 + T1.8-T1.9. Stage-close `AskUserQuestion` checkpoint at T1.8 before commit + tag + push.**

## Architecture

### Click-time flow

```
User clicks "+ Meeting" on any note (any context where SpaceNavButtons mounts)
  ↓
renderer dispatches runTemplaterTemplate
  → folder = moment().format("[beacon/meetings]/YYYY/MM-MMMM")
            = "beacon/meetings/2026/05-May"
  → filename = moment().format("YYYY-MM-DD-[Meeting]")
              = "2026-05-04-Meeting"
  → target = "beacon/meetings/2026/05-May/2026-05-04-Meeting.md"
  ↓
Existence check: target doesn't exist (or maybe DOES if user already clicked once today
                 without renaming — that case opens the existing intermediate file)
  ↓
Ensure folder exists (T-FIX1 mkdir already ran at install for {{module_directory}}; the
                       YYYY/MM-MMMM subfolder may not exist; renderer createFolder handles it)
  ↓
templater.create_new_note_from_template(templateFile, folder, filenameNoExt, true)
  ↓
Templater processes template body inline:
  → tp.file.creation_date / tp.date.now resolve in frontmatter
  → tp.system.prompt: "Meeting title?" → user types "Standup"
  → tp.file.rename("2026-05-04 Standup")
  → file moves to beacon/meetings/2026/05-May/2026-05-04 Standup.md
  → tp.system.prompt: "Attendees?" → user types "Alice, Bob"
  → person/Alice + person/Bob tags appended; [[Alice]] [[Bob]] wikilinks rendered
  ↓
File opens in active editor; SpaceDailyDashboard.getMeetings() picks it up via
file.name.startsWith("2026-05-04") → "Today's Meetings" panel populates with
link + summary (initially empty until user fills `summary` field).
```

### Multi-meeting-per-day handling

Re-clicking "+ Meeting" the same day:
- Renderer's existence check looks for `2026-05-04-Meeting.md` (the intermediate, deterministic name).
- After the first click's `tp.file.rename`, that intermediate path NO LONGER EXISTS (file is now at `2026-05-04 <title>.md`).
- Existence check passes → second meeting created at the intermediate → second prompt-and-rename → second meeting lives alongside the first.
- Dashboard panel lists BOTH meetings sorted by filename (which sorts by title since the date prefix is identical).

If user cancels the title prompt (empty input):
- `tp.file.rename` skipped → file stays at `2026-05-04-Meeting.md`.
- Re-click checks existence of the intermediate → file EXISTS → renderer's existence-check opens it. So if the user cancels twice, they get an idempotent open of the unrenamed-meeting file. Acceptable failure mode.

### Frontmatter contract for the dashboard

```jsonc
// dashboard reads:
//   p.file.name.startsWith(today)   → matches "2026-05-04 <title>.md"
//   p.file.link                      → wikilink to render
//   p.summary                        → frontmatter field, falls back to "" if missing
//   p.file.name.replace(`${today} `, "")  → strips date prefix for the visible label
```

Schema MUST have a `summary` field (even if empty string `""`) for the dashboard to render correctly. The template populates `summary: ""`; user fills in post-meeting.

## Trade-offs vs alternatives explicitly considered

> [!example]- Why these alternatives were rejected
>
> **A) Minimal schema (no attendees prompt; bare frontmatter only).** Faster authoring; no Templater interactivity. Rejected: user explicitly chose accuris-style during brainstorming. The attendees prompt + person/* tag generation is muscle-memory from accuris and load-bearing for cross-meeting people-querying in the future.
>
> **B) Verbatim accuris port (no title prompt; only attendees).** Accuris's template doesn't have a title prompt because accuris uses Templater's "Insert template" command from a hub UI which prompts for filename via Obsidian's native filename prompt. Rejected: our entry UX is a nav-button click, which doesn't trigger Obsidian's filename prompt. We need explicit `tp.system.prompt` + `tp.file.rename` to emulate.
>
> **C) Skip date-routing; flat `beacon/meetings/<TODAY> <title>.md`.** Simpler. Rejected: with v0.4.0 foundations, date-routed costs nothing (one moment-format string vs another) and scales to thousands of meetings without flat-dir pain. User locked date-routed during v0.3.1 (to-do) brainstorming for the same reasons; meetings inherits the convention.
>
> **D) Single meeting per day (deterministic filename, no rename).** Simpler renderer flow; idempotent re-click opens the same file. Rejected: accuris reality has 3+ meetings per day commonly; design must support it.
>
> **E) Ship MeetingsHub aggregation surface in same cycle.** Adds a `beacon/meetings/Hub.md` page with `MeetingsHubCards` view. Rejected as scope creep; SpaceDailyDashboard already covers today's meetings. Defer to a future cycle if cross-meeting browsing demand surfaces.
>
> **F) Add `daily_note: "[[YYYY-MM-DD-Day]]"` frontmatter wikilink to today's daily note.** Accuris convention. Rejected as YAGNI; today's daily note is one Cmd+P or Calendar-click away. Easy to add in a future cycle if requested.
>
> **G) Add helper-cases harness coverage for meetings.** No new installer mechanics → no new helper-cases needed. Workshop self-install + barebones install gates cover the install-time correctness; manual smokes cover runtime behavior.
