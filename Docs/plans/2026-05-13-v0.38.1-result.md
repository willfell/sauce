## Lessons (v0.38.1)

- (a) **The v0.38.0 substrate scales as designed.** Applying it to a new blueprint cost ≈3 file edits + a smoke expectation row + a version triple. No design judgment, no new plan, no clarification questions. The handoff doc was the operational spec; an autonomous session closed the cycle without intervention. This is the proof that v0.38.0's effort to ship a substrate was worth it — the marginal cost of the next 7 blueprint seeds is now tiny.

- (b) **Templater-literal frontmatter is harmless and accepted.** Seeded `Alice.md` shows the seed-managed frontmatter block (`company: Acme` / `tags: [person]` / `title: Engineering Manager`) followed by the template's own Templater-literal frontmatter block (empty `company:` / `location:` / `title:` lines that would be substituted at Templater-render time). substituteLenient only replaces `{{var}}` shape, not `<% %>` syntax. v0.38.0 daily β-seed and meetings α-seed both have this posture; v0.38.1 carries it forward. Audit-clean.

- (c) **Wikilink resolution is basename-keyed, not path-keyed.** Obsidian's link resolver matches `[[Alice]]` against any vault file with basename `Alice.md` — no path-prefix matching required. The meetings seed's dangling-wikilink warnings retire automatically the moment people files exist at any vault location; they land at `spice/people/Alice.md` per the module-directory invariant (landmine #11).

- (d) **No-handoff-needed cycles still benefit from a written handoff.** This cycle's handoff doc embedded the exact JSON content of the seed file, quoted the smoke harness's expectation array verbatim, and predicted the post-T2 sub-assert count (`17 pass · 0 fail`). A single session closed the cycle without re-planning or clarification questions. For tightly-specified substrate-application cycles, a handoff-as-spec is faster than a full plan + design + plan-review cycle.

- (e) **Controller-direct execution is the right cadence for small substrate-application cycles.** The handoff doc explicitly said "No subagents needed. Test-pass output IS the spec gate." This was correct: T1 verified via a tmp-vault `sauce seed` run, T2 verified via `node platform/test/run-integration-smoke.js`, T3 verified via `npm run release:preflight`. Subagent dispatch (implementer + spec-reviewer + code-quality-reviewer per task) would have added ~10× overhead with no quality gain on tasks this mechanical.
