# v0.4.2 Design — runTemplaterTemplate split-field schema

> [!abstract] Goal
> Replace the single moment-format `folder` + `filename` fields on the `runTemplaterTemplate` nav-button action with an explicit five-field split (`folder_prefix` + `folder_date_pattern` + `filename_prefix` + `filename_date_pattern` + `filename_suffix`). Literal text never reaches `moment().format()` again — the bracket-escape gap that surfaced in v0.4.0 FIX-1 (substituted-value escape) and v0.3.2 FIX-2 (non-substituted-literal escape) becomes architecturally impossible. Mirrors Obsidian Daily Notes plugin's clean separation between literal `folder` and moment-format `format`. Two consumer manifests (to-do + meetings) migrated in-cycle. Hard-cut, no deprecation path.

---

## Why this cycle, why now

> [!info] The bracket-escape gap surfaced TWICE — settle it now
> - **v0.4.0 FIX-1** (substituted-value bracket-escape, `substituteForMomentFormat` helper): `{{module_directory}}` → `beacon/to-do` was mangled by `moment.format` because `b/e/a/c/o/n/d/o` overlap moment tokens. Helper now wraps every substituted value in `[...]` literal-escape brackets.
> - **v0.3.2 FIX-2** (non-substituted-literal bracket-escape, author-discipline): the literal `hubs` segment in `"{{module_directory}}/hubs"` was mangled (`h` → "11", `u` → literal, `b` → literal, `s` → "1") because `substituteForMomentFormat` only brackets substituted values, not author-supplied literals. Tactical fix: bracket-escape the literal manually as `[hubs]`.
>
> The current contract requires authors to remember which letter sequences are safe (`/`, `-`, digits) and which need bracket-escaping (any non-substituted letter run). The discipline is documented in `Docs/how.md`, but enforcement is in the author's head — every future blueprint inherits the trap.
>
> The platform's "settle these underlying critical decisions while we ship out these first modules, ensuring few points of failure, and preventing long term tech debt" framing makes v0.4.2 the load-bearing fix BEFORE shipping more `runTemplaterTemplate`-using blueprints (journal, retros, weekly-hubs, daily-Today, etc.). Three occurrences would justify the schema bump per the doctrine; two have already happened. Settle now.

## Schema — split-field action shape

> [!example] New `runTemplaterTemplate` action shape
> ```json
> {
>   "id": "<button-id>",
>   "label": "<text>",
>   "icon": "<icon>",
>   "order": 100,
>   "action": {
>     "type": "runTemplaterTemplate",
>     "template_source": "<File>.md",
>     "folder_prefix":         "<literal>",
>     "folder_date_pattern":   "<moment-format>",
>     "filename_prefix":       "<literal>",
>     "filename_date_pattern": "<moment-format>",
>     "filename_suffix":       "<literal>"
>   }
> }
> ```
>
> | Field | Required | Install-time | Click-time |
> |---|---|---|---|
> | `folder_prefix` | yes | `substituteLenient` (literal substitution; e.g., `{{module_directory}}` → `beacon/<name>`) | passed through verbatim to renderer |
> | `folder_date_pattern` | no (default `""`) | passed through verbatim | `moment().format(folder_date_pattern)` |
> | `filename_prefix` | no (default `""`) | `substituteLenient` | verbatim |
> | `filename_date_pattern` | no (default `""`) | passed through verbatim | `moment().format(filename_date_pattern)` |
> | `filename_suffix` | no (default `""`) | `substituteLenient` | verbatim |
>
> **Renderer compose (click-time, in `space-nav-buttons.js` `runTemplaterTemplate` branch):**
> ```js
> const folder = action.folder_prefix +
>   (action.folder_date_pattern
>     ? "/" + window.moment().format(action.folder_date_pattern)
>     : "");
> const filename =
>   (action.filename_prefix || "") +
>   (action.filename_date_pattern
>     ? window.moment().format(action.filename_date_pattern)
>     : "") +
>   (action.filename_suffix || "");
> const filenameNoExt = filename.trim() ? filename : "Untitled";
> const target = folder ? `${folder}/${filenameNoExt}.md` : `${filenameNoExt}.md`;
> ```
>
> Hard-coded `/` separator between `folder_prefix` and `moment.format(folder_date_pattern)` — simpler than requiring authors to include trailing `/` in `folder_prefix`; current consumers all want it.

## Why three-field filename + two-field folder

> [!info] Asymmetry rationale
> - **Folder:** `prefix` + `date_pattern`. Two-field is enough — no current consumer needs a literal segment AFTER a date pattern (e.g., `beacon/<name>/<YYYY>/leftovers/`). Adding `folder_suffix` later is trivial if a consumer surfaces the need.
> - **Filename:** `prefix` + `date_pattern` + `suffix`. Three-field because both current consumers need a suffix (`-ToDo`, `-Meetings`); `prefix` retained for symmetry + future flexibility (e.g., `MTG-` prepending). The asymmetry is honest — folder vs filename naming patterns differ in practice.

## Migration — both consumers in-tree

> [!example] to-do@0.1.0 → 0.1.1
> ```diff
> -"folder": "{{module_directory}}/YYYY/MM-MMMM",
> -"filename": "YYYY-MM-DD-[ToDo]"
> +"folder_prefix": "{{module_directory}}",
> +"folder_date_pattern": "YYYY/MM-MMMM",
> +"filename_date_pattern": "YYYY-MM-DD",
> +"filename_suffix": "-ToDo"
> ```
>
> Click resolves to: `beacon/to-do` + `/` + `2026/05-May` → folder `beacon/to-do/2026/05-May`; filename `2026-05-04-ToDo`. Same target as v0.4.0; bracket-escape no longer needed in `[ToDo]`.

> [!example] meetings@0.1.0 → 0.1.1
> ```diff
> -"folder": "{{module_directory}}/[hubs]",
> -"filename": "YYYY-MM-DD-[Meetings]"
> +"folder_prefix": "{{module_directory}}/hubs",
> +"folder_date_pattern": "",
> +"filename_date_pattern": "YYYY-MM-DD",
> +"filename_suffix": "-Meetings"
> ```
>
> Click resolves to: folder `beacon/meetings/hubs` (no date routing — `folder_date_pattern: ""` → renderer skips moment.format); filename `2026-05-04-Meetings`. Same target as v0.3.2 post-FIX-2; `[hubs]` bracket-escape no longer needed.

## Backward-compat — hard cut

> [!warning] Old fields silently ignored
> - nav-buttons mechanism 2.1.0 → 2.2.0 BREAKING (action shape changed).
> - `validateAndResolve`'s `runTemplaterTemplate` branch reads ONLY the new fields. Manifests still declaring legacy `folder` / `filename` will install but won't dispatch correctly (renderer reads new fields → empty strings → `Untitled.md` at vault root).
> - No deprecation warning — single-developer pre-1.0 platform; only TWO consumer manifests; both migrated in S2. The workshop dogfood + barebones smokes catch any missed migration.
> - Both consumer manifests bump 0.1.0 → 0.1.1 (subscription version-bump triggers re-process per landmine #10).
>
> **Why no deprecation path:** Option B (keep both, deprecate old) doubles install.js complexity for a single-developer codebase with two known consumers. The bracket-escape gap would persist in the deprecated path — defeating the cycle's purpose.

## Installer + renderer changes

> [!example] `install.js` — validateAndResolve runTemplaterTemplate branch
> **Before (v0.4.0/v0.3.2):**
> ```js
> if (btn.action.type === "runTemplaterTemplate" && btn.action.template_source) {
>   const templatesPath = variables.templates_path || "Docs/Meta/Templates";
>   return {
>     ...btn,
>     action: {
>       ...btn.action,
>       folder:   substituteForMomentFormat(btn.action.folder || "",   variables),
>       filename: substituteForMomentFormat(btn.action.filename || "", variables),
>       template_source: `${templatesPath}/${btn.action.template_source}`,
>     },
>   };
> }
> ```
>
> **After (v0.4.2):**
> ```js
> if (btn.action.type === "runTemplaterTemplate" && btn.action.template_source) {
>   const templatesPath = variables.templates_path || "Docs/Meta/Templates";
>   if (typeof btn.action.folder_prefix !== "string" || btn.action.folder_prefix.length === 0) {
>     // failure-loud history; surface Notice; skip this entry
>     // (mirrors existing missing-id/label/action validation)
>     return null;
>   }
>   return {
>     ...btn,
>     action: {
>       ...btn.action,
>       folder_prefix:         substituteLenient(btn.action.folder_prefix, variables),
>       folder_date_pattern:   btn.action.folder_date_pattern || "",
>       filename_prefix:       substituteLenient(btn.action.filename_prefix || "", variables),
>       filename_date_pattern: btn.action.filename_date_pattern || "",
>       filename_suffix:       substituteLenient(btn.action.filename_suffix || "", variables),
>       template_source: `${templatesPath}/${btn.action.template_source}`,
>     },
>   };
> }
> ```
>
> `substituteForMomentFormat` helper deleted — last caller removed; helper has no other use.

> [!example] `space-nav-buttons.js` — runTemplaterTemplate dispatch
> The current branch reads `action.folder` + `action.filename` and runs both through `window.moment().format(...)`. The new branch composes from the five fields per the schema callout above. Click-time semantics unchanged (date routing, idempotent re-click, Templater create-from-template). Lines ~208-257 affected.

## Harness coverage

> [!info] R-cases growth (220 → ~245 sub-asserts)
> - **R1 rewritten** — `validateAndResolve` `runTemplaterTemplate` happy path with split-field schema; verifies `folder_prefix` substituteLenient applied; `folder_date_pattern` preserved verbatim; `filename_prefix`/`filename_date_pattern`/`filename_suffix` substituteLenient + verbatim per shape; `template_source` rewritten under `templates_path`. ## Risks + mitigations

> [!warning] Risk register
> | Risk | Mitigation |
> |---|---|
> | Renderer compose introduces off-by-one in folder/filename concatenation (e.g., missing/extra `/`) | Manual smokes A/B/C explicitly verify the resolved paths; harness R-cases verify install-time field shape; if path differs from v0.4.0/v0.3.2 baselines, smoke fails loudly |
> | Legacy `folder`/`filename` accidentally left in a manifest after migration | Renderer would compose empty folder + `Untitled` filename → smoke fails loudly at first click |
> | `substituteForMomentFormat` helper deletion breaks an unknown call-site | Single grep across `platform/` confirms zero remaining callers BEFORE delete; if any future-use comment references it, restore from git |
> | Harness R1 rewrite loses coverage of the old bracket-wrap behavior | Old behavior intentionally retired; R-cases cover NEW behavior; bracket-wrap is no longer the contract |
> | Subscription version-bump on barebones doesn't trigger re-process | Triple-bump per landmine #10 already covered (mechanism + workshop manifest + subscription); harness smoke catches stale install |
