## NEW v0.17.0 lessons (carry forward to v0.18.0+)

> [!warning] DV-shim composition discipline (latent bug class — codify in landmines)
> 1. **DV-shim must proxy `pages` + `current` + `el` when sub-cards delegate to BeaconCards.** A `{container: sub}`-only shim breaks any sub-renderer that calls `dv.pages(...)`. CF-1 surfaced this as a v0.16.0 latent bug masked by the user only having empty Pending sections in v0.16.0 smokes. **Rule:** when composing a hub renderer that hands a sub-container to another customJS class, the shim MUST proxy every dv method that sub-class might call. Bind to the parent dv, not invent a fresh API surface.

> [!info] Composability patterns (likely reusable across future blueprints)
> 2. **`badges[].style: "outline"` on BeaconCards** — cards@0.2.3 PATCH ADDITIVE. Outline pill shape: dot + alpha-bg + 999px pill (Linear/GitHub aesthetic). Default "fill" preserves prior behavior. Future blueprints with Planning/InProgress/Done state can opt in for the polished aesthetic.
> 3. **Outline-accent action button pattern** — repeated across NewBudgetButton/NewPaycheckButton/NewInvoiceButton + 3 widget "+ Add X" buttons + InvoiceControls Save/Submit. Mirrors project `_renderActionButton`. Likely candidate for a shared `BeaconButton` mechanism in a future cycle (would dedupe ~50 LOC across helpers).

> [!info] Frontmatter mutation pattern (FinanceFrontmatter as precedent)
> 4. **`FinanceFrontmatter.update(file, mutator)` API** — generic enough to be a candidate for promotion to a shared mechanism (e.g., `frontmatter-mutator@0.1.0`) for any future blueprint with widget-driven mutation. Would dedupe across blueprints. Defer until a second blueprint surfaces the same need.

> [!info] Cycle-shape patterns (refinement of established lessons)
> 5. **First-of-its-kind UI cycle pulls 5 CFs** — 5-data-point precedent (v0.5.0/v0.11.0/v0.12.0/v0.6.0/v0.16.0) now updated to **6-data-point**: v0.5.0/v0.11.0/v0.12.0/v0.6.0/v0.16.0/**v0.17.0**. Plan reserved Y.Z.x headroom upfront. CF-3 + CF-5 each were tiny single-purpose cleanup commits — proper use of in-cycle bumps to land small fixes incrementally rather than batching them all at the end.

> [!info] Visual cohesion as a CF magnet
> 6. **Visual cohesion + UX consistency are CF-heavy work** — three of five CFs (CF-1/CF-2/CF-4) were primarily styling. Lock-the-pattern up front via design quoting (per v0.6.0 lesson) prevents drift between widgets but cannot eliminate "feels off" smoke-time feedback. Plan accordingly.
