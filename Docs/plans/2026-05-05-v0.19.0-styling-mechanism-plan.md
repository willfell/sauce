# v0.19.0 styling mechanism — Implementation Plan

> **For the implementing agent:** REQUIRED SUB-SKILL: Use `executing-plans` to implement this plan task-by-task. Master-driven for manifest writes + version bumps + harness extensions + landmine doc edits. Per-task subagent dispatch is reserved for the 3 helpers (T2.5/T2.6/T2.7) which are independent and parallelizable.

**Goal:** Ship `styling@0.1.0` — a new mechanism vendoring the Baseline community theme + canonical rose-pine-light/melange-dark Style Settings JSON. Bumps mechanism count 5 → 6 and workshop_version 0.18.2 → 0.19.0 MINOR.

**Architecture:** Three new installer helpers (`applyVendoredThemes`, `applyAppearance`, `applyStyleSettings`) wired into `installItem` after `applyCorePluginSettings`. Each helper mirrors the v0.1.3/v0.3.0 plugin-data automation posture verbatim — additive merge, backup-on-edit, malformed-JSON guard, failure-loud history. Style Settings community plugin declared as `external_plugins[]` prerequisite (NOT vendored — same posture as Templater + Slash Commander). Theme is vendored; consumer-side assets land at `.obsidian/themes/Baseline/`, `.obsidian/appearance.json`, and `.obsidian/plugins/obsidian-style-settings/data.json`. Landmine #12 allowlist 3 → 6 paths; one new landmine (vendored theme is mechanism-owned; never hand-edit) takes the next-available number at execution time.

**Tech Stack:** Node.js (installer + harness); Obsidian vault adapter; JSON throughout (no YAML).

**Cycle shape:** Single stage S1 + reserved 0.19.x headroom for 1-2 inline CFs (visual cohesion smoke-time feedback expected on the rose-pine palette). 5-data-point reserved-headroom-UNUSED precedent for pure-mechanism cycles (v0.13.0 + v0.14.0 + v0.18.0 + v0.18.1 + v0.18.2) — but v0.5.0/v0.11.0/v0.12.0/v0.6.0/v0.16.0/v0.17.0 hit 2-6 CFs on first-of-its-kind UI surfaces. Visual smoke-time feedback is the most likely CF trigger here; plan budget for 1-2.

if (opts.styling) {
