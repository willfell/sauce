Steps use checkbox (`- [ ]`) syntax for tracking. Sauce conventions: single-branch direct push to `main`; HEREDOC for multi-line commits; **no Co-Authored-By trailer**; **no `--no-verify`**.

**Goal:** Resolve three independent issues in one cycle: (A) daily blueprint reclaims canonical `ranch/templates/Daily Note.md` ownership (cowork's collision was silently dropping the `SpaceDailyDashboard` block from new daily notes in all vaults); (A2) `SpaceDailyDashboard` grows a third "Today's Activity" panel that delegates to `ActivityFeed` with new `asOf` + `includeMtime` knobs (catches "added or updated today" across blueprints); (B) `CoworkHubNav` block falls out of per-day notes for free as a consequence of A; (C) Obsidian's `propertiesInDocument` defaults to `"visible"` via the existing `applyAppSettings` workshop helper.

**Architecture:** Three coordinated changes, one tagged release.
1. `activity-feed@0.1.0 → 0.1.1` — two additive render-opts: `asOf` (anchor time window to a date, default now) + `includeMtime` (OR file.mtime into the time-window predicate). Backwards-compatible — every existing call site still works.
2. `cowork@0.9.0 → 0.9.1` — `content/Daily Note.md` source file deleted; `files[]` entry dropped. Cowork no longer materializes a daily template. CoworkHubNav remains in cowork's hub files only.
3. `daily@0.4.0 → 0.5.0` — `content/daily-template.md` REWRITTEN to absorb cowork's prior Templater preamble + frontmatter shape + restored `SpaceDailyDashboard` block + restored `<!-- COWORK_CALLOUTS -->` marker. `helpers/space-daily-dashboard.js` grows a third "Today's Activity" panel that delegates to `customJS.ActivityFeed.render(...)`. New `depends_on: activity-feed >=0.1.1`.
4. Workshop `app_settings` += `"propertiesInDocument": "visible"`. `workshop_version: 0.63.3 → 0.64.0` MINOR.

**Tech Stack:** Node.js CLI (sauce installer), Obsidian CustomJS classes, Dataview + DataviewJS, Templater, JSON manifests, jsdom-driven preflight test harnesses, GitHub Actions `release.yml` chain, `willfell/homebrew-sauce` tap auto-bump.

  ```js
  .where((p) => {
    if (!p) return false;
    if (blueprints.indexOf(String(p.type)) < 0) return false;

    if (!tsRaw) return false;

  ```
  with:
  ```js
  .where((p) => {
    if (!p) return false;
    if (blueprints.indexOf(String(p.type)) < 0) return false;
    // ts-window predicate (created_at: or status_changed_at:)
    const tsRaw = p[tsKey];
    let tsHit = false;
    if (tsRaw) {
      const ts = String(tsRaw);
      if (/^\d{4}-\d{2}-\d{2}$/.test(ts)) {
        tsHit = ts >= start.slice(0, 10) && ts <= end.slice(0, 10);
      } else {
        tsHit = ts >= start && ts <= end;
      }
    }
    if (tsHit) return true;
    // mtime predicate (file.mtime), opt-in via includeMtime
    if (!includeMtime) return false;
    if (!p.file || !p.file.mtime) return false;
    const mIso = (typeof p.file.mtime.toISO === "function")
      ? p.file.mtime.toISO()
      : String(p.file.mtime);
    return mIso >= start && mIso <= end;
  })
  ```

  ```js
    /**
     * Pre-count activity matches for the hasContent gate. Mirrors
     * ActivityFeed._query semantics but returns just the length, so we can
     * short-circuit the dashboard render when nothing matches.
     */
    async _getActivityCount(dv, today) {
      const startIso = window.moment(today, "YYYY-MM-DD").startOf("day").format();
      const endIso   = window.moment(today, "YYYY-MM-DD").endOf("day").format();
      const allowed  = this._DEFAULT_DASHBOARD_BLUEPRINTS;
      const inDay = (p) => {

        if (allowed.indexOf(String(p.type)) < 0) return false;
        const tsRaw = p.created_at;
        if (tsRaw) {
          const ts = String(tsRaw);
          if (/^\d{4}-\d{2}-\d{2}$/.test(ts)) {
            if (ts >= startIso.slice(0, 10) && ts <= endIso.slice(0, 10)) return true;

      let count = 0;

      }
      return count;
  ```
