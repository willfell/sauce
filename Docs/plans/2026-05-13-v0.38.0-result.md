## Surprises

1. **Test-vault on stale layout.** S12.2 planned to re-seed `/Users/willfellhoelter/notes/sauce/test-vault`, but that vault's `ranch/templater/platformInstall.js` stub references `<vault>/../beacon/platform/install.js` (pre-v0.36.0 inside-vault pantry layout). `sauce reinstall --vault <test-vault>` failed because the legacy stub can't reach the brew-installed pantry. Workshop self-install (`sauce reinstall --vault $PWD`) used as substitute dogfood. Test-vault migration is out-of-scope for this cycle.
2. **macOS APFS case-insensitivity + `.gitignore`.** The repo's `.gitignore` rule `/Scripts/` is intended to mask the workshop's runtime-generated `Scripts/` activation dir. On APFS the lowercase `scripts/check-version-sync.js` collides with the same inode — `git check-ignore` reported the file ignored despite the casing. `git add -f` was required. On Linux CI (case-sensitive), `scripts/` and `Scripts/` are distinct, so this hack works there. A future small-cycle should replace `/Scripts/` with explicit named-file rules to avoid the collision.
3. **OAuth-token push restrictions on `.github/workflows/`.** HTTPS origin's GitHub OAuth token didn't have the `workflow` scope, blocking pushes of S9.1 (`ci.yml`) and S10.3 (`release.yml`). SSH push via `git@github.com:willfell/sauce.git` succeeded. The `github-personal` SSH alias from `CLAUDE.md` isn't configured on this machine — used the standard `github.com` host instead.
4. **Bootstrap `--mechanisms=all` was an inline correction.** S5.1 wrote the smoke harness to bootstrap with `--non-interactive --no-register`, but the `installed.json mechanism-version coherence` post-condition required the subscription to actually contain mechanisms. Adding `--mechanisms=all` to the bootstrap call in S8.5 fixed the last red assertion. Documented as FIX-LATER for future smoke-plan authors.
5. **Templater literals in seeded note bodies are expected.** The daily seed renders `content/daily-template.md` which contains Templater `<% tp.file.creation_date(...) %>` syntax. `substituteLenient` only replaces `{{var}}` shape, so Templater literals survive into the seeded files. Acceptable for audit-bar-ii (doesn't validate body content) but visually noisy. A future cycle could ship per-blueprint `seed-template.md` files alongside the user-facing `templates/*.md`.
