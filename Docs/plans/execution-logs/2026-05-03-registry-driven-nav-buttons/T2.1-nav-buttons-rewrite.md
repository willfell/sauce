## Added in v2.0.0

- `ICONS` map keyed by name. Includes `board, daily, todo, meetings, summary, projects, planning, plus`. Unknown icon name falls back to `<span class="nav-fallback-icon">${(label[0]||"?").toUpperCase()}</span>`.
- Async registry read via `app.vault.adapter.read("Docs/Meta/nav-buttons-registry.json")` then `JSON.parse`. ENOENT тЖТ silent return (empty install valid). Other read error тЖТ renders single error chip + return. Parse error тЖТ renders single error chip with `parseErr.message` + return.
- Flatten + sort: iterate `Object.entries(registry.contributions || {})`, carry `_source` for tie-breaking, sort by `(order ?? 100) - (order ?? 100) || _source.localeCompare || id.localeCompare`. Empty entries тЖТ return (no buttons, no error).
- `_dispatchAction(btn)` class method:
  - `openLink` тЖТ `app.workspace.openLinkText(action.target, "")`.
  - `createFromTemplate`:
    - If `action.target` exists тЖТ openLinkText and return.
    - Else read `action.template_source`. If read fails тЖТ Notice "nav-buttons: cannot read template <path>" + abort (no empty file).
    - Compute folder via `target.split("/").slice(0,-1).join("/")`. If folder missing, `app.vault.createFolder(folder)` (race-tolerant: ignore "exists" error).
    - `app.vault.create(target, body)` wrapped in try/catch тАФ on "already exists" race, treat as success and openLinkText.
    - On other create errors тЖТ Notice and abort.
  - Unknown `action.type` тЖТ `new Notice('nav-buttons: unknown action.type "${type}" from ${btn._source}', 8000)`.

## Deviations from design sketch

1. **Dispatcher placement.** Design sketch had `dispatchAction` as a file-scope async function. I made it a private-ish class method `_dispatchAction(btn)` to keep the file under a single top-level form (CustomJS-ergonomic) and to match the v1.0.0 file-shape. Functionally equivalent.
2. **Icon set.** Design sketch only mentioned `board` as required + others "for forward-compat". I added the seven additional icons (`daily, todo, meetings, summary, projects, planning, plus`) verbatim from v1.0.0 SVG bodies тАФ no `chevronLeft/Right`, `todoNew`, or `meetingsNew` carried (those belonged to dropped behaviors).
3. **Folder-create race tolerance.** Design sketch only tolerated "file exists" race on `app.vault.create`. I added matching tolerance to `app.vault.createFolder` since the same race applies (concurrent clicks both trying to create `boards/` first). Failure-loud preserved: any non-"exists" error surfaces a Notice and aborts.
4. **Read-error error chip.** Design sketch only handled ENOENT silently and parse error with chip. I added a separate non-ENOENT read error path that renders a chip тАФ covers permission errors and other edge cases without silently swallowing them.
