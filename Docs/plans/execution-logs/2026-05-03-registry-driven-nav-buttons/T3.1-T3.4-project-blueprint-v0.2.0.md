> [!warning]- Post-quality-review correction — dropped `{{vault_identity_tag}}` placeholder
> Quality reviewer flagged that `install.js:290 substituteLenient` only
> consults `paths.variables` (the `platform-config.json:variables` map),
> which has no `vault_identity_tag` key. Variant data lives in each
> blueprint's `variants.json` and is consumed by CustomJS helper classes at
> note-creation time, NOT by the installer's substitution pass. Net effect:
> if shipped as originally written, S4 install in barebones would land the
> literal string `{{vault_identity_tag}}` in the materialized file's
> frontmatter — invalid Obsidian tag.
>
> User decision: drop the placeholder entirely. Per-vault
> `boards/To-Do-Board.md` is already vault-scoped by location, so an
> identity tag adds little. Smallest delta; zero installer change; no
> per-consumer config friction; eliminates the substitution dependency.
> Edit applied to `platform/blueprints/project/content/kanban-board.md`
> immediately after quality review, before S3 close.
>
> Codified as new landmine in `Docs/landmines.md` (#9 — installer
> substitution variables scope) so this isn't re-discovered.
