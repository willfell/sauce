> [!abstract] Goal
> Add `pruneInstalledLedger` to `platform/install.js` so the install ledger
> auto-drops entries whose names are no longer in the current subscription —
> symmetric with `pruneNavButtonsRegistry` (S1 T1.3). Eliminates the need to
> hand-edit `Docs/Meta/platform-installed.json` (CLAUDE.md violation).

---

## Why

The S2 implementer surfaced an asymmetry: `pruneNavButtonsRegistry` prunes
`Docs/Meta/nav-buttons-registry.json:contributions.<X>` for items not in the
current subscription, but `Docs/Meta/platform-installed.json:{mechanisms,blueprints}[]`
kept stale entries when a subscription dropped items. Result: install ledger
drifted from subscription truth, and the implementer hand-edited
`platform-installed.json` to remove a stale `project` blueprint entry — a
CLAUDE.md non-negotiable violation (the file is auto-managed).

This task is the user-approved remediation: add the symmetric prune so
hand-editing is never needed again.

---

## Changes

> [!example]- `platform/install.js`
>
> **New function** at `install.js:725-841` — `pruneInstalledLedger(tp, subscription, installedNow)`.
> Mirrors `pruneNavButtonsRegistry`'s posture:
> - C4-hardened malformed-JSON guard: Notice + history `error` + return on parse failure.
> - Shape guard: Notice + history `error` + return on `null`/array/non-object.
> - Read-failure guard: Notice + history `error` + return on adapter.read throw.
> - First-install short-circuit: returns silently if the ledger file doesn't yet exist.
> - Per-removal `prune` history events: `{ event: "prune", kind, name, reason: "no longer subscribed", attempted_at }`.
> - `installedNow.history[]` preserved verbatim — only NEW prune events are appended.
> - Idempotency-on-mutation: when nothing is removed, no history events fire and
>   the finally-block write is byte-identical to prior on-disk state.
>
> **Wire-in** at `install.js:166-178` — invoked AFTER `pruneNavButtonsRegistry`
> in the install-loop tail, wrapped in the same try/catch posture so a failure
> here Notices + records history but does not abort install.

> [!warning] Deviation from `pruneNavButtonsRegistry`'s shape
> `pruneInstalledLedger` mutates the in-memory `installedNow` object instead
> of writing the on-disk file directly. Rationale: the install-loop's
> `finally` block at `install.js:172-181` unconditionally writes `installedNow`
> back to `platform-installed.json`. A disk-only mutation here would be
> silently clobbered milliseconds later. Disk is still read for malformed-JSON
> and shape-guard parity — if the on-disk state is unreadable or malformed,
> the function Notices + records a history error and skips the prune entirely
> (does NOT mutate installedNow). All other posture is preserved verbatim.
> This deviation is documented inline in the function comment and at the
> wire-in site.
