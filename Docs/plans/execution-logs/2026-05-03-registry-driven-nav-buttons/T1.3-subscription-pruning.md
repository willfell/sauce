# T1.3 — Subscription-aware pruning

## Integration point

At the END of the install for-loop, inside the same outer `try { ... }` that owns the `subscription` binding (so the variable is in scope) but AFTER every item has been processed. Wrapped in its own try/catch so a malformed registry never crashes the broader install.

```javascript
}
// for (const name of order) { ... } above

// 7. Subscription-aware pruning of Docs/Meta/nav-buttons-registry.json.
// Removes contributions.<source> for any source that is no longer in the
// current subscription. Self-cleaning registry — no separate uninstall
// mechanic needed. Wrapped in its own try/catch so a malformed registry
// (or a missing one) never aborts the broader install.
try {
  await pruneNavButtonsRegistry(tp, subscription, installedNow.history);
} catch (e) {
  new Notice(`platformInstall: nav-buttons registry prune failed — ${e.message}`, 6000);
  installedNow.history.push({
    event: "error",
    step: "nav_buttons_prune",
    message: e.message,
    attempted_at: new Date().toISOString(),
  });
}

topLevelOk = true;
new Notice("platformInstall: complete.", 4000);
```

The outer `finally` block already persists `installedNow` regardless of what happens in the prune step (E1 hardening), so any history pushes here survive a top-level rethrow.

## Helper body

```javascript
async function pruneNavButtonsRegistry(tp, subscription, history) {
  const adapter = tp.app.vault.adapter;
  const registryPath = "Docs/Meta/nav-buttons-registry.json";
  if (!(await adapter.exists(registryPath))) return;

  let raw;
  try { raw = await adapter.read(registryPath); }
  catch (e) {
    new Notice(`pruneNavButtonsRegistry: cannot read ${registryPath} (${e.message}). Skipping prune.`, 8000);
    history?.push({ event: "error", step: "nav_buttons_prune",
      message: `read failed for ${registryPath}: ${e.message}`, attempted_at: new Date().toISOString() });
    return;
  }
  let registry;
  try { registry = JSON.parse(raw); }
  catch (e) {
    new Notice(`pruneNavButtonsRegistry: ${registryPath} is malformed JSON (${e.message}). Skipping prune.`, 8000);
    history?.push({ event: "error", step: "nav_buttons_prune",
      message: `${registryPath} is malformed JSON: ${e.message}`, attempted_at: new Date().toISOString() });
    return;
  }
  if (!registry || typeof registry !== "object" || !registry.contributions) return;

  const subscribedNames = new Set([
    ...((subscription && subscription.mechanisms) || []).map((m) => m.name),
    ...((subscription && subscription.blueprints) || []).map((b) => b.name),
  ]);

  let mutated = false;
  for (const source of Object.keys(registry.contributions)) {
    if (!subscribedNames.has(source)) {
      delete registry.contributions[source];
      mutated = true;
    }
  }
  if (mutated) await adapter.write(registryPath, JSON.stringify(registry, null, 2));
}
```

(Actual file uses explicit `if (history) { history.push(...) }` rather than `history?.push(...)` — same convention as `applyNavButtons` and `applyRuleFragment`.)

## Behavior

- **No registry file** → no-op return. (Workshop's first install lands here.)
- **Malformed registry** → Notice + history error + skip. Does NOT clobber. (C4 hardening.)
- **Well-formed registry, all sources still subscribed** → no write (mutated=false).
- **Well-formed registry, some sources dropped from subscription** → delete those keys, write back.

## Deviations from the design sketch

The sketch in the design doc:

```javascript
const registry = readJsonOrEmpty(`${paths.meta_path}/nav-buttons-registry.json`, null);
if (registry) {
  for (const source of Object.keys(registry.contributions || {})) {
    if (!subscribedNames.has(source)) delete registry.contributions[source];
  }
  writeJson(`${paths.meta_path}/nav-buttons-registry.json`, registry);
}
```

Differences in my implementation:

1. **Hardcoded registry path** — same reason as T1.2: there is no `meta_path` variable in `platform-config.json`; `Docs/Meta/...` is hardcoded everywhere else.
2. **C4 malformed-JSON hardening** — the sketch's `readJsonOrEmpty(..., null)` collapses missing-and-malformed into the same null result. My version distinguishes them (existing-but-malformed → Notice + history; missing → silent no-op).
3. **Skip the write when nothing changed** — sketch always writes. I track `mutated` and only write on actual change. Idempotency: re-running install on a steady-state subscription leaves the file mtime alone — useful for the T1.6 "zero new file writes" idempotency check.
4. **Outer try/catch wrapper around the prune call** — defensive belt + suspenders. The helper itself is already failure-loud-but-non-throwing; the outer try still catches anything pathological (e.g., subscription is somehow not an object).
