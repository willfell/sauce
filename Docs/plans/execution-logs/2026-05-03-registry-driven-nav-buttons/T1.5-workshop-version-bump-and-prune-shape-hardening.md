### `platform/install.js` — nit 4 fix from quality review

The quality reviewer flagged a borderline failure-loud violation in `pruneNavButtonsRegistry`: a single guard `if (!registry || typeof registry !== "object" || !registry.contributions) return;` collapsed two distinct cases into one silent return — (a) registry parsed but is a malformed shape (null, array, primitive), (b) registry is a well-formed object but has no `contributions` key. The (b) case is a legitimate no-op; the (a) case is a real failure that should surface a Notice + history entry.

Split into two guards:

```js
if (registry === null || typeof registry !== "object" || Array.isArray(registry)) {
  new Notice(`pruneNavButtonsRegistry: ${registryPath} parsed but has unexpected shape (expected object). Skipping prune.`, 8000);
  if (history) {
    history.push({
      event: "error",
      step: "nav_buttons_prune",
      message: `${registryPath} parsed but has unexpected shape (expected object)`,
      attempted_at: new Date().toISOString(),
    });
  }
  return;
}
if (!registry.contributions) return;
```

The malformed-shape branch now matches the failure-loud posture of `applyNavButtons` and `applyRuleFragment`. The no-contributions branch stays silent (correct no-op for fresh `{schema_version: 1}` or `{}`).
