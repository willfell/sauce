## §8 — Late finding (T2.6 prereq): Templater "Template Hotkeys" registration is required

> [!warning] Surfaced 2026-05-04 during T2.6 manual smoke prep
> Slash Commander's "Add command" picker showed only the 4 base Templater commands (`Open insert template modal`, `Replace templates in active file`, `Jump to next cursor location`, `Create new note from template`) — NONE of the assumed `Templater: Insert Validate` / `Templater: Insert Audit` / `Templater: Insert Create New Project` commands appeared. The design + plan + initial `use.md` Slash Commander section all assumed Templater auto-surfaces a per-template command for every `.md` file in the templates folder. False.

### §8.1 — Root cause

Templater requires templates to be **explicitly registered** under Settings → Templater → "Template Hotkeys" section (one row per template). Each registration appends an entry to `.obsidian/plugins/templater-obsidian/data.json:enabled_templates_hotkeys[]`. Only registered templates surface as `Templater: Insert <name>` commands in the Obsidian command palette — and therefore only registered templates are mappable in Slash Commander (or any other command-palette-aware plugin).

Verified empirically by reading the barebones Templater config:

```json
// ../tmp-test-barebones-vault/.obsidian/plugins/templater-obsidian/data.json:31
"enabled_templates_hotkeys": [""]
```

Empty list → no per-template commands → exactly the symptom observed.

### §8.2 — Fix posture

User-side, manual, one-time per consumer vault. The platform installer **cannot automate this** — `.obsidian/plugins/templater-obsidian/data.json` is in the "ask before acting" set (CLAUDE.md non-negotiables: never edit `.obsidian/`). Same posture as the Slash Commander mapping itself — both are user-responsibility steps, not installer steps.

### §8.3 — Documentation updates landed in this session

- `Docs/use.md` "Slash Commander setup" section — step 2 added: register templates in Templater Template Hotkeys (between "Install plugin" and "Map slash keywords"). Added a "Why step 2 is required (Templater quirk)" warning callout explaining the auto-generation gap.
