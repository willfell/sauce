## New functions added

- `resolveDependencies(subscription, manifest)` â€” builds a `nodes` Map of subscribed items present in the workshop manifest at the pinned version, tagged with `kind` ("mechanism" | "blueprint"). Items missing from manifest or pinned at a wrong version are returned in `skipped`.
- `checkDeps(nodes, perItemManifest, subscriptionLookup)` â€” for each node, walks `depends_on` from the item's own manifest, marks the node `unfit` and emits a skip reason when a dependency is unsubscribed or its subscription pin doesn't satisfy the declared `range`.
- `satisfiesRange(version, range)` â€” minimal range checker. Accepts exact-equality (`range === version`) or `>=X.Y.Z` semver lower-bound. Returns false for unrecognized range syntax.
- `topoSort(nodes)` â€” DFS-based topological sort returning `{ order, cycle }`. Skips `unfit` nodes. Detects cycles via a `temp` (gray) set and returns the offending node name in `cycle`.

## Main flow refactor

Replaced the mechanism for-loop with a six-step pipeline:

1. `resolveDependencies` â€” turn subscription + manifest into a `nodes` Map plus a `missingItems` skip list.
2. Read each resolved item's `manifest.json` to populate a `perItemManifest` Map and a `subscriptionLookup` Map keyed by name.
3. `checkDeps` â€” annotate nodes with their satisfied dep edges or mark them unfit.
4. `topoSort` â€” produce install order; abort with a Notice on cycles.
5. Emit Notices and append `{ event: "skip", ... }` history entries for all skipped items (missing + dep-unsatisfied).
6. Iterate `order`, route each install to `installedNow.mechanisms` or `installedNow.blueprints` based on `node.target.kind`, call `installItem(tp, workshopPath, target, itemMan, variables)`, and on success update the bucket entry + push an `{ event: "install", kind, ... }` history record.
