## applyRuleFragment summary

`applyRuleFragment` is a module-scope helper that reads a single `rule_fragments[]` entry contributed by an item's manifest and merges it into a per-target JSON rule file at `${variables.rules_path}/${frag.target}.json`. The fragment is recorded under `contributions[sourceName]` so multiple items can co-own a target rule (e.g., both `daily` and `project` blueprints can contribute fragments into `_global.json`) without overwriting each other. It creates the rules directory on first use, tolerates a missing/corrupt existing file by starting fresh, and skips with a Notice if `rules_path` is not configured in `platform-config.json`.
