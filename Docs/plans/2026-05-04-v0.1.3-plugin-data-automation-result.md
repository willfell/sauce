## The three-step consumer flow — locked

> [!success] What consumers do, end-to-end
> 1. **Install Slash Commander.** Settings → Community plugins → Browse → "Slash Commander" → Install + Enable. Irreducible per Obsidian's plugin-install API not being script-accessible.
> 2. **Run the platform installer.** It writes Templater Template Hotkeys + Slash Commander bindings into the two plugin data.jsons. Idempotent re-runs; backup-on-edit; never overwrites malformed files.
> 3. **Reload Obsidian** (Cmd+R / Ctrl+R or restart). Templater re-registers per-template `Insert <name>` commands; SC re-reads bindings. The three slash commands go live.
>
> Two irreducible manual keystrokes total: SC install + Cmd+R reload. Both are Obsidian API limitations, not platform debt.

## Surprises during execution

> [!warning] Surprise 1 (resolved): plan spec had wrong manifest path field
> The plan's test-runner spec used `path: "platform/mechanisms/test-fixture"` for the fake workshop's manifest entry. `install.js`'s manifest resolver joins `${workshopPath}/platform/` with `node.target.path`, so the spec value produced a doubled `platform/platform/...` path and the installer logged "missing manifest" without reaching the helpers. Caught during T1.5 implementer execution; fixed in `run-helper-cases.js` (changed to `path: "mechanisms/test-fixture"` matching the canonical workshop manifest format at `platform/manifest.json:11-14`). No installer change required; T1.1/T1.2 spec compliance reviews remained valid.

> [!warning] Surprise 2 (resolved): tmp-acc deliberately lags on project + nav-buttons + workshop_version
> The plan's T2.2 said "Same diff applied to tmp-acc subscription," but tmp-acc was pinned to lagging versions (workshop 0.3.0, project 0.1.0, nav-buttons 1.0.0). A literal apply would have set workshop_version to 0.4.1, contradicting tmp-acc's deliberate lag posture. Conservative call: bumped only tmp-acc's validator + audit (its current canonical mechanism versions) to 0.1.1; left project / nav-buttons / workshop_version at their existing lagging pins. Honors tmp-acc's deliberate lag while still propagating the bug-fix-grade v0.1.3 improvements.

> [!info] Confirmation: workshop dogfoods correctly even without SC installed
> The workshop has no Slash Commander installed (it's a dev environment). On self-install: `applyExternalPlugins` warns 3× (validator/audit/project all declare SC required); `applyTemplaterHotkeys` applies 2× (validator + audit; project is not subscribed in workshop self-sub); `applySlashCommanderBindings` warns 2× because target absent. Harness exits 0 — warnings don't trigger exit-1. The platform self-install correctly handles the "developer machine without SC" case via the malformed/missing-target safety mechanic.

## Pacing rhythm — what worked / what to improve

### What worked

- **Subagent-driven execution with two-stage review (spec then quality) per task.** T1.1, T1.2, T1.5 each got a fresh implementer subagent + spec reviewer + code-quality reviewer. Caught the manifest-path-field bug in T1.5 + zero correctness issues in T1.1/T1.2. The discipline scaled across the 5 code-bearing tasks (T1.1-T1.5).
- **Master-driven mechanical tasks bypassed subagent overhead.** T1.3 (cp + md5), T1.4 (one Edit + smoke run), T1.6 (CLAUDE.md + landmines.md edits), T2.1 (3 manifest edits), T2.2 (4 file version bumps), T2.3 (3 harness invocations), T2.7 (regression smoke), T2.8 (3 doc edits), T2.9 (this writeup) — all done by master directly. Subagent dispatch would have multiplied wall-clock for no quality gain on these.
- **Disk-locked schemas before brainstorming.** Q1 (SC binding shape) + Q2 (Templater command-id format) both resolved from disk reads (barebones SC data.json + `templater-obsidian/main.js:add_template_hotkey`) before Phase 1 opened. Saved one user round-trip + caught the design's incorrect `obsidian-templater:insert-Validate` speculation early.
- **Public-API-only posture.** User explicitly rejected internal-API in-flight registration in favor of accepting the Cmd+R reload as the irreducible third step. v0.1.3 didn't take a dependency on Templater's `command_handler.add_template_hotkey()` or any SC internal API. Future Templater / SC version churn doesn't break this cycle's helpers.
- **Failure-loud + C4 malformed-JSON guard reused as the template** for both new helpers. New helpers inherited `applyExternalPlugins`'s posture without bespoke design.

### What to improve

- **The plan's test-fixture `path` field error should have been caught at plan-write time.** A test fixture that uses installer-internal path resolution semantics needs to be validated against the canonical workshop manifest format. Plans that prescribe fake-fixture shapes should require a verification step where the writer round-trips the fixture through the harness on paper. Carry forward as a writing-plans discipline note.
- **Plan didn't anticipate tmp-acc's lag posture.** Surfaced in T2.2 — the literal "Same diff applied to tmp-acc" instruction made no sense for a vault deliberately at older versions. Future plans touching multi-vault subscription state should explicitly enumerate per-vault target versions, not lazily say "same diff." Carry forward as a writing-plans discipline note.
