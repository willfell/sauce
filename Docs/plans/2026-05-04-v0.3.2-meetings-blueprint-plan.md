# v0.3.2 — `meetings` Blueprint Implementation Plan

> **For the implementing agent:** REQUIRED SUB-SKILL: Use `de:executing-plans` to implement this plan task-by-task.

**Goal:** Ship the `meetings@0.1.0` blueprint at `beacon/meetings/YYYY/MM-MMMM/YYYY-MM-DD <title>.md` with title + attendees prompts at click time. Lights up SpaceDailyDashboard's "Today's Meetings" panel. Workshop bump 0.8.0 → 0.8.1; annotated tag `v0.8.1`.

**Architecture:** Single stage. Pure additive blueprint on top of v0.4.0 foundations (runTemplaterTemplate + applyTemplaterFolderTemplates + substituteForMomentFormat). No installer changes, no schema additions, no mechanism bumps, no harness extensions. Manifest declares a `runTemplaterTemplate` nav-button with deterministic intermediate filename `YYYY-MM-DD-[Meeting]`; the template body uses `tp.system.prompt` + `tp.file.rename` to switch to user-typed `YYYY-MM-DD <title>.md`. A second prompt collects comma-separated attendees and auto-generates `person/<Name>` tags + `[[Wikilink]]` references in frontmatter + Attendees section. Folder-template registration backstops manual creation.

**Tech Stack:** Node.js (Templater user script context); Obsidian community Templater plugin (`tp.system.prompt`, `tp.file.rename`, `tp.date.now`); CustomJS + Dataview (renderer + dashboard).

// Prompt for title; rename file from "YYYY-MM-DD-Meeting" to "YYYY-MM-DD <title>"
