# v0.3.2 — meetings hub redesign Implementation Plan (v2)

> **For the implementing agent:** REQUIRED SUB-SKILL: Use `de:executing-plans` to implement this plan task-by-task.

**Goal:** Ship the `meetings@0.1.0` blueprint with the hub-centric pattern (one Meetings Hub per day with inline NewMeetingButton + MeetingsHubCards) at `beacon/meetings/{hubs,notes}/`. Bump `daily@0.1.0 → 0.1.1` for the `meetingsPath` update. Workshop bump 0.8.0 → 0.8.1 (PATCH); annotated tag `v0.8.1` at close.

**Architecture:** Single stage. Layered redesign on top of v0.4.0 foundations (runTemplaterTemplate + applyTemplaterFolderTemplates + substituteForMomentFormat). The "Meetings" nav-button targets the hub template at `{{module_directory}}/hubs/YYYY-MM-DD-Meetings.md`; the hub renders three dataviewjs blocks (SpaceNavButtons + NewMeetingButton + MeetingsHubCards). The NewMeetingButton CustomJS class opens a modal, prompts for title, then calls `templater.create_new_note_from_template` directly to create a meeting at `{{module_directory}}/notes/YYYY/MM-MMMM/YYYY-MM-DD <title>.md`. The MeetingsHubCards CustomJS class queries `beacon/meetings/notes` for files matching today's date and renders rich cards. Daily blueprint's space-daily-dashboard.js gets a 1-line update to point its `meetingsPath` at `beacon/meetings/notes` so the dashboard panel doesn't include hubs.

**Tech Stack:** Node.js + Templater plugin (`tp.system.prompt`, `tp.file.creation_date`, `tp.date.now`, `templater.create_new_note_from_template` API); CustomJS + Dataview; moment.js (already in renderer runtime).

/**
 * New Meeting Button (CustomJS)
 * Creates a button to add new meeting notes with date prefix.
 *
 * Usage in DataviewJS:
 *   await dv.view("Docs/Meta/Views/customjs-guard", { class: "NewMeetingButton" });
 */
class NewMeetingButton {
  async render(dv) {
    const currentFile = dv.current();
    const currentDateStr = currentFile.file.name.substring(0, 10);

    // (no spacePrefix — beacon has no Life/ namespace)
    const datePart = window.moment(currentDateStr);
    const folder = `beacon/meetings/notes/${datePart.format("YYYY")}/${datePart.format("MM-MMMM")}`;

    const newMeetingIcon = `<svg ... (verbatim from accuris) ... />`;

    const inputModal = (promptText) => {
      // verbatim from accuris lines 23-85
    };

    const container = dv.el("div", "");
    const btn = container.createEl("button");
    btn.innerHTML = newMeetingIcon + `<span style="margin-left: 8px;">New Meeting</span>`;
    btn.style.cssText = "cursor: pointer; padding: 12px 24px; border-radius: 8px; font-size: 1em; display: inline-flex; align-items: center; justify-content: center;";
    btn.onclick = async () => {
      const title = await inputModal("Meeting title:");
      if (!title) return;

      const meetingFilename = currentDateStr + " " + title;
      const template = app.vault.getAbstractFileByPath("Docs/Meta/Templates/Meeting.md");
      if (template) {
        const templater = app.plugins.plugins["templater-obsidian"];
        await templater.templater.create_new_note_from_template(template, folder, meetingFilename, true);
      }
    };
  }
}
