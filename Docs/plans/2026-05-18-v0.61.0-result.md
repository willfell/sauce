## What shipped

NEW `backlink-panel@0.1.0` mechanism. Universal cross-blueprint backlink renderer wrapping a Dataview reverse-query over the 6 canonical cross-ref keys shipped FA-2..FA-7 (`people:`, `projects:`, `teams:`, `products:`, `trips:`, `meetings:`).

`BacklinkPanel.render(dv, opts)` API:
- `entityType`: one of 6 values (required)
- `groupBy`: `"type"` | `"month"` | `"none"` (default `"none"`)
- `limit`: number (default 25)
- `sortBy`: `"created_at"` | `"title"` (default `"created_at"` desc)

Renders via existing `customJS.BeaconCards.render` (no new CSS). Notice-on-error for invalid entityType + missing BeaconCards.

## Known limitations (intentional, captured for future cycles)

1. **Existing entity notes don't auto-receive the new Mentions panel.** Entity-create's `body_template` / `inline_body` controls the embed only at note creation time. Existing Person / Project / Team / Product / Trip pages need manual addition. Captured as a "template-resync" verb FLN for future brainstorm.
2. **Renderer harness cases (+5) deferred.** Card scope listed `run-renderer.js` deltas; this cycle ships lint-only coverage. Live render verification happens via workshop dogfood install + headspace manual smoke. Captured as v0.61.1 candidate if drift observed.
3. **PeopleRendering FLN (latent).** Existing PeopleRendering template embeds use `args: [dv, query, opts]` which causes `(dv_dispatcher, dv_template, query, opts)` double-bind via customjs-guard — `query.mode` resolves to `undefined`, silently producing "unknown mode undefined" Notice. NOT a v0.61.0 regression; pre-existing. BacklinkPanel uses the clean `args: [opts]` convention. Captured as a Bugs-workstream candidate.
4. **`groupBy: "month"` rich rendering.** Declared in API + implemented minimally (H4 month headers + BeaconCards groups). Richer "month rollup" UI deferred to FA-9 territory.
5. **No backlink panel on Project/Person consolidation.** Project keeps `ProjectReferencedByCards` (outlinks-based) alongside the new BacklinkPanel (canonical-key-based). Person keeps PeopleRendering's scoped meetings + daily panels alongside BacklinkPanel's universal panel. Consolidation deferred to a future cleanup cycle.

## FIX-LATER-NOTEs (v0.61.0)

- **FLN-v61-1:** workshop subscription does NOT subscribe to teams / products / trips / people blueprints, so the workshop dogfood install does not exercise those template updates end-to-end. The blueprint version bumps + manifest depends_on are validated via the test harnesses + JSON parsers, but the templater-folder-template materialization paths only get exercised on consumer vault installs. **Mitigation:** the 4-vault reinstall step at deploy time exercises all 5 templates against real vaults.
- **FLN-v61-2:** CLAUDE.md `Status (live)` section claims "Workshop version: 0.49.2" — that's been stale for many cycles. Not a v0.61.0 problem; a CLAUDE.md maintenance candidate (revise-claude-md skill suggestion). The cycle-history.md archive carries the authoritative narrative; CLAUDE.md drifts during fast-cycle bursts.
- **FLN-v61-3:** `BacklinkPanel._renderGroupedByMonth` lumps "(undated)" pages into a single trailing bucket but doesn't surface a Notice about it. If a vault has lots of pre-FA notes (no `created_at`), the panel may look weird. Cosmetic — `groupBy: "month"` is opt-in and currently used by zero templates.
- **FLN-v61-4:** The `meta` callback on `_renderFlat` reads `p.type` directly. If a backlinked page hasn't migrated to canonical `type:` (FA-1..FA-7 holdover), the meta will be empty. Acceptable degradation — empty meta is harmless; doesn't break the panel.
