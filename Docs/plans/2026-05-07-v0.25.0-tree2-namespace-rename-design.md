# v0.25.0 Design — Tree 2 module-namespace rename + `.beacon-backup` → `.sauce-backup`

> [!abstract] Goals
> Final of three multi-cycle Sauce rebrand renames. Tree 1 (workshop clone `Beacon/` → `pantry/`) shipped in v0.23.0. Tree 3 (runtime plumbing `Docs/Meta/` → `ranch/`) shipped in v0.24.0. **Tree 2 (consumer module namespace `beacon/<module>/` → `spice/<module>/`) ships here.** Bundles `.beacon-backup` → `.sauce-backup` suffix rename as opportunistic brand cleanup under fresh-start posture (no legacy consumer state to preserve).

> [!info] Posture
> - Fresh-start: no consumer vaults using the old structure. NO migration helpers needed.
> - Master-driven on rename mechanics + harness updates + version bumps + cycle close.
> - Two-stage subagent review (spec then quality) at S2 close per v0.20.0 + v0.22.1 + v0.23.0 + v0.24.0 reaffirmation.
> - Manual smoke is the close gate.

## 1. Architecture / scope

**Single source of truth for Tree 2 prefix:** `platform/install.js:250`:

```js
itemVars = { ...variables, module_directory: `beacon/${itemMan.module_directory}` };
```

Change `` `beacon/${...}` `` → `` `spice/${...}` ``. Every `{{module_directory}}` substitution downstream inherits the new namespace at install time. Blueprint manifests are UNCHANGED — they already declare bare `module_directory: "<name>"`.

**`.beacon-backup` → `.sauce-backup`:** ~17 backup-write call-sites + 4 read sites + jsdoc + landmine #12 wording. Mechanical mass-replace. v0.23.0 + v0.24.0 lessons that preserved `.beacon-backup` were conditional on legacy consumer state; user's fresh-start stance retires that constraint.

**Helper .js files (~10 files):** hardcoded `"beacon/<module>/..."` literals in `dv.pages(...)`, `pathParts[?] !== "beacon"` checks, etc.

**Templates (4 files):** `[[beacon/...]]` wikilinks in `Kanban Card.md`, `Project Board.md`, `Task Board Card.md`, `Trip Board.md`.

**Out of scope:**
- Plugin data.json migration helpers (no real consumer vaults to migrate).
- Host-path rename (`/Users/willfell/Documents/obsidian/sync/workshop/beacon` → `/workshop/sauce`).
- Mechanism file rename (`platform/mechanisms/cards/beacon-cards.js` stays).
- Dogfood resync.
