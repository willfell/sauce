# v0.46.0 entity-create mechanism + 7-site big-bang migration

> **For the implementing agent:** REQUIRED SUB-SKILL: `executing-plans`. Implement stage-by-stage; each stage is one commit + push. The strategy doc at `Docs/plans/2026-05-14-blueprint-modularization-design.md` is the authoritative spec — Lego principles #1-#6, governing rules (schema-coverage, aesthetic), and Cycle 1 stage spine all live there. This plan realizes Cycle 1; it does NOT re-derive the principles.

## Overview

**Goal:** Extract a NEW mechanism `entity-create@0.1.0` and big-bang migrate seven existing one-off `New<X>Button` create flows (meetings, people, project, scratch, finance × 3) to a single declarative manifest schema (`new_entity_buttons[]`). Per the strategy doc:

- One mechanism, one concern (principle #1): "Click a button → prompt user → create file at a date-routed or flat path with declarative frontmatter."
- Manifest is the API surface (principle #2): a blueprint declares `new_entity_buttons[]`; CustomJS escape hatches are findings, not features.
- Mechanism owns its materialization (principle #3): installer step, validator schema, audit rules, and `EntityCreate` runtime class all ship in this cycle.
- Schema-coverage governing rule: Appendix A below validates every existing site against the schema before S1 ships. **If any row of Appendix A is a `BLOCKED` or `needs-extension`, S1 must extend the schema before stage tickets are touched.**
- Aesthetic governing rule: AccentButton-row rendering. No new rendering primitive.

**Architecture:** One new mechanism directory `platform/mechanisms/entity-create/`; one new top-level blueprint manifest field `new_entity_buttons[]`; one new installer step `installNewEntityButtons`; three new audit rules; one NEW test harness `run-entity-create.js`; seven `New<X>Button` CustomJS classes deleted across 5 blueprints. The blueprint trio inside finance (Budget/Paycheck/Invoice) composes a single finance MINOR bump.

**Tech stack:** Obsidian + CustomJS + Dataview + Templater + AccentButton (existing mechanism). No new runtime plugins.

**Cadence:** 12 stages, sequential, each = 1 commit + push. Subagent-driven per stage.

**Schema extensions needed before S1 ships** (rolled into S1's authoring stage):

1. `extra_files[]` — array of sidecar file specs alongside the primary destination. Each entry: `{filename_pattern, frontmatter_template?, inline_body?, body_template?}`. Resolves project's atlas+map+board trio + invoice's atlas+time-log+board trio.
2. `{{current_file.frontmatter.<field>}}` substitution token — reads frontmatter of the note from which the button was clicked (the "current" note in Dataview/CustomJS context). Resolves scratch's `day`-rooted folder path.
3. `prompts[].type === "month"` — HTML5 `<input type="month">` (YYYY-MM). Resolves budget + invoice.
4. `inline_body` field (string, optional, peer to `body_template`) — short canonical body authored inline in manifest rather than shipped as a separate template file. Resolves budget + paycheck + invoice's three files. Keeps manifest the API surface for small bodies; reserve `body_template` for long templates (meetings, people, scratch).
5. `prompts[].type === "number"` + `validate` predicate (`gte:<key>`, `min:<n>`, `max:<n>`, `safe-filename`) — resolves paycheck's amount + cross-field constraint + people's name-character constraint.
6. Substitution pipe syntax `|number` — emits unquoted JSON numeric literal in frontmatter_template rather than YAML-quoted string. Resolves paycheck's `paycheck_amount`. (Alternative: type-aware substitution that infers from `prompts[i].type === "number"`. Pipe is more explicit; defer the inference to a future PATCH if it surfaces.)
7. `prompts[].derive` — derived (non-input) values computed from other prompts. For project's `slug`, the spec is: `{key:"slug", derive:"slugify(prompts.name)"}`. Resolves project. (Implementation: small `derive` DSL — initial set `slugify` + `lowercase` + `sanitize-filename`; extend later.)

**Verdict:** **0 of 7 rows fit the strategy-doc-as-written schema.** All 7 require at least one of extensions #1-#7. Of the 7 extensions, none change the *shape* of the strategy doc's schema (the table in the strategy doc remains correct: `id`, `label`, `prompts[]`, `destination`, `frontmatter_template`, `body_template`, `render_in` all stay) — they are *additive*. The strategy doc anticipated this in the "Coverage check" paragraph ("Project's mutable `status_changed_at` and the finance trio's status-enum defaults are the schema's stress tests"). The 7 extensions are catalogued here, authored in S1, and tested in S11.

**Schema clarification A (template-as-hub):** For meetings + scratch (where the "hub" is itself a Templater-materialized note, not a static file in `files[]`), `render_in.target_path` points to the *template body file* (e.g., `templates/Meeting Hub.md`). The installer injects the AccentButton dataviewjs block into that template body at install time, so every newly-materialized hub carries it. This is consistent with the strategy doc's wording ("a path inside the blueprint's `files[]`") because templates are listed in `files[]` — but it's worth surfacing explicitly so the implementer doesn't mistakenly look for a static path.

**Schema clarification B (catalogue drift):** People's blueprint manifest is at `0.1.2` while the platform catalogue records `0.1.1`. S12 reconciles by bumping both catalogue (→ `0.2.0`) and blueprint manifest (→ `0.2.0`) cleanly. Not a defect of this cycle; surfaces a pre-existing drift.

- [ ] **Step 3: Author the EntityCreate CustomJS class.**

Author `platform/mechanisms/entity-create/entity-create.js` with a single class `EntityCreate` exposing `static async render(dv, opts)` (dataviewjs-callable) and `static async create({instance, blueprintId, dv})` (programmatic). Skeleton (~40 lines, embed-dedupe + prompt loop + substitution + file create):

```js
/**
 * EntityCreate (CustomJS) — declarative entity-creation runtime.
 *
 * Reads ranch/entity-create-registry.json (materialized by the installer
 * from each blueprint's new_entity_buttons[]); each entry's `id` keys a
 * declarative spec: prompts → frontmatter_template → destination →
 * extra_files[].
 *
 * Usage in DataviewJS:
 *   await customJS.EntityCreate.render(dv, { instance: "meeting" });
 */
class EntityCreate {
    async render(dv, opts) {
        if (dv.container.closest(".markdown-embed")) return;
        const { instance } = opts || {};
        const spec = await this._loadSpec(instance);
        if (!spec) { dv.paragraph(`EntityCreate: no spec for "${instance}"`); return; }

        const plusIcon = `<svg xmlns="http://www.w3.org/2000/svg" width="16" height="16" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round"><path d="M12 5v14"/><path d="M5 12h14"/></svg>`;
        customJS.AccentButton.render(dv.container, {
            label: spec.label,
            icon: spec.icon || plusIcon,
            onClick: () => this.create({ instance, dv })
        });
    }

    async create({ instance, dv }) {
        const spec = await this._loadSpec(instance);
        if (!spec) return;
        const ctx = { now: window.moment(), current_file: dv ? dv.current() : null, prompts: {} };
        for (const p of spec.prompts || []) {
            if (p.derive) { ctx.prompts[p.key] = this._evalDerive(p.derive, ctx); continue; }
            const v = await this._prompt(p, ctx);
            if (v === null) return; // user cancelled
            ctx.prompts[p.key] = v;
        }
        const targetPath = this._substitute(this._joinDestination(spec.destination), ctx);
        const folder = this._substitute(this._destFolder(spec.destination), ctx);
        await this._ensureFolder(folder);
        if (app.vault.getAbstractFileByPath(targetPath)) {
            new Notice(`${targetPath} already exists; opening.`);
            app.workspace.openLinkText(targetPath, "");
            return;
        }
        const fm = this._renderFrontmatter(spec.frontmatter_template, ctx);
        const body = spec.body_template
            ? await this._readBody(spec.body_template, ctx)
            : (spec.inline_body ? this._substitute(spec.inline_body, ctx) : "");
        await app.vault.create(targetPath, `---\n${fm}---\n\n${body}`);
        for (const xf of (spec.extra_files || [])) await this._createExtra(xf, ctx, folder);
        app.workspace.openLinkText(targetPath, "");
    }

    async _loadSpec(instance) { /* read ranch/entity-create-registry.json; find by id */ }
    async _prompt(p, ctx) { /* dispatch on p.type → overlay dialog returning value-or-null */ }
    _substitute(str, ctx) { /* {{prompts.X}}, {{now.YYYY-MM-DD}}, {{now.HH-mm}}, {{current_file.frontmatter.X}}, |number, |sanitize-filename */ }
    _evalDerive(expr, ctx) { /* slugify(prompts.name) etc — small DSL */ }
    _renderFrontmatter(tmpl, ctx) { /* walk object literal, substitute strings, preserve types */ }
    _joinDestination(d) { return `${this._destFolder(d)}/${d.filename_prefix}${d.filename_date_pattern ? "{{now."+d.filename_date_pattern+"}}" : ""}${d.filename_suffix||""}.md`; }
    _destFolder(d) { return `${d.folder_prefix}${d.folder_date_pattern ? "/{{now."+d.folder_date_pattern+"}}" : ""}`; }
    async _ensureFolder(folder) { if (!app.vault.getAbstractFileByPath(folder)) await app.vault.createFolder(folder).catch(()=>{}); }
    async _readBody(relPath, ctx) { /* read body_template from {{templates_path}}/<file>, substitute */ }
    async _createExtra(xf, ctx, folder) { /* subfolder-aware; same substitution pipeline */ }
}
```

```js
// applyNewEntityButtons — for each item with new_entity_buttons[],
// either: (a) aggregate a synthetic entry into the nav_buttons registry
// when render_in.kind === "nav_buttons" — so SpaceNavButtons renders the
// button identical to a manifest-declared nav_buttons[] entry, dispatching
// to customJS.EntityCreate.create({instance: <id>}); or (b) inject an
// AccentButton dataviewjs block at install time into the file at
// render_in.target_path. Path substitution lenient (substituteLenient
// already imported at top of install.js).
async function applyNewEntityButtons(tp, manifest, history, git, variables, registry) {
  if (!manifest || !Array.isArray(manifest.new_entity_buttons)) return;
  const sourceName = manifest.name;
  for (const entry of manifest.new_entity_buttons) {
    const resolved = resolveEntityCreateEntry(entry, variables, sourceName, history, git);
    if (!resolved) continue;
    registry.entries.push({ blueprint: sourceName, ...resolved });

```js
// platform/audit/entity-create-walker.js — v0.46.0 S3
//
// Walks every blueprint manifest in ranch/platform-installed.json + every
// .js file under {{scripts_path}}/<blueprint>/ to surface entity-create
// modularization drift:
//
//   - manual_implementation_at_risk (HIGH) — blueprint has a CustomJS class
//     whose body matches /class\s+New\w+Button\b/ (per the New.*Button regex
//     in the strategy doc) but no new_entity_buttons[] entry. Either the
//     blueprint hasn't migrated, or the migration is incomplete.
//
//   - escape_hatch_used (INFO) — blueprint has BOTH a new_entity_buttons[]
//     entry AND a CustomJS class matching New.*Button — the escape hatch is
//     engaged. Expected when intentional; demands a justification line in
//     the cycle design doc per principle #2.
//
//   - dead_path (MEDIUM) — destination.folder_prefix doesn't resolve to an
//     existing folder under spice/ at audit time, OR body_template path
//     doesn't resolve to a Templater template file.

"use strict";
const fs = require("fs"), path = require("path");

async function walkEntityCreate(vaultPath, opts = {}) {
  const findings = [];
  const counts = { manual_implementation_at_risk: 0, escape_hatch_used: 0, dead_path: 0, aligned: 0 };
  const installed = readJsonSafe(path.join(vaultPath, "ranch/platform-installed.json")) || {};
  for (const bp of (installed.blueprints || [])) {
    const bpManifest = readJsonSafe(path.join(vaultPath, bp.path || "", "manifest.json"));
    const declaredIds = new Set((bpManifest?.new_entity_buttons || []).map(e => e.id));
    const scriptDir = path.join(vaultPath, "ranch/scripts", bp.name);
    const helperFiles = fs.existsSync(scriptDir) ? fs.readdirSync(scriptDir).filter(f => f.endsWith(".js")) : [];
    for (const hf of helperFiles) {
      const body = fs.readFileSync(path.join(scriptDir, hf), "utf8");
      const m = body.match(/class\s+(New\w+Button)\b/);
      if (!m) continue;
      const className = m[1];
      if (declaredIds.size === 0) {
        findings.push({ severity: "manual_implementation_at_risk", path: `${bp.name}/${hf}`, message: `class ${className} exists but blueprint has no new_entity_buttons[] entry` });
        counts.manual_implementation_at_risk++;
      } else {
        findings.push({ severity: "escape_hatch_used", path: `${bp.name}/${hf}`, message: `class ${className} coexists with new_entity_buttons[] (escape hatch intentional?)` });
        counts.escape_hatch_used++;
      }
    }
    for (const entry of (bpManifest?.new_entity_buttons || [])) {
      const folder = entry.destination?.folder_prefix?.split("/").slice(0,2).join("/"); // e.g. "spice/meetings"
      if (folder && !fs.existsSync(path.join(vaultPath, folder))) {
        findings.push({ severity: "dead_path", path: `${bp.name}#${entry.id}`, message: `destination.folder_prefix ${folder} does not resolve` });
        counts.dead_path++;
      }
      if (entry.body_template) {
        const tplPath = path.join(vaultPath, "ranch/templates", entry.body_template);
        if (!fs.existsSync(tplPath)) {
          findings.push({ severity: "dead_path", path: `${bp.name}#${entry.id}`, message: `body_template ${entry.body_template} does not resolve` });
          counts.dead_path++;
        }
      }
    }
    if ((bpManifest?.new_entity_buttons || []).length > 0 && findings.length === 0) counts.aligned++;
  }
  return { findings, counts };
}

function readJsonSafe(absPath) { try { return JSON.parse(fs.readFileSync(absPath, "utf8")); } catch (e) { return null; } }

module.exports = { walkEntityCreate };
```
