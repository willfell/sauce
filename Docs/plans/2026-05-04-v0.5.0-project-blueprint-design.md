# v0.5.0 Design — `project` blueprint port (replacement)

> [!abstract] Goal
> Replace `project@0.3.0` (the `projects-legacy` stopgap shipping a single mis-located `boards/To-Do-Board.md`) with `project@1.0.0` — a full port of the accuris projects pattern materialized under `beacon/projects/` per the module-directory invariant. Delivers per-project atlas + structure + Kanban board, hierarchical tasks (task hub + notes + task-board + folder-style sub-task cards), workstream-tagged cards, and an All-Projects hub with a New Project action.

> [!info] Reference
> Canonical example: `/Users/willfell/Documents/obsidian/sync/accuris/boards/planning/example-project-for-blueprit/` (full project tree with workstreams, tasks, sub-tasks, task notes). Source CustomJS class: `accuris/Docs/Meta/Scripts/project-nav-buttons.js` (~630 LOC).

## Architecture

### Materialization layout

```
beacon/projects/
├── Projects.md                                 ← All-Projects hub (NEW; New Project action)
├── templates/                                  ← installer-managed templates (no user edits)
│   ├── Template, Project.md
│   ├── Template, Project Structure.md
│   ├── Template, Project Board.md
│   ├── Template, Kanban Card.md                ← main-task card (project board → tasks/<X>.md)
│   ├── Template, Task Note.md
│   ├── Template, Task Board.md
│   └── Template, Task Board Card.md            ← sub-task card (task board → board/<Y>/<Y>.md)
└── <slug>/                                     ← created at runtime via ProjectNavButtons
    ├── <Project Name>.md                       ← atlas (#project; workstreams[] frontmatter)
    ├── <Project Name> - Structure.md           ← structure note (workstream-grouped task tiles)
    ├── <slug>-board.md                         ← project Kanban
    └── tasks/<TaskName>/
        ├── <TaskName>.md                       ← task hub (kanban-card; workstream id)
        ├── notes/<sub>.md                      ← task notes (task_parent backlink)
        └── board/
            ├── <TaskName>-board.md             ← task Kanban
            └── <CardName>/<CardName>.md        ← sub-task (folder-style card; workstream id)
```

### Mechanisms — no new installer mechanics required

> [!success] Pure additive blueprint authoring on existing platform
> - `applyTemplaterFolderTemplates` (v0.4.0) — wires Kanban `new-note-folder` paths to per-folder Templater templates. Used twice per project tree: project-board → main-task template; task-board → sub-task template.
> - `applyNavButtons` registry (v2.3.0) — adds one entry: `{name: "projects-hub", target: "{{module_directory}}/Projects.md", order: 60}` so every other note's platform nav row gets a Projects button.
> - `pre_install: [{type: "delete", path: "boards/To-Do-Board.md"}]` (v0.2.0) — retires the legacy mis-located file from any consumer that has it.
> - T-FIX1 mkdir (v0.3.0) — installer auto-creates `beacon/projects/`.
> - No new helper, no new schema, no new action_type.

## Data flow — three user-facing actions

### A. Create new project (Projects.md → New Project button)

```
User on beacon/projects/Projects.md → clicks "New Project"
  → ProjectNavButtons._promptForProjectName() (modal: name + slug preview + collision check)
  → slugify (kebab-case, alphanum + hyphen only)
  → mkdir beacon/projects/<slug>/
  → read templates/Template, Project.md → token-substitute (project name, slug, date) → write <slug>/<Project Name>.md
  → read templates/Template, Project Structure.md → write <slug>/<Project Name> - Structure.md
  → read templates/Template, Project Board.md → write <slug>/<slug>-board.md
  → app.workspace.openLinkText(<atlas>)
```

### B. Create new task (project board card via Kanban)

Kanban card creation goes through Templater folder-templates (existing v0.4.0 mechanic). Project board's `new-note-folder: beacon/projects/<slug>/tasks` + `new-note-template: <module_directory>/templates/Template, Kanban Card.md`. Templater runs the template body, which:

1. Accepts the Kanban-supplied filename as the task name.
2. Promotes flat `tasks/<X>.md` → folder-style `tasks/<X>/<X>.md` via `tp.file.move` (or `app.vault.rename`; pick during plan).
3. Sets `source_board: <project-board-path>` frontmatter.

### C. Create sub-task (task board card via Kanban) + task note

- **Sub-task:** identical to B, one level deeper. Task board's `new-note-folder: tasks/<X>/board` + `new-note-template: templates/Template, Task Board Card.md`. Template promotes flat → folder-style + sets `task_parent` + `source_board` pointing back to task hub + task board.
- **Task note:** ProjectNavButtons in `task-hub` or `task-note` context already renders a "New Note" action button. Click → modal prompt → write to `tasks/<X>/notes/<title>.md` from `Template, Task Note.md`. Already implemented in accuris source (`_createTaskNote`); ports verbatim.

### D. Workstream assign (Kanban card → workstream widget)

On any card note (`source_board` frontmatter present), ProjectNavButtons renders a workstream widget that reads the project hub's `workstreams[]` frontmatter, lets the user pick one (or unassign), and writes `workstream: <id>` to the card's frontmatter via `processFrontMatter`. Already in source.
