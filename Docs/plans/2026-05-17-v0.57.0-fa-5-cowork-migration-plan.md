# v0.57.0 — FA-5 Cowork migration (plan)

Cowork alone, the heaviest timeline wave per the umbrella design (§C Wave 3). `cowork@0.7.0 → 0.8.0` MINOR + `workshop_version 0.56.2 → 0.57.0` MINOR. Controller-direct cadence (no subagents; FA-4 / FA-3 pattern).

## Scope (cycle delta)

### Templates (3)
- `content/Daily Note.md` — `tags: [cowork-daily, daily]` → `tags: [daily]`; `created:` → `created_at:` with `Z` (ISO+TZ).
- `content/Weekly Note.md` — `tags: [cowork-weekly, weekly]` → `tags: [weekly]`; `created:` → `created_at:` ISO+TZ.
- `content/Monthly Note.md` — `tags: [cowork-monthly, monthly]` → `tags: [monthly]`; `created:` → `created_at:` ISO+TZ; **`month_iso:` → `month:`** (canonical YYYY-MM key); `month_label:` retained as friendly free-form display.

### Hub content files (5) — add canonical `created_at:` static
Hubs are static-materialized; consumer vaults rely on `cmd-migrate-frontmatter` backfill-from-mtime for existing copies.
- `content/Cowork.md`
- `content/Daily Hub.md`
- `content/Weekly Hub.md`
- `content/Monthly Hub.md`
- `content/About Cowork.md`

### Rule fragments (13) — add `extends: "_canonical-vocab"` to all; targeted edits to 3
All 13 fragments gain `extends: "_canonical-vocab"` (inherits canonical `type:` regex + `created_at:` ISO+TZ requirement). Plus targeted edits:
- **daily fragment** (`spice/cowork/daily/**/*.md`): drop `required_frontmatter.created`; drop `required_tags: [{ tag: "cowork-daily" }]`. Keep `day:` + naming_pattern.
- **weekly fragment** (`spice/cowork/weekly/**/*.md`): drop `required_frontmatter.created`. Keep `week_label`, `week_start`, `week_end`, `required_tags: [{ tag: "weekly" }]`, naming_pattern.
- **monthly fragment** (`spice/cowork/monthly/**/*.md`): drop `required_frontmatter.created`; **add `month: { required: true, type: string, matches: "^\\d{4}-\\d{2}$" }`** (canonical); keep `month_label` (free-form), `month_start`, `month_end`, `required_tags: [{ tag: "monthly" }]`, naming_pattern.
- Other 10 rule fragments (cowork-hub, active-threads, active-projects, .claude/skills, vault-config, Daily Hub, Weekly Hub, Monthly Hub, About Cowork, prompts): just gain `extends:`.

### Manifest
- `version: "0.7.0"` → `"0.8.0"` MINOR.
- `description:` updated with FA-5 summary line.
- `post_install` notice updated.

### Helpers — NO functional changes this cycle
Audit revealed the existing helpers already read the canonical-keys-or-better:
- `cowork-monthly-hub-cards.js` reads `p.month_label` → works (no change needed; we keep month_label, add canonical `month:` parallel).
- `cowork-weekly-hub-cards.js` reads `p.week_label`, `p.week_start`, `p.week_end` → unchanged.
- `cowork-daily-hub-cards.js` reads `p.file.name` + body → no frontmatter touch.
- `cowork-daily-actions.js` delegates frontmatter emission to the template (which we update) → no helper change.

Net: helpers unchanged. Plan-author note: design doc §C Wave 3 mentioned helper edits, but inspection at exec time shows they're either no-ops or already correct.
