# v0.1.2 Multi-Vault Automation — Implementation Plan

> **For the implementing agent:** REQUIRED SUB-SKILL — use `de:executing-plans` to drive this plan task-by-task. TDD discipline applies: write the failing harness assertion FIRST in T1.4, then implement `gitState()` in T1.1 / wire-up in T1.2. The S1 → S2 boundary is a **commit + push** boundary; do NOT begin S2 until S1's acceptance gate is fully green AND the S1 commit is on `origin/main`.

**Goal:** retire the bootstrap-copy × 3 resync ritual. Replace each consumer's `Docs/Meta/Templater/platformInstall.js` with a content-static ~12-line thin stub that dispatches at runtime to the workshop's canonical `platform/install.js`. Add `gitState()` helper that records `git_commit / git_tag / git_dirty` into `platform-installed.json` history at every install. Bump workshop_version 0.4.1 → 0.5.0; tag `v0.5.0` annotated.

**Architecture:** stub-only dispatch (no automatic `git pull` in any code path); `gitState()` mirrors v0.1.3 helper posture (failure-loud, never throws, returns nulls on any sub-call failure); install correctness decoupled from git correctness. Mirrors v0.1.3's 2-stage cadence: S1 helper + harness + LAST bootstrap-copy resync, S2 stub deployment + ritual retirement + version bump + tag.

**Tech stack:** Node.js (`child_process.execSync`, `path`, `fs`); Templater user-script ABI; Obsidian `app.vault.adapter`; install harness (`platform/test/run-install.js`); JSON-only metadata (landmine #6).

**Pacing:** 2-stage. S1 closes with one commit (`feat(installer): v0.1.2 S1 — gitState()...`). S2 closes with one commit + one annotated tag (`feat(installer,workshop): v0.1.2 S2 — stub deployment...` + `git tag -a v0.5.0`). Subagent dispatches MAY commit per-task during S1 (mirroring v0.1.3); master agent finalizes the stage commit. Implementer dispatches use `de:subagent-driven-development`.

**Constraints carried forward (non-negotiable):**

- JSON everywhere (#6).
- Substitution variables only from `platform-config.json:variables` (#9).
- Module-directory invariant under `beacon/` (#11) — N/A this cycle (no new blueprint content).
- Landmine #12 (`.obsidian/` allowlist) — N/A this cycle (no `.obsidian/` writes).
- **NEW landmines #13 + #14** (this cycle introduces them) — codified in S2 T2.6.
- **Bootstrap copies × 3 STILL IN FORCE through S1** — every `install.js` edit during S1 resyncs to the 3 bootstrap copies + md5-verifies. **S2 T2.1 retires the ritual** — at that point the bootstrap copies become stubs, content-static, never re-synced again.
- Failure-loud posture (Notice + history entry on every error path).
- Workshop dogfods every release. Workshop self-install (via stub in S2) must be green at S2 close.
- Desktop-only.
- "Ask before" gates: workshop_version bump (T2.6 covers this — pre-approved by Phase 2 design decision §0.4.1→0.5.0), force-push, history rewrite. `git push origin main` after each stage commit is NORMAL flow per CLAUDE.md.

### Task 1.1 — `install.js`: add `gitState(workshopPath)` helper

**Files:**
- Modify: `platform/install.js` (insert helper near top of file, before `module.exports = async function (tp) {...}`)

**Step 1.1.1 — write the helper**

Insert at the top of `platform/install.js` (after the file-header comment block, before `module.exports`). Mirrors v0.1.3's helper posture: try/catch around every external call; returns nulls on failure; never throws.

```js
// gitState(workshopPath) — best-effort capture of workshop git state for
// installed.history audit. Returns {commit, tag, dirty} where any field may
// be null if the workshop is not a git repo, git is unavailable, or HEAD has
// no exact-match tag. NEVER throws — install correctness must NOT depend on
// git correctness (landmine #14).
//
// Used by every installed.history.push() site post-workshopPath-resolution.
// Pre-resolution push sites (step: read_config, step: read_subscription) MUST
// record git_commit:null, git_tag:null, git_dirty:null explicitly.
function gitState(workshopPath) {
  const { execSync } = require("child_process");
  const result = { commit: null, tag: null, dirty: null };
  try {
    result.commit = execSync(`git -C "${workshopPath}" rev-parse HEAD`, { encoding: "utf8" }).trim();
  } catch { /* not a git repo, or git unavailable; leave null */ }
  try {
    const out = execSync(`git -C "${workshopPath}" describe --tags --exact-match HEAD 2>/dev/null`, { encoding: "utf8" }).trim();
    result.tag = out.length > 0 ? out : null;
  } catch { /* HEAD has no exact tag; leave null */ }
  try {
    const status = execSync(`git -C "${workshopPath}" status --porcelain`, { encoding: "utf8" });
    result.dirty = status.length > 0;
  } catch { /* leave null */ }
  return result;
}
```

```js
// v0.1.2 — verify gitState() captured git_commit/git_tag/git_dirty on every history entry
// post-workshopPath-resolution. Pre-resolution entries (read_config, read_subscription)
// record explicit nulls; verify the field is present even when null.
{
  const installed = JSON.parse(fs.readFileSync(path.join(vaultPath, "Docs/Meta/platform-installed.json"), "utf8"));
  const history = installed.history || [];
  const missingFields = [];
  for (const [idx, entry] of history.entries()) {
    if (!("git_commit" in entry)) missingFields.push(`history[${idx}].git_commit`);
    if (!("git_tag" in entry)) missingFields.push(`history[${idx}].git_tag`);
    if (!("git_dirty" in entry)) missingFields.push(`history[${idx}].git_dirty`);

  if (missingFields.length > 0) {

    process.exit(1);

  const hasRealCommit = history.some(e => e.git_commit !== null && /^[0-9a-f]{40}$/.test(e.git_commit));

    console.error(`FAIL: no history entry has a real git_commit (all null) — gitState() not capturing on a real git workshop?`);
    process.exit(1);
  }
  console.log(`OK: ${history.length} history entries; git fields present on all; at least one has real commit sha.`);
```

**Step 2.6.9 — add landmine #14 to `Docs/landmines.md`**

Append after #13:

```markdown
### 14. `gitState()` is best-effort; must never throw

The `gitState()` helper at the top of `platform/install.js` records workshop git revision into installed.json history. It wraps every `execSync` in try/catch and returns `{commit:null, tag:null, dirty:null}` on any failure. Install proceeds regardless of git state — even on a non-git workshop, even on a missing `git` binary.

**Why:** install correctness must NOT depend on git correctness. The lean v0.1.2 model is "stub dispatches; install runs; we record what we can about workshop state."
```
