**Goal:** Promote the outline-accent action button pattern (currently duplicated across 12 call sites with subtle drift) into a shared `beacon-button@0.1.0` mechanism. Visual surface stays IDENTICAL by design.

**Architecture:** New mechanism `platform/mechanisms/beacon-button/` mirrors `cards` mechanism layout. Single `class BeaconButton` with one method `render(parent, opts) → HTMLButtonElement`. Loaded into `customJS` post-install. Callers are inside other CustomJS classes (`project-nav-buttons`, finance helpers) — no `customjs-guard` wrapper layer needed. Migration replaces inline cssText + hover handlers with a single `customJS.BeaconButton.render(...)` call across 4 project sites + 8 finance sites.

**Tech Stack:** CustomJS classes (vanilla JS) ; standard mechanism install posture via `platform/install.js` ; Node renderer harness `platform/test/run-renderer.js` (DOM stub) ; `node platform/test/run-install.js` for install green-check.
