# v0.48.0 — project kanban workstream fix + icons polish (CLOSED-WITH-KNOWN-ISSUE 2026-05-15)

## Summary

v0.48.0 ships the `ProjectTaskCreateListener` CustomJS class + Templater startup_templates wiring + `applyTemplaterStartupTemplates` installer helper, intended to fix BUG-5 (kanban "+ Add a card" doesn't trigger Templater post-paste at projects). Workshop preflight passes across 15 harnesses with +10 sub-asserts vs v0.47.0 baseline. Brew formula auto-bumped via release.yml chain on tag push. Bonus FLN-a/b/c icons polish bundle: `icons@0.1.0 → 0.1.1` PATCH (resolve() _tier1 local capture; __cache → _cache rename) + `entity-create@0.2.0 → 0.2.1` PATCH (plusIcon 16x16 → 15x15 size consistency). 

**Known issue (S10 escalation):** at headspace re-validation, the listener never subscribed at boot. Cmd-R reload of Obsidian produced no `[ProjectTaskCreateListener] subscribed to vault.create` console log. Templater's `startup_templates[]` mechanism did not activate the listener at headspace's installation. Net effect: BUG-5 fix is **dormant at headspace** — kanban "+ Add a card" still doesn't trigger workstream picker. The v0.48.0 listener class + data.json wiring + test harness coverage are correct and reusable; only the registration vector failed.

**Disposition (per remote-control escalation gate):** close v0.48.0 as-is + v0.49.0 follow-up cycle swaps registration to L2 (customjs `startupScriptNames[]`). v0.49.0 design committed at `9095f33` immediately after v0.48.0 close. The v0.48.0 listener class is fully reusable in v0.49.0 — only the registration mechanism (templater startup_template → customjs startup script) changes.

## Plan vs reality

### S0 verification gate retrospect

S0 had 4 sub-steps. Steps 1.1 + 1.2 were retired headlessly:
- 1.1 ✅ Templater data.json shape + plugin enabled at headspace
- 1.2 ✅ `replace-in-file-templater` command id literal verified in plugin main.js source

Steps 1.3 + 1.4 required interactive Obsidian access. User chose **defensive-hedge path** (option C of 4 surfaced choices): bump `_postCreateDelayMs` 50 → 200ms (4× safety margin) + skip live smoke + defer empirical timing verification to S10 manual smoke.

**The defensive hedge turned out to be irrelevant.** The 200ms defer never came into play because the listener never registered at all — the actual failure was earlier in the chain (Templater `startup_templates[]` didn't fire). Lesson for future cycles: a defensive-hedge on Mechanism B doesn't help if Mechanism A (registration itself) fails.

### S10 escalation — discovery sequence

After brew upgrade + headspace reinstall + the data.json showing the startup_templates entry correctly registered + the `applyTemplaterStartupTemplates` helper firing during install (Plugin-data writes section showed data.json grew from 1122B → 1189B), user reported: **"i got no console logs"**. They reloaded Obsidian; the listener's subscribe log line never appeared.

Phase 1 of systematic-debugging revealed the actual failure surface:
- Templater `startup_templates[]` did NOT fire the listener template at boot at headspace
- The listener class itself is structurally correct (verified by extract + sub-assert in S5)
- The data.json registration is correct (verified post-install)
- Kanban plugin's per-board template config IS correct (verified `%% kanban:settings %%` blocks across all 9 project boards including the new test project)
- The user's `Sauce` project + its 7 task hubs prove that Template, Kanban Card.md works end-to-end WHEN Templater processes it
- The artifact `maybe.md` (created by the user during testing) proves Template, Kanban Card.md content was processed by Templater via SOME path — but not via the v0.48.0 listener

Root cause: the v0.48.0 listener registration mechanism (Templater `startup_templates[]`) does not reliably fire at headspace's installation. v0.49.0 swaps to L2 (customjs `startupScriptNames[]`) which is bound to a different plugin's startup lifecycle.

### S10 surfaced TWO additional bugs unrelated to v0.48.0

- **BUG-7 (HIGH, pre-existing):** `project-nav-buttons.js:597` gates the "Create Board" AccentButton on `ctx.context === "task-hub"` only. Inside a task note (e.g., `tasks/<X>/<X>.md` is task-hub but `tasks/<X>/notes/<Y>.md` is task-note), no Create Board button. NOT a v0.48.0 regression — gate has been task-hub-only since at least v0.39.0. v0.49.0 widens the gate.
- **BUG-8 (cosmetic, pervasive):** `<!-- entity-create:<id> -->` markers visible in source/edit mode of all 7 entity-create-using hubs. Hidden in reading mode. NOT a v0.48.0 regression — added in v0.46.0 entity-create migration. v0.49.0 relocates the sentinel to inside-block JS comment across all 7 hub files.

### v0.48.0 mechanism evaluation

Even though the listener didn't register at headspace, the **mechanism design itself is sound and remains in code**:
- The listener class with idempotency guard, path regex, content guard, Templater command invoke, failure-loud Notice — all reusable in v0.49.0 unchanged
- The `applyTemplaterStartupTemplates` helper stays in install.js (kept for potential future use; harmless)
- The test harness coverage (PTCL-1..5 + KC-1..3 + ICN-1 + smoke-prj-startup-tpl) all pass

v0.49.0 only swaps the REGISTRATION VECTOR. The class + data.json wiring + tests are belt-and-suspenders backstop for v0.49.0's customjs-startup-script primary path.

## NEW FIX-LATER-NOTEs

11 FIX-LATER items captured during S1-S10 (most are minor; FLN-v48-1 is the most actionable):

- **FLN-v48-1 (Important, deferred):** Listener double-subscribe risk on customjs manual reload. `_initialized` resets on fresh class instance; no `EventRef` stored, no `unregister()` method. Mitigated in practice by content guard (`<%` already-processed files skip). 4-line fix candidate: store EventRef in `this._eventRef` and skip re-registration if already set. (S1 code review)
- **FLN-v48-2 (Minor):** `openLinkText` UI flicker — opens task file as active before Templater invoke; unavoidable given Templater command API. (S1 code review)
- **FLN-v48-3 (Minor):** No try/catch around `app.vault.on(...)` registration itself — leaves class permanently unregistered if throws. Low probability; 4-line fix. (S1 code review)
- **FLN-v48-4 (Minor):** `e.message` may be undefined for non-Error throws (e.g., string thrown). `String(e)` is safer than `e.message`. (S1 code review)
- **FLN-v48-5 (Important, pre-existing):** Strategy 2 atlas read at `Template, Kanban Card.md:41-49` — `processFrontMatter` async callback never awaited; `resolve()` fires immediately, making Strategy 2 a silent no-op. Pre-existing v0.46.0+ bug; NOT introduced by S2. Strategy 1 (metadata cache) covers normal cases. (S2 code review)
- **FLN-v48-6 (Minor):** Empty-workstreams Notice wording references `'+ Create new'` which the user can't pick when ws is empty. Confusing UX. Suggest: "No workstreams defined. Add one to the project atlas, then pick a workstream on the next card." (S2 code review)
- **FLN-v48-7 (Minor):** Double `newName.trim()` call in Kanban Card.md picker — collapsible to single trim assigned to a const. (S2 code review)
- **FLN-v48-8 (Minor):** Whitespace-only post-substitution gap in `applyTemplaterStartupTemplates` — `if (!resolved)` doesn't catch `"   "`. Same gap exists in `applyTemplaterFolderTemplates`. Add `.trim()` check. (S4 code review)
- **FLN-v48-9 (Minor):** DRY opportunity — `applyTemplaterStartupTemplates` + `applyTemplaterFolderTemplates` share ~85% scaffolding (exists → read → parse → array-guard → loop → backup → write). Future cleanup cycle candidate if a third Templater-data.json field appears. (S4 code review)
- **FLN-v48-10 (Minor):** PTCL-2..5 lack header comment explaining why no `appOverride` is needed (only PTCL-1 calls `init()`). Cosmetic. (S5 code review)
- **FLN-v48-11 (Minor):** ICN-1's brace-matcher is naive (counts raw `{`/`}` including string-literal nested braces). OK for current icons.js but would miscount if future refactor adds template literals inside `resolve()`. Add comment noting the assumption. (S5 code review)

## v0.49.0 follow-up

v0.49.0 design committed at `9095f33` immediately after v0.48.0 close. Three independent fixes designed as one cohesive cycle:

1. **L2 listener registration** — customjs `startupScriptNames[]` replaces v0.48.0's Templater `startup_templates[]`. Listener class unchanged. v0.48.0's wiring stays as belt-and-suspenders backstop.
2. **BUG-7 fix** — widen `Create Board` button gate at `project-nav-buttons.js:597` to include `task-note` context.
3. **BUG-8 fix** — relocate entity-create sentinel from outside-block HTML comment to inside-block JS comment across all 7 entity-create-using hubs + install.js's `injectAccentButtonBlock` regex change.

S0 hard-fail gate elevated: empirically verify customjs `startupScriptNames[]` fires at boot at headspace BEFORE any v0.49.0 implementation code lands.
