# v0.70.0 — Daily Activity Cohesion (design)

**Cycle:** workshop v0.70.0
**Date:** 2026-05-20
**Scope:** `activity-feed` mechanism 0.3.x → **0.4.0**; `daily` blueprint 0.9.0 → **0.10.0**.

## Problem

The daily-note dashboard's Activity panel surfaces every blueprint type as its own muted UPPERCASE header above free-standing card rectangles. With the v0.69.0 cowork-scheduling work shipped, a typical day now produces 10–15 cards across 4–6 group headers, and the six `cowork-*` sub-types each appear as a separate sibling group (alphabetically clustered but not visually unified). Two failure modes:

- **Cowork fragmentation** — six near-identical headers (`cowork-eod-review`, `cowork-morning-briefing`, …) when conceptually they're one bucket. The pill on each card is already enough to differentiate sub-type.
- **Scratch noise** — a typical day produces 5–10 scratches, all expanded by default, pushing meaningful project/kanban activity below the fold.

The cards themselves render as floating dark rectangles with their own backgrounds; the group header above feels disconnected. The user wants each group to "feel like its own section" rather than a label hovering over orphan cards.

## Solution overview

Three coordinated changes:

1. **Bucket merge** — collapse `cowork-*` types into a single `cowork` group via a new `bucketRules` opt on `ActivityFeed.render`.
2. **Pinned ordering + per-group default state** — new `groupOrder`, `groupOrderBottom`, `defaultClosed` opts. The dashboard pins spine types (Cowork, Project, Kanban, Trip) at the top open; alphabetical middle; Scratch pinned last, collapsed.
3. **Framed group renderer** — new `framed: true` opt replaces `flatGrouped`. Each group renders as a soft container with a colored left stripe, a clickable chevron header, and a flat list of rows (hairline-divided) inside. The per-row card-rectangle treatment is dropped in this surface.

Together these changes turn the Activity panel from "list of labels above scattered cards" into "list of self-contained sections, important first, noisy last."

## Detailed changes

### `activity-feed` mechanism → 0.4.0

**New opts** (all additive except `framed` which subsumes `flatGrouped`):

- `bucketRules: Array<{bucketKey: string, match: (type: string) => boolean}>` — applied after grouping by `p.type`, before render. Any group whose key matches a rule is folded into the rule's `bucketKey`. Inside the bucket, pages are sorted by `tsKey` descending. The synthetic bucket key flows through `groupOrder` / `colorByType` / `defaultClosed` exactly like a real type.
- `groupOrder: string[]` — group keys (post-bucket-merge) to render first, in the given order. Keys with no cards are skipped silently.
- `groupOrderBottom: string[]` — group keys to render last, in the given order. Same skip-if-empty.
- `defaultClosed: string[]` — group keys whose `<details>` element renders without the `open` attribute.
- `framed: boolean` (default `false`) — when `true`, replaces the v0.3.0 `flatGrouped` renderer with the new framed-group DOM (see below). Mutually exclusive with `collapsible`; if both are passed, `framed` wins (deterministic, no error).

**Removed:** `flatGrouped` opt. Sole caller (`space-daily-dashboard.js`) is updated in the same cycle to pass `framed: true`. No deprecation period — workshop dogfoods the change.

**Untouched:** the `collapsible: true` path (v0.1.2). No known caller passes it today, but it remains as a low-cost legacy renderer for any future surface that wants nested `<details>` without the framed treatment. Removal can happen in a later cycle if it stays unused.

**Group resolution order inside `_renderGroupedByBlueprint`:**

1. Build `groups: Map<type, page[]>` keyed by `p.type` (existing logic).
2. Apply `bucketRules` — for each `(bucketKey, match)` rule, walk groups and re-key any matching group into the bucket. Pages from multiple matched groups concatenate; sort by `tsKey` desc inside the bucket.
3. Resolve render order: `[...groupOrder filtered to keys-with-cards, ...remaining keys sorted alphabetically, ...groupOrderBottom filtered to keys-with-cards]`. A key cannot appear in both `groupOrder` and `groupOrderBottom`; if it does, top wins (deterministic, no error).
4. For each key, emit the framed-group DOM (or fallback if `framed:false`). `<details open>` unless the key is in `defaultClosed`.

**Framed-group DOM shape:**

```
.sauce-group[data-group="<key>"]
├── <summary>.sauce-group-header     ← native <details>/<summary>
│   ├── .sauce-group-chevron         ← Lucide chevron, rotates 90° on [open]
│   ├── .sauce-group-dot             ← blueprint color from colorByType
│   ├── .sauce-group-label           ← human-cased key
│   └── .sauce-group-count           ← (N)
└── .sauce-group-body                ← <details> body
    ├── .sauce-group-row × N         ← title + meta (no per-row bg)
```

Title resolution still uses `getTitle`. Meta is rendered via `metaBuilder` (the existing `_renderActivityMeta`) directly into the row's meta container. Drill-in for `_isRollUp` pages still works: `_renderActivityMeta` already appends the drill-in panel to the card root via a `parentElement.parentElement` walk — when framed, the row IS the card root, so the walk lands correctly. Verified by inspection; covered by test #6 below.

### `daily` blueprint → 0.10.0

**`space-daily-dashboard.js`:**

- Replace `flatGrouped: true` with `framed: true` in the `ActivityFeed.render` call.
- Pass:
  ```js
  bucketRules: [{ bucketKey: "cowork", match: t => t.startsWith("cowork-") }],
  groupOrder: ["cowork", "project", "kanban", "trip"],
  groupOrderBottom: ["scratch"],
  defaultClosed: ["scratch"],
  ```
- Add `cowork: "var(--color-blue)"` to `_BLUEPRINT_COLORS`. Drives the group frame's left stripe + header dot. Tone shared with the Meetings outer section — pairs the two conceptually. Per-card pills inside the bucket continue to use `_BLUEPRINT_PILL_COLORS[p.type]` keyed on the original sub-type (`cowork-eod-review` etc.); since those keys aren't in the color map today, the pill dots stay neutral grey — which matches the accepted v0.69.0 screenshot. Coloring each cowork-* sub-pill is a follow-up if desired.
- Delete `_buildAccentSegments` helper (already retired per the v0.8.3/v0.67.3 comment — was kept as dead code; this cycle removes it).
- The `_renderActivityMeta` helper is unchanged; it produces the meta line the framed row consumes.

**`sauce-daily-dashboard.css`:**

- Add `.sauce-group`, `.sauce-group-header`, `.sauce-group-chevron`, `.sauce-group-dot`, `.sauce-group-label`, `.sauce-group-count`, `.sauce-group-body`, `.sauce-group-row` rules. All scoped under `.space-daily-dashboard`.
- Group frame: `background: var(--background-primary-alt); border-radius: 6px; border-left: 4px solid var(--group-accent, var(--color-base-50)); padding: 0; margin-bottom: 10px;` — accent color set inline via `--group-accent` from `colorByType`.
- Group body row: no `background-color`; `padding: 8px 14px`; `border-top: 1px solid var(--background-modifier-border)` on rows 2..N (`.sauce-group-row + .sauce-group-row`).
- Row layout: `display: flex; align-items: center; justify-content: space-between; gap: 12px;` — title left, meta right.
- Header chevron rotates 90° on `[open]` (mirrors existing `.sauce-section-chevron` pattern).
- Remove `data-segmented` rules + the `sauce-group-header` (old flat) + `sauce-group-dot` (old flat) + `sauce-group-count` (old flat) rules. Re-introduce the same class names under the new framed semantics (no collision because the old rules are deleted).
- Mobile (`@media (max-width: 480px)`): group padding `6px 10px`, chevron + dot shrink 1px, divider stays at 1px.

## Test coverage

Extend `platform/test/run-helper-cases.js` and `platform/test/run-renderer.js` (whichever fits the existing fixture style — confirm during plan-writing).

1. **`bucketRules` merge** — feed pages typed `cowork-eod-review` + `cowork-morning-briefing`; assert exactly one rendered group with `data-group="cowork"` and two `.sauce-group-row` children; assert rows are in `tsKey` desc order; assert each row's per-card type pill renders the original sub-type string.
2. **`groupOrder` top + bottom** — synthesize pages across `cowork`, `project`, `scratch`, `journal`; assert rendered `.sauce-group` DOM children are in the order `cowork → project → journal → scratch`. (Project pinned top, journal alphabetical middle, scratch pinned bottom.)
3. **`defaultClosed`** — assert `data-group="scratch"` `<details>` lacks the `open` attribute; assert `data-group="cowork"` `<details>` has the `open` attribute.
4. **`framed` DOM shape** — for each rendered group, assert structure `.sauce-group > details > summary.sauce-group-header + div.sauce-group-body`; assert no `.sauce-group-row` carries an inline `background-color`; assert second-and-later rows match `.sauce-group-row + .sauce-group-row` (relied on by the divider rule).
5. **Empty groups skipped** — pass `groupOrder: ["cowork", "trip"]` with cards only for `cowork`; assert no `data-group="trip"` element exists in the DOM.
6. **Drill-in still works under `framed`** — synthesize a `_isRollUp: true` page with 3 children; assert the breadcrumb chevron is present in the row's meta; programmatically click it; assert the drill-in panel transitions from `hidden` to visible and contains 3 `.sauce-drill-row` anchors.

Existing tests for `colorByType`, time-window resolution, and rollup-rule application remain unchanged and must continue to pass.

## Non-goals

- No change to the outer Tasks / Meetings / Activity section frames — only the inner Activity sub-group rendering.
- No change to `BeaconCards` itself. The framed renderer bypasses it; other surfaces using `BeaconCards` (Meetings panel, project hubs, etc.) are untouched.
- No change to rollup rules, time-window logic, or `_resolveTitle`.
- No new opt for "default-open everything except N" / wildcard `defaultClosed` — explicit list is fine at current scale (one entry).

## Risks

- **Breaking rename `flatGrouped` → `framed`.** Mitigated by single caller — `grep` confirms `space-daily-dashboard.js` is the only consumer in the workshop. Updated in the same cycle.
- **`_renderActivityMeta` walks `parentElement.parentElement.parentElement` to find the card root for drill-in append.** The framed row's DOM depth differs from the BeaconCards row. Test #6 verifies the walk lands correctly; if not, the plan will adjust the walk (or accept a `cardRoot` accessor from the caller).
- **`details/summary` accessibility** — keyboard activation (Space/Enter on summary) is native; no custom handler needed. Tab order is preserved.

## Out-of-scope follow-ups (not this cycle)

- Allowing users to override `defaultClosed` per-blueprint in vault config.
- Showing a one-line summary of collapsed groups (e.g., last edited title).
- Generalizing the framed renderer for the Tasks + Meetings outer sections.
