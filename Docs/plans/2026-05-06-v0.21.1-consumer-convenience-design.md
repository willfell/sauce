## Manifest schema additions

> [!example]+ NEW field `hotkeys[]` (mechanism + blueprint manifests)
> ```json
> "hotkeys": [
>   { "command_id": "workspace:copy-full-path", "modifiers": ["Mod"], "key": "-" },
>   { "command_id": "workspace:copy-path",      "modifiers": ["Mod"], "key": "=" }
> ]
> ```
> - Flat shape (one binding per entry); for two bindings on one command, declare two entries.
> - `command_id` non-empty string. `modifiers` array subset of `["Mod","Ctrl","Meta","Alt","Shift"]`. `key` non-empty.
> - `Mod` is Obsidian's portable modifier (Cmd on macOS).
> - Mirrors `templater_hotkeys[]` flat shape.

> [!example]+ NEW field `community_plugin_settings[]`
> ```json
> "community_plugin_settings": [
>   { "id": "dataview", "settings": { "enableDataviewJs": true, "enableInlineDataviewJs": true } }
> ]
> ```
> - Mirror of `core_plugin_settings[]` exactly except target = `.obsidian/plugins/${id}/data.json`.
> - Per-key shallow merge; `substituteLenient` round-trip on JSON.stringify(settings) (parity with applyCorePluginSettings:2114).

> [!example]+ Daily blueprint enrichment
> Add `"autorun": true` to existing `core_plugin_settings.daily-notes.settings`. Add new `hotkeys: [{command_id: "daily-notes", modifiers: ["Mod"], key: "["}]`. No new schema; just the new shared `hotkeys[]` field.

> [!example]+ convenience@0.1.0 manifest
> ```json
> {
>   "name": "convenience",
>   "version": "0.1.0",
>   "kind": "mechanism",

## Helper signatures (LITERAL CONTRACTS)

> [!example]+ `applyHotkeys(tp, manifest, history, git)`
> - Iterates `manifest.hotkeys[]` (no-op if absent/empty).
> - Target: `.obsidian/hotkeys.json`.
> - Validates each entry: `command_id` non-empty string; `modifiers` array of allowed strings; `key` non-empty.
> - Invalid entry → Notice + history `warning/hotkeys/invalid_entry` + skip.
> - Read existing: malformed-JSON guard (parse error OR not-an-object → Notice + history `error/hotkeys/malformed_json` + RETURN).
> - Backup-on-edit: `.obsidian/hotkeys.json.beacon-backup` BEFORE first write in this run (one-deep, single backup per target).
> - Per command_id: **FIRST-WINS** — if existing[command_id] is non-empty array, skip; never modify pre-existing user binding. If absent or empty, set to `[{ modifiers, key }]`.
> - Idempotent skip-write: structural equality.
> - Failure-loud history under `step: "hotkeys"`.
> - **No `variables` parameter** — command_ids are runtime-only literals (parity with v0.4.2 invoke_command passthrough).

> [!example]+ `applyCommunityPluginData(tp, manifest, variables, history, git)`
> - Iterates `manifest.community_plugin_settings[]` (no-op if absent/empty).
> - **PREREQ GATE** (v0.19.0 lesson): `_externalPluginsSatisfied(tp, manifest)` → if missing, history `info/community_plugin_data/skipped_missing_prereq`; zero writes; RETURN.
> - For each `{ id, settings }`:
>   - Target: `.obsidian/plugins/${id}/data.json`.
>   - Validate `id`: non-empty; **no slashes/dots** (path-traversal guard per v0.21.0 lesson d).
>   - `substituteLenient` round-trip on JSON.stringify(settings).
>   - Plugin dir `.obsidian/plugins/${id}/` absent → skip + history `info/skipped_plugin_dir_absent`.
>   - Read existing: malformed-JSON guard.
>   - Backup-on-edit: `<target>.beacon-backup`.
>   - Shallow merge: substituted (manifest) wins on key collisions (parity with applyCorePluginSettings:2197).
>   - Idempotent skip-write: structural equality.
>   - Failure-loud history under `step: "community_plugin_data"`.

> [!info] Insertion point in `installItem`
> After `applyCorePluginSettings`. Order: applyCommunityPluginData (8th), applyHotkeys (9th). **Helper count 7 → 9.**
