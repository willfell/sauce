## API extension shape (cards 0.1.1 → 0.2.0 MINOR)

### subtitle polymorphism

```js
subtitle: (page) => string | null | { text: string, secondaryText?: string }
```

> [!example]- Renderer behavior
> - `null` / `""` → no subtitle element (existing).
> - `string` → existing single-line behavior unchanged.
> - `{ text, secondaryText }` → primary line `text` rendered as today; if `secondaryText` truthy, second muted-italic line below: `font-size: 0.78em; font-style: italic; color: var(--text-faint); margin-top: 1px;`. Same indent as primary; mobile word-break preserved.
> - DRY into a `_renderSubtitle(parent, page, ctx)` helper called from both `_renderStackedCard` and `_renderRowCard`.

### badges[].icon

```js
badges: (page) => Array<{ label, tone?, icon? }>
```

> [!example]- Renderer behavior
> - If `b.icon` truthy, prepend an inner span with `innerHTML = b.icon`, sized to chip font; `gap: 4px` between icon and label.
> - Chip remains `inline-block` with same padding/radius/tone palette.
> - All existing badge callsites omit `icon` → no behavior change.

### Synthetic-page pattern (jsdoc only)

Codify in BeaconCards header docstring: `pages` may be hand-rolled objects rather than Dataview pages, as long as each exposes a `file` shape (minimum `{ name, path }`) for default `target` resolution. Custom `onClick` permits line-anchored or non-file destinations. Reference `SpaceDailyDashboard` tasks panel as canonical example after S3 lands.
