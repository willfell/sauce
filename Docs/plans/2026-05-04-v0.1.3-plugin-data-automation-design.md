# v0.1.3 ŌĆö Plugin-data automation (design)

> [!abstract] Cycle goal
> Eliminate manual Templater Template Hotkeys registration + manual Slash Commander mapping from the consumer onboarding flow. End state per consumer vault: **(1) install Slash Commander, (2) run installer, (3) Cmd+R**. Two new manifest fields, two new installer helpers, one scoped `.obsidian/` ban-lift, one new landmine. T2.6's four deferred smokes (from v0.1.x) become v0.1.3's S2 acceptance gate, executed against the automated flow.

> [!success] Phase 1 decisions (locked)
> - **Slash UX:** plain verb names ŌĆö `/validate`, `/audit`, `/new-project`. Bindings carry `name: "validate" | "audit" | "new-project"`; SC fuzzy-matches.
> - **Schema shape:** ergonomic. Manifests declare basenames + verb names; installer derives full Obsidian command ids, paths, and constant SC fields.
> - **Reload step:** three irreducible steps; public-API-only. Cmd+R stays. (User rejected internal-API in-flight registration to avoid Templater/SC internals coupling.)
> - **Stage + version:** 2-stage. Workshop 0.4.0 ŌåÆ **0.4.1**; validator 0.1.0 ŌåÆ **0.1.1**; audit 0.1.0 ŌåÆ **0.1.1**; project 0.2.0 ŌåÆ **0.2.1**. Bumps force consumer re-processing without triple-bump-protocol gymnastics.

> [!info] Disk-discovery findings (Phase 0)
> Both schemas locked from disk before brainstorming opened.
> - **Slash Commander binding shape** (12 default bindings on barebones disk): `{ name, id, action, icon, mode, triggerMode }`. No `keyword`. No `description`. Top-level: `{ version: 2, mainTrigger: "/", bindings: [...] }`.
> - **Templater command-id format** (templater main.js `add_template_hotkey()`): `addCommand({ id: <full-template-path>, name: "Insert <basename-no-ext>", icon: "templater-icon" })`. Obsidian prefixes with plugin id ŌåÆ resolved id is `templater-obsidian:<full-template-path>`. Templater also registers a sibling `create-<path>` command per template; v0.1.3 binds the `Insert` variant only (the runner templates are self-contained `app.vault.create()` scripts).

## ┬¦1 Manifest schema additions

Two new optional fields, applied to mechanism + blueprint manifests. Both additive; absence = no-op.

```jsonc
"templater_hotkeys": [
  { "template": "Validate.md" }
],
"slash_commander_bindings": [
  { "name": "validate", "template": "Validate.md" }
]
```

| Field | Required keys | Installer derivation |
|---|---|---|
| `templater_hotkeys[].template` | `template` (basename) | Prepends `{{templates_path}}/`; result lands in Templater's `enabled_templates_hotkeys[]`. |
| `slash_commander_bindings[].name` | `name` (lowercase verb) | What the user types after `/`; SC fuzzy-matches on this. |
| `slash_commander_bindings[].template` | `template` (basename) | Installer derives `id` = `action` = `"templater-obsidian:" + <full-template-path>`; sets `icon: "templater-icon"`, `mode: "any"`, `triggerMode: "anywhere"`. |

> [!warning] Cross-validation
> Each `slash_commander_bindings[].template` must also appear in the same manifest's `templater_hotkeys[].template` OR `files[]`. The installer emits `event: warning, step: slash_commander_bindings` and skips any binding referencing a template the manifest doesn't ship ŌĆö catches the typo class.

## ┬¦2 Two new installer helpers

Both mirror `applyExternalPlugins`'s posture: failure-loud, never throws, history entry per action. Wired into `installItem` immediately after `applyExternalPlugins` (`platform/install.js:269`).

> [!example]- A. `applyTemplaterHotkeys(tp, manifest, variables, history)` ŌĆö pseudocode
> ```
> target = ".obsidian/plugins/templater-obsidian/data.json"
> if !manifest.templater_hotkeys || empty: return
> if !exists(target): warn-history + Notice; return
> read raw ŌåÆ parse JSON; on error: warn-history + Notice + return (NEVER overwrite malformed)
> if data.enabled_templates_hotkeys not array: warn-history + Notice + return
> backup ŌåÆ write `${target}.beacon-backup` (overwrite existing backup; one-deep)
> for each entry in manifest.templater_hotkeys:
>   fullPath = `${variables.templates_path}/${entry.template}`
>   if data.enabled_templates_hotkeys.includes(fullPath): skip-history (action: skipped_existing)
>   else: data.enabled_templates_hotkeys.push(fullPath); applied-history (action: applied)
> write data.json (JSON.stringify(data, null, 2)) ONLY if any entries were appended
> ```
> Idempotency key: **string equality on full path**. Empty-seed `[""]` (Templater's first-install state) preserved.

> [!example]- B. `applySlashCommanderBindings(tp, manifest, variables, history)` ŌĆö pseudocode
> ```
> target = ".obsidian/plugins/slash-commander/data.json"
> if !manifest.slash_commander_bindings || empty: return
> if !exists(target): warn-history + Notice; return
> read raw ŌåÆ parse JSON; on error: warn-history + return
> if data.bindings not array: warn-history + return
> backup ŌåÆ `${target}.beacon-backup`
> for each entry in manifest.slash_commander_bindings:
>   cross-validate: entry.template must appear in manifest.templater_hotkeys OR manifest.files
>     (else: warn-history + skip)
>   fullPath = `${variables.templates_path}/${entry.template}`
>   cmdId = `templater-obsidian:${fullPath}`
>   if data.bindings.some(b => b.id === cmdId): skip-history (action: skipped_existing)
>   else: push {
>     name: entry.name, id: cmdId, action: cmdId,
>     icon: "templater-icon", mode: "any", triggerMode: "anywhere"
>   }; applied-history (action: applied)
> write data.json ONLY if any entries were appended
> ```
> Idempotency key: **`id` field equality** (full Obsidian command id). User-customized bindings on different ids stay; same-id collisions are no-ops.

### History entry schemas

| event | step | extra fields |
|---|---|---|
| `warning` | `templater_hotkeys` | `name` (manifest), `message`, `attempted_at` |
| `warning` | `slash_commander_bindings` | `name` (manifest), `binding_name`, `template`, `message`, `attempted_at` |
| `info` | `templater_hotkeys` | `name`, `template_path`, `action: applied \| skipped_existing`, `attempted_at` |
| `info` | `slash_commander_bindings` | `name`, `binding_name`, `command_id`, `action: applied \| skipped_existing`, `attempted_at` |
| `error` | either | `name`, `message`, `attempted_at` |

### Failure semantics

Every error path = Notice + history entry; **never throws**. Install of the rest of `installItem` continues. Same posture as `applyExternalPlugins`. Backup is one-deep, overwrite-on-edit, single file `<target>.beacon-backup` per data.json ŌĆö no timestamping (would accumulate). Recovery doc lives in landmine #12.

## ┬¦3 `.obsidian/` ban-lift + landmine #12

> [!warning] CLAUDE.md "Ask before acting" ŌĆö replace existing first bullet
> ```
> - Modifying anything inside `.obsidian/` of any vault EXCEPT for the two
>   installer-managed paths allowlisted in landmine #12:
>     - .obsidian/plugins/templater-obsidian/data.json
>     - .obsidian/plugins/slash-commander/data.json
>   Edits to these two paths must follow landmine #12's safety mechanics
>   (additive-merge-only, backup-on-edit, malformed-JSON guard, failure-loud history).
> ```
> Everything else in `.obsidian/` (snippets, appearance.json, hotkeys.json, workspace, every other plugin's data.json) stays ask-before-acting.

> [!info]- New landmine entry ŌĆö `Docs/landmines.md` #12
> ### 12. `.obsidian/plugin-data` ban-lift allowlist ŌĆö only two paths, only via the installer, only with the four safety mechanics
>
> The `.obsidian/` ask-before-acting gate is lifted for exactly two files:
> - `.obsidian/plugins/templater-obsidian/data.json`
> - `.obsidian/plugins/slash-commander/data.json`
>
> Both touched **only** by the installer, only via `applyTemplaterHotkeys` / `applySlashCommanderBindings`, and only under all four of:
> 1. **Additive merge.** Never strip, modify, or reorder pre-existing entries; only append new entries the manifest declares.
> 2. **Backup on edit.** Write `<target>.beacon-backup` before any modification (one-deep, overwrite-on-edit, single backup per target).
> 3. **Malformed-JSON guard (C4 parity).** If file is unreadable, unparseable, or has unexpected top-level shape (e.g., `enabled_templates_hotkeys` not an array, `bindings` not an array), installer logs a warning history entry + surfaces a Notice + returns. NEVER overwrites a malformed file.
> 4. **Failure-loud history.** Every applied / skipped / warning / error path writes a history entry under `step: templater_hotkeys` or `step: slash_commander_bindings` with full context.
>
> **Why the allowlist exists.** Templater's per-template `Insert <name>` commands are only registered when `enabled_templates_hotkeys[]` is populated; SC's slash bindings are persisted in its `data.json:bindings[]`. Both must be populated before the consumer's first reload after install, OR the user has to do manual configuration the platform can otherwise fully automate.
>
> **Why nothing else.** Editing other `.obsidian/` files cuts across user-customizable territory the platform has no claim on. The allowlist is exhaustive: any future cycle proposing a third path requires the same four safety mechanics, updates to CLAUDE.md + this landmine, AND explicit user approval.
>
> **Recovery.** If a `.beacon-backup` file diverges from the live data.json in a way the user wants to revert, copy the backup over the live file + reload Obsidian. Backups are not auto-rotated; manual cleanup is the consumer's call.
>
> Surfaced 2026-05-04 during v0.1.x close (T2.1-discovery ┬¦8 + T2.6 deferral); codified in v0.1.3.

## ┬¦7 Open trails, non-goals, risks

> [!info]- Open trails (resolved during S1 T1.x discovery)
> 1. **Empty-seed `enabled_templates_hotkeys: [""]`.** Templater seeds this on first install. Helper appends; empty stays in slot 0. Verified harmless ŌĆö Templater's `register_templates_hotkeys()` early-returns on falsy entries. Documented in helper source comment.
> 2. **Slash Commander `version: 2`.** Disk reports version 2; we don't write/touch this field. If a future SC version migrates to v3 with a different bindings shape, the helper needs version branching. Out of scope for v0.1.3.
> 3. **`.beacon-backup` collision with Obsidian Sync.** Sync may include or exclude `.obsidian/` depending on user settings. Landmine #12 documents the file's existence; consumer manages sync exclusions if relevant.

> [!warning]- Non-goals (reject if surfaced mid-cycle)
> - **Companion Obsidian plugin** ŌĆö would absorb both responsibilities via official APIs. Future cycle if ever; not v0.1.3.
> - **Generic `plugin_config[]` schema** ŌĆö YAGNI. Two targeted helpers ship; v0.2.0's Kanban plugin may extend with a third targeted helper. Reassess generic-vs-targeted then.
> - **Auto-installing Slash Commander** ŌĆö Obsidian's plugin install API is not exposed to scripts. Physics. Step 1 stays manual.
> - **Auto-reloading Obsidian** ŌĆö no script-accessible API. Step 3 stays manual. (User explicitly rejected internal-API in-flight registration in favor of public-API-only posture.)
> - **Conflict detection on user-customized `name` collisions** ŌĆö idempotency on `id` only. SC's picker disambiguates if multiple bindings fuzzy-match. YAGNI for v0.1.3.

> [!example]- Risks
> | Risk | Likelihood | Mitigation |
> |---|:-:|---|
> | Templater data.json schema changes between plugin versions | low | Helper does narrow append on `enabled_templates_hotkeys[]` only; malformed-JSON guard refuses to overwrite. |
> | Slash Commander data.json schema changes (e.g., version 3) | low | Helper checks `bindings` is array; otherwise warn + return. Future cycle adds version branching if needed. |
> | Backup file accumulates junk over re-installs | low | One-deep, overwrite-on-edit, single backup per target. No timestamping. Documented in landmine #12. |
> | Consumer's user-edited binding collides on `id` | low | Idempotency key is `id` (full Obsidian command id) ŌĆö same id means same intended action; binding is a no-op. User customizations on different ids stay. |
> | User runs installer with `[""]` empty-seed Templater data, expects clean array | low | We append; we don't compact. Empty string is a valid Templater no-op entry. Documented in helper source. |
> | Consumer's SC plugin not enabled | low | `if !exists(target)` fires; warn-history + Notice; install of rest continues. `external_plugins[]` helper covers the user-facing "install + enable" prompt. |

> [!info]- De-scoped explicitly
> - **Sub-folder runner templates.** All current runners live at `{{templates_path}}/<name>.md`. Helpers handle nesting via concatenation, but `name` derivation in the SC binding stays manual.
> - **Multi-vault parallel install testing.** Current harness is one vault per invocation. Parallel orchestration is v0.1.2's territory.
> - **Removing `external_plugins[]` warning when SC enabled but bindings present.** No-op ŌĆö warning only fires when SC is `required: true` and not enabled.
