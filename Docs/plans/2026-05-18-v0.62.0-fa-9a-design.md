## Why now

FA-2..FA-7 (v0.53..v0.59.1) shipped canonical `created_at:` + canonical `type:` across all 13 blueprints. Every note in every blueprint now carries a uniform creation timestamp + a uniform discriminator. The timeline reverse-query is mechanical.

FA-8 (v0.61.0) shipped `BacklinkPanel` — universal cross-blueprint reverse-query for the 6 canonical cross-ref keys (`people:`, `projects:`, ...). FA-9a generalizes the SAME reverse-query pattern across TIME instead of entity: "every note created in this timeframe across these blueprints" rather than "every note mentioning this entity".

The mechanism is a thin (~180 LOC) wrapper over a uniform Dataview query parameterized by scope (today/week/month) and blueprint-type filter.

### Class: `ActivityFeed`

```js
class ActivityFeed {
  /**
   * Render a time-windowed activity feed across blueprints.
   * @param {object} dv  — Dataview API in dataviewjs scope
   * @param {object} opts
   * @param {string} opts.scope        — "today" | "week" | "month" (default "today")
   * @param {string[]} [opts.blueprints] — canonical type: values to include (default: ALL entity-shape + timeline-shape types in vocab)
   * @param {string} [opts.groupBy="blueprint"] — "blueprint" | "hour" | "none"
   * @param {number} [opts.limit=50]
   * @param {string} [opts.title]      — optional H3 above the panel (e.g. "Today's activity")
   */
  render(dv, opts) { ... }
}
```

Internals:
- `_DEFAULT_BLUEPRINTS = ["daily", "meeting", "scratch", "scratch-day", "cowork-daily", "cowork-weekly", "cowork-monthly", "to-do", "journal", "project", "person", "team", "product", "trip", "budget", "paycheck", "invoice"]` — canonical type values from FA-1 vocab.
- `_resolveTimeWindow(scope)` → returns `{start, end}` ISO strings using `window.moment`. "today" = current YYYY-MM-DD ±0; "week" = isoWeek start..end; "month" = month start..end.
- `_query(dv, blueprints, window)` → `dv.pages().where(p => blueprints.includes(p.type) && p.created_at falls in window)`.
- `_renderEmpty(dv)` → italic-dimmed "No activity in this <scope>."
- `_renderGroupedByBlueprint(dv, pages)` → partitions by `p.type`, emits H4 per blueprint + BeaconCards row layout per group.
- `_renderGroupedByHour(dv, pages)` → partitions by `created_at` HH:00 bucket, emits H4 per hour (used by "today" scope).
- `_renderFlat(dv, pages)` → single BeaconCards row layout.

## Risk + mitigations

| Risk | Mitigation |
|---|---|
| ActivityFeed.render throws in dataviewjs scope when no notes match scope window | Empty-state ("No activity in this <scope>") + try/catch around query — degrade gracefully to Notice |
| `window.moment` not available in some Obsidian render contexts | Defensive `if (typeof window === "undefined" \|\| !window.moment)` → fall back to native Date parse; Notice on degraded path |
| `created_at` parse failure (e.g., legacy un-migrated notes with `created_at` as Date object vs ISO string) | `String(p.created_at)` coercion + permissive ISO regex; skip page on parse error |
| Performance: vault-wide query repeated 3× per cowork hub + 4× per Today.md | Acceptable for typical vault sizes; Dataview caches per-render. Future cycle may share query across embeds. |
| Day-rollover edge case (`scope: "today"` evaluated near midnight) | Resolve window AT render time, not class-load time. Each render computes `today` fresh. |
