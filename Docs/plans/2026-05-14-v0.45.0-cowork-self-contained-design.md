# v0.45.0 — cowork self-contained + UX polish wave 2

## Why

Post-v0.44.0 install in `accuris-sauce` surfaced six follow-on issues that cluster into two themes — (a) one architectural / namespacing concern and (b) five UX-consistency concerns. v0.45.0 fixes both in one MINOR.

### Architectural realization

The v0.44.0 Daily Hub points its `cowork-daily-hub-cards.js` query at `spice/daily/**/*.md` — the **daily blueprint's** notes. The user's mental model: **cowork should be a self-contained system that owns its own daily/weekly/monthly notes inside `spice/cowork/` and never reads from or writes to consumer notes outside its own directory**. Future scheduling/aggregation cycles will have cowork orchestrators write into these cowork-owned files; the existing patch-daily-callouts pathway can optionally mirror into the main daily note as a downstream output step, but the **primary write target stays inside `spice/cowork/`**.

This is the same principle as landmine #11 (module-directory invariant) — every blueprint owns one directory, and cross-module data flows via wikilinks only. v0.44.0 inherited a query that reached outside the cowork module. v0.45.0 closes that hole by introducing a new cowork-owned daily path family.

### UX-consistency defects

1. `Cowork.md` has the cowork buttons at top but is missing the global `SpaceNavButtons` row above them. Every other blueprint hub renders SpaceNavButtons at top.
2. The `CoworkHubNav` 4-card BeaconCards row looks visually different from the AccentButton-row aesthetic used by ScratchDayActions / scratch's hub. User wants the latter.
3. The "you are here" muted subtitle on the current-hub card is awkward — better to just omit the current-hub button entirely.
4. Daily Hub query (see Architectural Realization above) reaches into the daily blueprint, surfacing every auto-created daily note going back to April.
5. Clicking "This Week" opens a freshly created `2026-W20.md` that does NOT carry the cowork nav-button pattern at top.
6. The H1 title on weekly/monthly notes uses the raw ISO label `2026-W20` / `2026-05` rather than a human-friendly title.

## Scope

- `cowork@0.6.0 → 0.7.0` (MINOR — new path convention + NEW Daily Note template + NEW CoworkDailyActions helper + CoworkHubNav rewrite + nav-pattern enforcement across 7 files + better Templater-computed titles + Timeframes row 5 → 6 cards)
- `workshop_version 0.44.0 → 0.45.0` (MINOR) + `package.json` lockstep
- nav-buttons unchanged at `2.6.2`

### Out of scope (deferred)

- Cowork orchestrators rewiring to actually write into `spice/cowork/daily/YYYY-MM-DD.md` — this is the scheduling/aggregation cycle's job. v0.45.0 ships the path shape + the manual creation flow; the orchestrators will be re-pointed at the new path in a separate future cycle.
- Migration of existing daily-blueprint notes into cowork's directory — irrelevant since they serve different purposes.
- `Docs/Meta/Cowork-System.md` retirement — already a no-op (file does not exist).

## Decisions

| # | Decision | Why |
|---|---|---|
| D1 | NEW path convention `spice/cowork/daily/YYYY/MM-MMMM/YYYY-MM-DD.md` mirroring the daily blueprint's folder shape. | Visual consistency for users navigating the file tree. Self-contained inside `spice/cowork/` per the user's mental model. |
| D2 | NEW `Daily Note.md` Templater template at `ranch/templates/Daily Note.md`. Templater computes a Weekday-Month-Day-Year H1 at note creation. | Friendly display title without changing filename. |
| D3 | NEW `CoworkDailyActions` CustomJS class — provides a "+ Today" button on the Daily Hub that opens-or-creates today's cowork daily note via Templater. Mirrors `CoworkTimeframeButtons` semantics for the "This Week" / "This Month" create-cards. | Manual creation path until orchestrators are rewired. |
| D4 | Cowork.md Timeframes row grows 5 → 6 cards: `[Daily Hub] [Today] [Weekly Hub] [This Week] [Monthly Hub] [This Month]`. | Symmetry across the three timeframe pairs (Hub + Period). User-approved. |
| D5 | `CoworkHubNav` rewrites from BeaconCards 4-card row → AccentButton centered flex row, matching ScratchDayActions aesthetic. Drops "you are here" text. Current hub button is **excluded** (not rendered) instead of greyed-out. | User direction: match scratch's button style; drop "you are here". |
| D6 | Universal nav pattern across **every** cowork note: `SpaceNavButtons` → `---` → `CoworkHubNav` → `---` → content. Applies to: `Cowork.md`, `About Cowork.md`, `Daily Hub.md`, `Weekly Hub.md`, `Monthly Hub.md`, plus the 3 note templates (`Daily Note.md`, `Weekly Note.md`, `Monthly Note.md`). | User direction: "ensure all notes contain that pattern within this blueprint". |
| D7 | Templater-computed friendly H1 titles in note templates: Daily = `# <Weekday>, <Month> <D>, <Year>` (e.g. `# Thursday, May 14, 2026`); Weekly = `# Week <NN> · <Month> <D>–<Month> <D>, <Year>` (e.g. `# Week 20 · May 11–17, 2026`); Monthly = `# <Month> <Year>` (e.g. `# May 2026`). Filenames unchanged. | User direction on weekly/monthly titles; daily follows the same friendly pattern for consistency. |
| D8 | `cowork-daily-hub-cards.js` retargets from `dv.pages('"spice/daily"')` → `dv.pages('"spice/cowork/daily"')`. Filename regex changes from `\w+-YYYY-MM-DD` → `YYYY-MM-DD`. Card title uses friendly Weekday-Date label. | Closes the daily-blueprint cross-reach. |
| D9 | Add `cowork-daily` rule_fragment for `spice/cowork/daily/**/*.md` — required frontmatter `type: cowork-daily`, `day`, `created`; naming pattern `^\d{4}-\d{2}-\d{2}\.md$`. | Audit coverage parity with the existing cowork-weekly / cowork-monthly fragments. |

## Open items / FIX-LATER candidates

1. **Cowork orchestrators still reference daily-blueprint paths** in their SKILL.md bodies (`ensure-daily-note`, `patch-daily-callouts`). v0.45.0 ships the new path shape but does NOT rewire the orchestrators yet — that's the scheduling/aggregation cycle's scope. The orchestrator skill bodies will need updating in that cycle to choose between writing into cowork-owned files vs. patching main daily notes (or both).
2. **Migration tool for existing cowork users** — if anyone had existing weekly/monthly notes from v0.42.0/v0.43.0/v0.44.0, those notes stay where they are (their path conventions did not change in v0.45.0). The Daily Hub's retarget means any prior daily-blueprint references from `accuris-sauce` simply stop appearing; no data loss. Out of scope.
3. **AccentButton mobile behavior** — ScratchDayActions uses `flex: true` for AccentButton which makes buttons grow to fill row. CoworkHubNav will use the same posture; if mobile rendering is awkward (e.g. 3 buttons fight for width on a narrow viewport), revisit in a small patch.
