#### Shared API contract (paste verbatim into every implementer prompt)

```
======================== SHARED API CONTRACT ========================

Every skill body lives at platform/blueprints/cowork/skills/{orchestrators|skills}/<id>/SKILL.md

YAML frontmatter (REQUIRED, verbatim field names):
---
name: cowork:<id>
description: <one-line summary, <120 chars>
inputs:                         # sub-skills only; orchestrators omit
  <key>: <type>                 # e.g. range: string
outputs:                        # sub-skills only; orchestrators omit
  <key>: <type>                 # e.g. markdown: string
schedule: <human cron>          # orchestrators only; e.g. "Daily 7:03 AM MT (~2.5min jitter)"
scope: life | work | shared     # orchestrators only
tags: [<tag>, ...]              # both; e.g. [cowork, gather]
---

ORCHESTRATOR body shape (25-70 lines):
- Opening 1-2 sentence summary
- "## Pre-flight" section: 1-3 numbered Skill invocations (check-vault-routing, date-context, ensure-daily-note)
- "## Gather" section: numbered Skill invocations
- "## Write" section: numbered Skill invocations + daily-note patch at <!-- COWORK_CALLOUTS -->
- "## State" section: numbered Skill invocations (active-threads/weekly-snapshot updates)
- Each step: "Use Skill cowork:<sub-id> with { <args> }"
- NO raw MCP tool calls; NO inline data parsing
- Closing "Done." or "## Done"

SUB-SKILL body shape (FAT, 25-80 lines):
- Opening 1-2 sentence summary
- "## Inputs" section: list keys + types
- "## Outputs" section: list keys + types
- "## Steps" section: numbered procedural body — agent MAY call MCP tools here
- "## Returns" section: literal output shape (markdown block or JSON)
- "## Errors" section: failure modes + Notice/abort/empty-string convention
- FAT means: orchestrator never sees raw MCP returns. Sub-skill normalizes.

WRITING CONVENTIONS:
- No emojis. Severity markers (🔴🟡🟢) ONLY in audit reports (per ero CLAUDE.md non-negotiable).
- Wikilinks for note targets: [[Note Name]] or [[Note|Display]].
- Absolute dates: YYYY-MM-DD only. No relative.
- Vault-internal paths use sauce-shape: spice/<module>/, ranch/, NOT legacy Timestamps/, Boards/, etc.
- Daily-note callout insertion: target <!-- COWORK_CALLOUTS --> marker; fall back to ## Notes heading-parse with Notice on absence.

PROHIBITIONS:
- No mocks/stubs without TODO block.
- No "TBD" frontmatter values; if data unknown, use empty-string + comment.
- No new top-level vault dirs; everything under spice/cowork/ or .claude/skills/cowork/.

REFERENCES:
- Design doc: Docs/plans/2026-05-10-v0.30.0-cowork-blueprint-design.md §3
- Existing prompt bodies (legacy reference for content): /Users/willfell/Documents/obsidian/sync/sauce/{ero,headspace}-sauce/Cowork/prompts/*.md
- Existing context bodies (legacy reference): /Users/willfell/Documents/obsidian/sync/sauce/{ero,headspace}-sauce/Cowork/context/*.md
- Path translation: predecessor-machine /Users/willfellhoelter/notes/sauce/... → /Users/willfell/Documents/obsidian/sync/sauce/...
=====================================================================
```
