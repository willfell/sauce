# v0.18.1 BeaconButton Adoption Sweep Implementation Plan

> **For the implementing agent:** REQUIRED SUB-SKILL: Use `de:executing-plans` (or `de:subagent-driven-development` if running in this session) to implement this plan task-by-task.

**Goal:** Migrate the remaining 3 outline-accent action button sites (new-meeting-button + workstream Add + workstream Remove) to `customJS.BeaconButton.render(...)`, AND shorten 2 nav-button labels (`"+ Today's To-Do"` → `"To Do"`; `"+ Today's Journal"` → `"Journal"`). BeaconButton API contract (quoted from `platform/mechanisms/beacon-button/beacon-button.js:39`): `customJS.BeaconButton.render(parent, { label, icon, onClick, flex?, disabled?, tone? }) → HTMLButtonElement`.

**Tech Stack:** CustomJS classes (no transpile), Templater dataviewjs blocks, JSON manifests, Node.js test harness (run-install + run-helper-cases + run-renderer).

**Versions at start:** workshop 0.18.0 / meetings 0.2.0 / project 1.3.3 / to-do 0.1.2 / journal 0.1.0.
