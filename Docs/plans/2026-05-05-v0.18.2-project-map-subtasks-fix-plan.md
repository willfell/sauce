# v0.18.2 Project Map Sub-Tasks Bug Fix — Implementation Plan

> **For the implementing agent:** REQUIRED SUB-SKILL: Use executing-plans to implement this plan task-by-task.

**Goal:** Stop sub-task cards (Task Board Cards) from leaking into the Project Map's "Unassigned" bucket by adding two predicates (`!p.task_parent` + `!p.file.path.includes("/board/")`) to the existing `dv.pages().where(...)` filter in `ProjectWorkstreams`.

**Architecture:** Single-file behavior change in `platform/blueprints/project/helpers/project-workstreams.js`. The recursive `dv.pages(projectDir)` is preserved; the existing filter gains two more predicates that together form a belt-and-suspenders sub-* exclusion (frontmatter signal + layout signal). No installer / schema / mechanism changes. Lockstep version bumps across blueprint manifest + workshop catalogue + barebones subscription per gotcha 11. PATCH workshop 0.18.1 → 0.18.2; project 1.3.4 → 1.3.5.

**Tech Stack:** Vault platform; Node.js test harnesses (`run-install.js`, `run-helper-cases.js`, `run-renderer.js`); Obsidian 1.x + Dataview + CustomJS + Templater + Kanban plugins (manual smoke surface).
