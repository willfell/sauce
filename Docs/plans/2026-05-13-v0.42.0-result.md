## What shipped

- **Three new sub-hubs** under `spice/cowork/`: `Daily Hub.md`, `Weekly Hub.md`, `Monthly Hub.md`. Each renders cards via a dedicated CustomJS helper (CoworkDailyHubCards / CoworkWeeklyHubCards / CoworkMonthlyHubCards) that queries the relevant timeframe path, groups by year/month, sorts recent-first, and delegates layout to BeaconCards.
- **Two new note-shell families**: weekly notes at `spice/cowork/weekly/YYYY/YYYY-Www.md` and monthly notes at `spice/cowork/monthly/YYYY/YYYY-MM.md`. Templates live at `ranch/templates/Weekly Note.md` and `Monthly Note.md`. Both inherit the daily template's `<!-- COWORK_CALLOUTS -->` anchor for future cowork patching.
- **Four prompt-file stubs** at `spice/cowork/prompts/{morning-briefing,eod-review,weekly-review,monthly-review}.md` — empty-bodied path-convention placeholders for the future scheduling cycle to fill.
- **Three new global nav-buttons**: `cowork-hub` (openLink → `Cowork.md`), `cowork-weekly-this` (runTemplaterTemplate for this-week), `cowork-monthly-this` (runTemplaterTemplate for this-month). Net global row: **Daily · Cowork · This Week · This Month** (left-to-right by `order`).
- **Two new slash commands** owned by cowork: `/weekly` (open this-week, or `/weekly hub`) + `/monthly` (open this-month, or `/monthly hub`). Existing `/cowork` body refreshed with new Timeframes section.
- **One new sub-skill**: `cowork:scaffold-timeframes` — idempotent, creates this-week + this-month notes from templates if missing, verifies prompt-stub presence, returns JSON receipt. Composed into `cowork:bootstrap-vault` as a final step (step 25; prior step 25 renumbered to 26); standalone-invocable.
- **Cowork.md master hub** updated: scaffold-status dataviewjs callout (detects engagements + this-week + this-month + prompt-stubs gaps; renders `> [!warning]+ Cowork scaffold incomplete` if anything missing); new Timeframes section (3-card BeaconCards row pointing at the sub-hubs) above the existing Engagements section.
- **Six new rule_fragments** validating the new artifacts — 3 hub note types, 2 note-shell types, 1 prompt-stub schema. cowork manifest's `rule_fragments[]` now has 11 entries.

## FIX-LATER-NOTEs

- **FLN-1: Workshop subscription does NOT include cowork.** The workshop's `ranch/platform-subscription.json` only subscribes to project / daily / meetings / scratch. cowork has never been part of the workshop's self-dogfood subscription. S10's plan called for a workshop subscription bump for cowork, but there was no entry to bump — skipped without action. The cowork@0.4.0 surface is verified end-to-end by the test harnesses (run-cowork-smoke + run-integration-smoke + run-claude-surface scaffold tmpdir vaults that DO subscribe to cowork). Future cycle: decide whether cowork belongs in the workshop's self-subscription. If yes, that cycle bumps the workshop subscription + handles the materialization side effects.
- **FLN-2: Project version skew.** `ranch/platform-subscription.json` pins `project@1.5.0` but the workshop manifest catalogue has `project@1.6.1`. `sauce reinstall --vault $PWD` exits 1 with `skipping project — subscription pins project@1.5.0 but workshop has 1.6.1`. Pre-existing skew; introduced before v0.42.0. Trivially fixed by bumping the subscription to `1.6.1`. Out of scope for v0.42.0 but blocks any reinstall-based dogfood until resolved.
- **FLN-3: Manual Obsidian smoke skipped.** S10.5 (open Obsidian, click each new nav-button, verify the timeframe notes get created, verify scaffold-status callout state-detection narrows) was skipped per the autonomous-execution mandate. The whole-suite preflight green (especially run-renderer's nav-button composition cases, run-cowork-smoke's hub presence cases, and run-integration-smoke's bootstrap-then-reinstall cases) gives strong confidence the surface works correctly, but a real Obsidian smoke would catch e.g. visual rendering issues, icon-resolution issues, or DataviewJS runtime quirks. Recommended: run the manual smoke at next opportunity to validate the user-facing experience before personal-vault adoption.
- **FLN-4: nav-button manifest defects caught only in S8 review.** S6's manifest delta initially specified `"template": "Weekly Note"` (wrong field — installer expects `template_source` with `.md` extension) and `"folder_prefix": "{{module_directory}}/weekly/"` (trailing slash; renderer joins with `/` producing double slash). Both were defects in the design doc + plan, NOT introduced by the implementer. S8b fix landed at `81bf07b`. Lesson for future plans: when prescribing nav-button manifest entries, cross-check against the most recent shipped example (here: scratch's `runTemplaterTemplate` entry would have shown the correct field name + no-trailing-slash convention).
- **FLN-5: Test-asset overage in run-helper-cases / run-cowork-smoke.** S9 implementer wrote more sub-asserts than the design's targets (+31 vs +18 for helper-cases; +10 vs +6 for cowork-smoke). All passing — overage came from being thorough on lint + structural checks. Note for future plan-baseline calibration: when the design says "+N sub-asserts", that's typically the floor.

## What this enables for follow-on cycles

- **Aggregation cycle (next priority).** cowork's existing `write-summary-weekly` + `write-summary-monthly` skills can now be retargeted to write into `spice/cowork/weekly/<this-year>/<this-week>.md` instead of `spice/cowork/summaries/weekly/<engagement>/<this-week>.md`. The path convention is locked in. A new sub-skill `cowork:aggregate-daily-into-weekly` reads daily notes' tagged lines (`#decision`, `#shipped`, `#blocker`, etc.) and patches the weekly note's `<!-- COWORK_CALLOUTS -->` anchor — same patch mechanism the daily blueprint already uses. Symmetric for weekly → monthly.
- **Scheduling cycle.** A runner (launchd / cron / GitHub Actions / Anthropic Managed Agent) fires a headless Claude session against the vault with `cowork:morning-briefing --prompt-file spice/cowork/prompts/morning-briefing.md` (or equivalent). The prompts/ directory's path convention is already locked in. The scheduling runner reads the file body at runtime; user edits the file, next run picks it up — no redeploy. Empty stubs become populated bodies.
- **CLAUDE.md cohesion** is largely already covered by claude_surface[]'s 4 new resolvers rows. Future minor refresh may add a cowork-specific "Timeframe taxonomy" context-doc surface if scheduling needs it.
- **Per-note prev/next inline navigation** is a straightforward small follow-up cycle — just add inline markdown links computed at template-creation time to Weekly Note + Monthly Note + (optionally) the daily template. Hub-to-hub global nav covers the primary use case for now.
