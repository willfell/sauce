## Outcome summary

> [!abstract] What landed
> The journal-blueprint cycle adds a journaling surface to the platform with near-zero novel mechanism work — every primitive it depends on already shipped in earlier cycles. journal@0.1.0 is a near-clone of to-do@0.1.0: same `runTemplaterTemplate` nav-button action, same `templater_folder_templates` registration, same date-routed `module_directory` layout. > [!abstract] Why this matters
> v0.13.0 validates that the v0.4.0/v0.4.2 runTemplaterTemplate foundation is stable enough to absorb new date-routed blueprints with **manifest + template + one-line ICONS entry** — no installer changes, no harness changes, no new schema fields. The cycle finished single-stage with zero inline-CF sub-stages because the design phase locked the right precedent (to-do) and the implementation followed it near-verbatim. Future date-routed blueprints (trips, finance, focus-sessions, etc.) inherit this same minimal-touch posture.

## Lessons learned

> [!warning]- 1. Plan-correction patterns in flight (3 corrections; 1 doc commit)
> The plan needed three corrections during execution, all caught at task entry by the implementer (commit `b36b8db`):
>
> - **Workshop CWD path prefix.** Plan initially used `tmp-test-barebones-vault/...` (no `../`); corrected mid-execution to `../tmp-test-barebones-vault/...`. The workshop's working directory is `workshop/poc-vault/`, and `tmp-test-barebones-vault/` is a sibling directory at `workshop/`, so any harness invocation from the workshop CWD requires the `../` prefix.
> - **`{{views_path}}` placeholder discipline.** Plan T1.5 originally specified hardcoded `Docs/Meta/Views/customjs-guard` in the template body; corrected to `{{views_path}}/customjs-guard`. The v0.5.0 lesson (consumer-vault path-portability via installer placeholders) needs to be enforced in plans, not just discovered in execution.
> - **Workshop self-sub blueprints[] semantic.** Plan T1.8 implied workshop self-sub should subscribe to journal blueprint; corrected during execution — workshop dogfoods **mechanisms only** (`blueprints[]` stays empty). Barebones is the blueprint regression target.
> - **T1.9 commit-message scoping.** Plan implied a single bundled commit covering both workshop + barebones subscriptions; corrected — barebones is not git-tracked, only the workshop self-sub edit committed.
>
> **Takeaway:** plans authored from a fresh-session prompt can miss workshop conventions (CWD prefixes, placeholder discipline, what's git-tracked vs. filesystem-only). Codify these as plan-author checklist items: (a) every harness/path command from workshop CWD prefixes `../`; (b) every template body uses `{{views_path}}` / `{{templates_path}}` / `{{content_path}}` placeholders; (c) workshop self-sub `blueprints[]` always empty.

> [!info]- 2. Single-stage cycles can land clean (first since v0.4.0 to-do)
> v0.13.0 is the first cycle since v0.4.0 to-do to land smokes-green on the first try with **zero inline-CF sub-stages**. Reserved 0.13.x headroom went unused. The cycle achieved this because: (a) design correctly identified journal as a near-clone of an existing precedent (to-do), (b) the only mechanism touch was a single ICONS map line, (c) all primitives (runTemplaterTemplate, templater_folder_templates, module_directory mkdir, beacon/ namespace) were already battle-tested.
>
> **Takeaway:** when a cycle's design phase finds a strong precedent + zero novel mechanism surface, single-stage is the right shape. The reserved-headroom posture (Y.Z.x for inline-CF) is still cheap insurance, but its non-use is itself a positive signal — the design phase priced the work correctly.

> [!info]- 3. Subagent-driven execution worked well for near-clone cycles
> T1.2 (nav-buttons ICONS patch), T1.4 (journal manifest), and T1.5 (Templater template body) were dispatched as subagent tasks; master handled mechanical edits (catalogue bumps, version bumps, harness runs, commits). The split worked because each subagent task was: (a) self-contained (one or two file edits), (b) authored against a clear precedent (to-do manifest + headspace journal template), (c) verifiable by master via diff review before commit.
>
> **Takeaway:** for near-clone-of-existing-blueprint cycles, the subagent + master mechanical-edits split scales well. Reserve subagent dispatches for content-authoring tasks where the precedent is locked; keep master in the loop for catalogue/version/harness mechanics where workshop conventions matter.
