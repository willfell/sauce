# v0.44.0 — Cowork UX polish + character pass

## Why

Six concrete defects surfaced after the v0.43.0 cowork@0.5.0 install. They cluster into two themes — broken renderer plumbing and missing visual character. v0.44.0 fixes both in one MINOR.

### Defects

1. **Cowork nav-bar button renders as a "C" letter fallback.** `space-nav-buttons.js`'s `ICONS` map has no `users-round` entry; unknown icon names fall back to first letter of label.
2. **Cowork.md hub is bloated with documentation** (what-is-cowork prose, full skills tables, context callout, getting-started) — belongs on a separate About page, not the operational hub.
3. **Cowork hub has no in-document nav-buttons row** — unlike scratch which renders `SpaceNavButtons` at top of `Scratch Hub.md`.
4. **Daily/Weekly/Monthly Hub render an ugly markdown text-link row** (`> [[Cowork|◀ Cowork]] · [[Daily Hub]] · …`) instead of real buttons.
5. **"Nothing here yet" repeats under every month/year heading.** `cowork-{daily,weekly,monthly}-hub-cards.js` call `BeaconCards.render({items, titleField, subtitleField, bodyField, linkField})` but the real API is `{pages, title, subtitle, target, …}`. `pages` is always `[]` → BeaconCards prints its default empty-state per heading. Never tested with real data.
6. **Weekly Hub empty-state copy** ("No weekly notes yet · Click This Week in the nav row to create one") references a nav-button that was retired in v0.43.0; the create-this-week affordance now lives inside Cowork.md's Timeframes row.

## Scope

- `cowork@0.5.0 → 0.6.0` (MINOR) — new helper, new content file, hub restructure, broken-API fix, manifest icon swap.
- `nav-buttons@2.6.0 → 2.6.1` (PATCH) — add `briefcase` SVG to ICONS map.

## CoworkHubNav shape (reference)

```js
class CoworkHubNav {
  async render(dv) {
    if (dv.container.closest(".markdown-embed")) return;
    while (dv.container.firstChild) dv.container.removeChild(dv.container.firstChild);

    const ICONS = {
      cowork:   "<svg ... briefcase ...>",
      daily:    "<svg ... calendar ...>",
      weekly:   "<svg ... calendar-range ...>",
      monthly:  "<svg ... calendar-days ...>",
    };

    const currentBaseName = (dv.current()?.file?.name || "");
    const items = [
      { label: "Cowork",      path: "spice/cowork/Cowork.md",      _icon: ICONS.cowork  },
      { label: "Daily Hub",   path: "spice/cowork/Daily Hub.md",   _icon: ICONS.daily   },
      { label: "Weekly Hub",  path: "spice/cowork/Weekly Hub.md",  _icon: ICONS.weekly  },
      { label: "Monthly Hub", path: "spice/cowork/Monthly Hub.md", _icon: ICONS.monthly },
    ].map(it => ({
      ...it,
      _isCurrent: currentBaseName === it.label || currentBaseName === it.path.replace(/^.*\//,"").replace(/\.md$/,""),
      file: { name: it.label, path: it.path }
    }));

    await customJS.BeaconCards.render(dv, {
      pages: items,
      title: p => p.label,
      icon:  p => p._icon,
      subtitle: p => p._isCurrent ? "you are here" : null,
      target: p => p.path,
      onClick: (p) => { if (!p._isCurrent) app.workspace.openLinkText(p.path, ""); },
      layout: "row",
      columns: 4,
    });
  }
}
```

## Visual model (post-v0.44.0)

### Cowork.md
```
# Cowork
[ Cowork (you are here) ] [ Daily Hub ] [ Weekly Hub ] [ Monthly Hub ]
> [!warning]+ Cowork scaffold incomplete (only when gaps exist)
---
## Timeframes
[ Daily Hub ] [ Weekly Hub ] [ This Week ] [ Monthly Hub ] [ This Month ]
---
## Engagements + cadences
<bootstrap-rendered table>
---
[ About Cowork → ]
```

### Daily Hub.md
```
[ Cowork ] [ Daily Hub (you are here) ] [ Weekly Hub ] [ Monthly Hub ]
# Daily Notes
### May 2026
[card] [card] [card]
### April 2026
[card] [card]
```
When no daily notes exist: only the nav row renders. Heading + nothing.

### Weekly Hub.md / Monthly Hub.md
Symmetric.

## Open items / FIX-LATER candidates

1. **Trip-friendly empty-state for hubs with zero notes** — current direction is "render nothing". If you later want a single tasteful "Hit This Week in Cowork to create your first" callout, that's a one-line revert. Tracked as a possible v0.44.x patch only if it nags in actual use.
2. **`cowork-timeframe-buttons.js` create cards' click affordance** — the "This Week" / "This Month" cards have an arrow-glyph removed but the create-or-open semantic is otherwise unchanged. If users find it ambiguous which cards create vs. open, consider a subtle "Create" badge in v0.45.0.
3. **Wave-3 `Docs/Meta/Cowork-System.md` retirement** — file does not exist; no-op this cycle. Logged here for cycle-record completeness so a future audit doesn't relitigate.
