## What shipped

Per-project wiki area at `spice/projects/<slug>/wiki/`:

- Per-project `Wiki.md` hub note embedding "+ New Wiki Note" `AccentButton` (v0.49.0 inside-block sentinel) + `ProjectWikiCards` grid of sibling wiki-notes (sorted by `created` desc; empty-state callout).
- Wiki notes are user-titled (e.g. `Architecture-brainstorm.md`); frontmatter shape `{type: wiki-note, project: [[<Name>]], project_slug, created, tags: [wiki-note]}`.
- Existing projects backfilled via NEW `applyWikiBackfill` installer helper (~200 LOC in `platform/install.js`) — walks `spice/projects/*/`, detects project root by `type: project` frontmatter regex, materializes `wiki/Wiki.md` from `Template, Wiki Hub.md` with `{{prompts.slug}}` + `{{prompts.name}}` substituted from project directory + frontmatter. Idempotent (skip-if-exists).
- New projects get the wiki sidecar automatically via project entity-create's `extra_files[]` extension (`{ filename_pattern: "wiki/Wiki.md", body_template: "Template, Wiki Hub.md" }`).
- "Wiki" tab-button on `ProjectNavButtons` (Lucide `book-open` SVG) — visible in project / task / wiki-note contexts; auto-hidden on the wiki-hub itself. Adds `wiki-hub` + `wiki-note` context detection in `detectContext()`.
- Two new rule_fragments validate wiki-hub + wiki-note frontmatter shapes; wiki-note fragment uses `scope.exclude_basenames: ["Wiki.md"]` to skip the hub.

## Stage adjustments mid-flight

- **S4 retired in S0.3:** plan called for adding `scope.exclude` support to the audit rule-runner, but baseline check showed the runner already supports `scope.exclude_basenames`. Manifest fragment + tests use the existing field; S4 task removed.
- **R-WIKI design simplified:** plan originally had 3 dedicated R-WIKI cases; in practice, adding `wiki-note` to the existing `ENTITY_CREATE_SITES` table earned 6/7 of the assertion coverage for free (R-EC-ICON-8), plus one targeted R-WIKI-1 case for the template body shape (sentinel + AccentButton + ProjectWikiCards).
- **install.js metadataCache substitute:** install.js cannot use Obsidian's `parseYaml` (per top-of-file note); the backfill helper extracts `type: project` from the `---` frontmatter block via narrow regex instead of `metadataCache.getFileCache`.
