# v0.23.0 Sauce Rebrand Implementation Plan

> **For the implementing agent:** REQUIRED SUB-SKILL: Use `de:executing-plans` (or `de:subagent-driven-development`) to implement this plan task-by-task.

**Goal:** Rename workshop clone dir `Beacon/` → `pantry/`; rename CLI binary `beacon` → `sauce`; full Sauce platform rebrand (project name, env vars, ASCII banner, GitHub repo). Resolves the macOS APFS case-collision RUNTIME breakage from v0.22.1.

**Architecture:** Three-stage cycle. S1 lands TDD harness baseline (3 harnesses RED for new behavior); S2 atomic mass-rename across install.sh + platform/ + docs (harnesses GREEN); S3 smoke + GitHub rename + cycle close.

**Tech Stack:** Node.js (CLI dispatcher), bash (install.sh), 5 test harnesses (run-install-sh.js / run-bootstrap.js / run-cli.js / run-helper-cases.js / run-renderer.js). All green at start (45/20/14/387/23). Workshop_version 0.22.1 → 0.23.0 MINOR.

**Pre-design reference:** `Docs/plans/2026-05-06-v0.23.0-sauce-rebrand-design.md` (sections 1–14 quote literal call-sites for every file touched).

**Reserved-headroom:** Wide-touching mechanical + brand prose. Expect 2-4 CFs from missed call-sites (notably sed regex collision risk on `beacon-button` mechanism name). Two-stage subagent review (spec then quality) is load-bearing per v0.20.0 + v0.22.1 precedent.
