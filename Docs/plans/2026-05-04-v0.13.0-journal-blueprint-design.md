# v0.13.0 Design — journal blueprint

> [!abstract] Cycle goal
> Ship a NEW `journal@0.1.0` blueprint mirroring headspace's `Daily Journal` pattern. One date-routed file per day under `beacon/journal/YYYY/MM-MMMM/Journal-YYYY-MM-DD.md`. ## Architecture

> [!info] Wiring (right-to-left)
> 1. User clicks `+ Today's Journal` button rendered by `nav-buttons` mechanism inside any note containing the dataviewjs nav-buttons block.
> 2. `space-nav-buttons.js` dispatches the button's `runTemplaterTemplate` action (existing branch, shipped v0.4.0 with split-field schema in v0.4.2).
> 3. Renderer resolves folder = `moment().format("[beacon/journal]/YYYY/MM-MMMM")`; filename = `Journal-` + `moment().format("YYYY-MM-DD")` = `Journal-2026-05-04`.
> 4. `templater.create_new_note_from_template` materializes the file from the journal blueprint's template, running its frontmatter `<% tp.date.now(...) %>` substitutions.
> 5. Manual file creation under `beacon/journal/...` (e.g., via Calendar plugin or file explorer) auto-applies the same template via the registered `templater_folder_templates` entry.

> [!info] Cross-blueprint wiring
> Frontmatter `daily_note` field carries `[[<% tp.date.now("dddd-YYYY-MM-DD") %>]]` — wikilinks to the consumer's daily blueprint file at `beacon/daily/YYYY/MM-MMMM/<DayName>-YYYY-MM-DD.md`. Resolves correctly when both blueprints are installed (which the workshop self-subscription + barebones subscription both do). Loose coupling — if daily isn't installed, the wikilink is unresolved-but-harmless.
