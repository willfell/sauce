## FIX-LATER-NOTEs

- **FLN-v64-1** (Important, code-quality reviewer flagged at S1): `_resolveTimeWindow(scope, asOf)` does NOT validate that `window.moment(asOf)` returns a valid moment. If `asOf` is garbage (e.g. `"banana"`), `.format()` yields `"Invalid date"` strings, the time window is non-null, the `!timeWindow` guard at line 74 doesn't fire, and the predicate `ts >= "Invalid date"` silently filters out everything. Suggest adding `if (!now.isValid()) throw new Error("invalid asOf: " + String(asOf));` in both useMoment + native-Date branches. Currently dormant: `SpaceDailyDashboard` extracts `asOf` from a filename-regex-validated date string, so the garbage-input scenario doesn't fire in practice. Worth fixing in a future activity-feed PATCH.

- **FLN-v64-2** (Suggestion, code-quality reviewer flagged at S5): `caseFA4TimelineManifests` in `run-helper-cases.js` uses `m.version >= floors[bp]` — string-lexicographic compare. Will silently false-negative once any timeline blueprint reaches a two-digit minor (e.g., `"0.10.0" < "0.5.0"` lexicographically). Replace with semver-aware comparison helper.

- **FLN-v64-3** (Suggestion): `run-activity-feed.js` shim's `textContent` recursive getter prepends `" "` between children; real DOM concatenates without separators. Irrelevant since asserts use `indexOf` substring search, but worth a one-line comment for future maintainers.

- **FLN-v64-4** (Observation, S10): all four consumer vaults were 5+ workshop versions behind at cycle start (`workshop_version: 0.59.11`). The plan's S10 assumed one-version-bump cascades. Reality required programmatic catalog-alignment across 5-6 blueprints + 2 missing mechanisms per vault. Future cycles should call out "vault is N versions behind" as a baseline check and decide alignment scope before bumping workshop.

- **FLN-v64-5** (Observation, S10): No tap-PR auto-merge automation exists in the `willfell/homebrew-sauce` repo (no GitHub Actions workflows). Past PRs were merged manually by the user within 1-10 minutes of creation. Consider adding a tap-side workflow that auto-merges PRs from the bump-tap branch when CI is green, to reduce cycle latency.
