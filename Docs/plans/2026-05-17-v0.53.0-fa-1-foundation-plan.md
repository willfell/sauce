## Risk / FIX-LATER pre-notes

- **No consumer migration in this cycle.** `--apply` is wired but no blueprint has been migrated yet, so `sauce audit --frontmatter-alignment` against headspace will surface a large `legacy_key_used` count (intentional — informational only at this severity).
- **In-vault validator extends loader** is implemented but won't be exercised against any consumer this cycle (no blueprint opts in). Real-vault exercise lands at FA-2.
- **YAML coverage scope.** The `cmd-migrate-frontmatter` parser reuses the audit walker's minimal YAML parser (`platform/audit/walker.js` `_parseFrontmatterYaml`). Same scope: top-level scalars + list-of-scalars + quoting variants. Inline-flow arrays `[a, b]` and nested mappings beyond list-of-strings remain unsupported (failure-loud, same posture as existing audit).
- **Migration backup invariant.** `--apply` writes a `.sauce-backup/<rel-path>/<timestamp>/` sidecar per touched file. Mirrors v0.52.0's wiki-to-docs migration backup convention.
