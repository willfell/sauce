# v0.60.0 — public-readiness follow-up (CLOSED 2026-05-18, CI validated)

## Summary

Three independent fixes shipped as one MINOR bundle. All flagged earlier today by the public-readiness cycle's security report (`Docs/plans/2026-05-18-public-readiness-security-report.md`) and the post-Phase-2 GitHub audit. Net effect: every defense-in-depth call-site identified in that report is now closed; CI on `main` is green for the first time since 2026-05-17.

| Finding | File | Fix shape |
|---|---|---|
| F-1 (medium) | `platform/install.js` | `gitState` body rewrites three `execSync` template-literal calls into a local `_git()` helper that uses `spawnSync` with array-form args. Shell-string interpolation of user-controlled paths eliminated. The original `2>/dev/null` shell redirection preserved for `describe` via `stdio[2] = "ignore"` on that one call. |
| F-1b (medium) | `platform/bootstrap.js` | New `_shellSingleQuote(s)` helper at file top wraps interpolated paths in POSIX single quotes using the standard `'\''` close-reopen idiom for embedded quotes. Applied to `vaultPath` (becomes `SAUCE_VAULT` assignment), `scriptsDir` (becomes new `_SAUCE_SCRIPTS_DIR` variable referenced by the `case` PATH-already-present check), and `cliPath` (the `sauce` shim). Helper exported as `_shellSingleQuote` (underscore-prefixed; private-but-testable). |
| CI default-subscription | `platform/bootstrap-lib/wizard.js` | One-line: `"icons"` inserted between `"customjs-guard"` and `"nav-buttons"` in `DEFAULT_MECHANISMS_CHECKED`. Resolves the 50+-consecutive-CI-failure streak rooted in nav-buttons skipping itself because its `icons` dependency was missing from the default subscription. |

### Tests added

- **V60-SQ** (run-helper-cases.js, +40 LOC) — 9 sub-asserts piping `_shellSingleQuote` output through real `bash` via a tmpdir script. Covers: plain path, spaces, `$VAR`, backticks, double quotes, single quotes, backslash, and a deliberate injection attempt (`"; rm -rf ~; #`). All round-trip literally; no expansion.
- **BS10 updated** (run-bootstrap.js, +3 LOC) — replaces the old `SAUCE_VAULT="${vaultPath}"` substring assertion with `SAUCE_VAULT='${vaultPath}'` + a separate `export SAUCE_VAULT` line assertion, matching the new POSIX-safe split form.

## Decisions confirmed at execution time

Following the design's "Open questions" section:

1. **`v0.60.0` MINOR** for the bundle (vs three separate PATCHes). Confirmed by user before execution.
2. **F-1 `describe` stderr handling** — kept the suppression by passing `stdio: ["ignore", "pipe", "ignore"]` to the helper invocation for `describe` only. The other two calls keep `stderr: "pipe"` because their failures (not-a-git-repo / git-unavailable) should surface in the throw-message rather than silently null.
3. **F-2 (`eval`-dispatch in `run-audit.js`)** — deferred as designed. Strictly cosmetic; the test runner isn't shipped via brew. Picks up if/when a test-harness cleanup cycle is otherwise warranted.
