# v0.60.x — public-readiness follow-up

## Background

The public-readiness cycle closed 2026-05-18 as report-only by design. That cycle's security/PII report surfaced two medium defense-in-depth findings (F-1, F-1b) that explicitly belong in a follow-up cycle. The same cycle's GitHub-side audit independently surfaced a pre-existing CI defect on `main`: the `preflight` workflow has failed on every push for at least two days because the default subscription generated by `sauce bootstrap` includes `nav-buttons` but not `icons`, and the installer's "1 error/skip history entry → exit 1" verdict treats nav-buttons' skip-for-missing-dep as a failure.

This cycle bundles all three. None of them are individually urgent (admin bypass keeps the cycle moving; the shell-interp findings are not exploitable in the current single-user trust model), but together they retire the three remaining loose ends from the public-readiness drop and they're small enough to ship in one MINOR.

## Scope

In:

- **F-1 fix:** `platform/install.js:32, 35, 39` — three `execSync(\`git -C "${workshopPath}" ...\`)` calls become `spawnSync("git", ["-C", workshopPath, ...], { encoding: "utf8" })`. Array-form arg passing skips shell parsing entirely.
- **F-1b fix:** `platform/bootstrap.js:175-186` — `phaseWriteActivation` template-literal bodies for `actBody` and `binBody` shell-escape `vaultPath`, `scriptsDir`, `cliPath` via a small `shellSingleQuote` helper before substitution.
- **CI default-subscription fix:** `platform/bootstrap-lib/wizard.js:36-43` `DEFAULT_MECHANISMS_CHECKED` — add `"icons"` (the `nav-buttons` dependency) to the default-checked list. Alternative discussed under Decisions.
- **Manifest bumps:** the two `platform/` edits touch `platform/install.js` and `platform/bootstrap.js`. Neither belongs to a mechanism manifest; they live at the platform-orchestration layer. `workshop_version` bumps to `0.60.0`. `package.json` version follows.
- **Brew formula:** auto-bumps via `release.yml` on tag push (no manual edit).

Out:

- Any other `platform/` code change.
- Any blueprint / mechanism manifest version bump beyond what the orchestration-layer edits force.
- F-2 (`eval`-dispatch in `run-audit.js`) and F-3..F-7 (info-class findings) — defer further; this cycle is the medium-severity bundle only.
- PII scrub of `accuris` / `headspace` / `ERO` / personal email / absolute paths — separate policy-decision cycle if the user opts in.
- README polish, issue templates, code of conduct, Discussions — out per public-readiness design.

## Per-finding fix details

### F-1 — `gitState` array-form swap

Current (`platform/install.js:32, 35, 39`):

```js
result.commit = execSync(`git -C "${workshopPath}" rev-parse HEAD`, { encoding: "utf8" }).trim();
const out    = execSync(`git -C "${workshopPath}" describe --tags --exact-match HEAD 2>/dev/null`, { encoding: "utf8" }).trim();
const status = execSync(`git -C "${workshopPath}" status --porcelain`, { encoding: "utf8" });
```

Proposed:

```js
const { spawnSync } = require("child_process");
function _git(workshopPath, args, opts) {
    const r = spawnSync("git", ["-C", workshopPath, ...args], { encoding: "utf8", ...opts });
    if (r.status !== 0) throw new Error(r.stderr || `git ${args.join(" ")} failed`);
    return (r.stdout || "").trim();
}
// ... inside gitState():
try { result.commit = _git(workshopPath, ["rev-parse", "HEAD"]); } catch { /* leave null */ }
try {
    const out = _git(workshopPath, ["describe", "--tags", "--exact-match", "HEAD"]);
    result.tag = out.length > 0 ? out : null;
} catch { /* leave null */ }
try {
    const status = _git(workshopPath, ["status", "--porcelain"]);
    result.dirty = status.length > 0;
} catch { /* leave null */ }
```

Note the `describe ... HEAD 2>/dev/null` original silenced stderr via shell redirection; the array-form version achieves equivalent behavior by swallowing the throw in the catch (stderr goes to the parent stderr of the spawn, which the installer's caller already silences via Notice surfaces). If the installer caller is found to surface stderr, add `stdio: ["ignore", "pipe", "ignore"]` to the `_git` call for `describe`.

### F-1b — `phaseWriteActivation` shell-escape

Add helper (file-private):

```js
function _shellSingleQuote(s) {
    return "'" + String(s).replace(/'/g, "'\\''") + "'";
}
```

Then rewrite the two template literals:

```js
const actBody = `# Sauce activation — sourced into your shell.
# Generated by phaseWriteActivation; absolute paths resolved at install time.
export SAUCE_VAULT=${_shellSingleQuote(vaultPath)}
case ":$PATH:" in
  *:${_shellSingleQuote(scriptsDir)}:*) ;;
  *) export PATH=${_shellSingleQuote(scriptsDir)}:"$PATH" ;;
esac
echo "sauce active. Try: sauce status"
`;
const binBody = `#!/usr/bin/env bash
exec node ${_shellSingleQuote(cliPath)} "$@"
`;
```

Single-quoted bash strings do not undergo `$VAR`, backtick, or `\` expansion — only `'` itself needs escaping, and the helper handles that with the standard `'\''` close-reopen idiom.

The `case` pattern matching still needs special handling: the inner `*:${...}:*` glob uses single-quoted match strings, which bash treats as literal patterns. This is correct (we want to literal-match the scripts dir, not glob it).

### CI default-subscription fix

Current (`platform/bootstrap-lib/wizard.js:36-43`):

```js
const DEFAULT_MECHANISMS_CHECKED = [
    "customjs-guard",
    "nav-buttons",
    "cards",
    "accent-button",
    "styling",
    "convenience"
];
```

Proposed:

```js
const DEFAULT_MECHANISMS_CHECKED = [
    "customjs-guard",
    "icons",          // dep of nav-buttons; must precede it for deps-first ordering
    "nav-buttons",
    "cards",
    "accent-button",
    "styling",
    "convenience"
];
```

Rationale: nav-buttons has depended on icons since the icons mechanism shipped (v0.21.0+), but the wizard's default-checked list was never updated to include the new dep. The installer's dep-order skip is correct behavior; the data fix is in the wizard.

The same edit also fixes `sauce bootstrap --non-interactive --no-register` (the CI invocation), which uses these defaults verbatim, so the green CI run is the integration test.

## Success criteria

- `gitState` no longer interpolates a free-form path into a shell string.
- The activation file + sauce shim are robust to spaces, `$`, `'`, `"`, backtick, and `\` in any of the three interpolated paths.
- CI on `main` shows preflight ✓ on both macOS and Ubuntu after the cycle lands.
- `sauce bootstrap --non-interactive --no-register` against a fresh tempdir produces a vault with zero error/skip history entries on first run.
- CodeQL JavaScript scan on `main` no longer flags the shell-string interpolation patterns at the cited line numbers.
