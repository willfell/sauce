# v0.17.0 Finance widget UIs Implementation Plan

> **For the implementing agent:** REQUIRED SUB-SKILL: Use executing-plans to implement this plan task-by-task.

**Goal:** Ship in-place editor widgets on every finance entity page so users never edit frontmatter manually. Widget DOM inputs bypass every YAML edge case from v0.16.0 (auto-parse-to-Date, inline-flow NBSP, boolean→string).

**Architecture:** Single shared `FinanceFrontmatter` helper class wraps `app.fileManager.processFrontMatter` (Obsidian 1.4+ official API). Each widget reads frontmatter via `dv.current()` (cached) and writes via the shared helper. Hybrid edit UX: read-only rows + Add modal + click-row-to-edit modal + per-row delete `✕`. Two new path-prefix-detect nav-button classes mirror `InvoiceNavButtons`. Hub button styling polish lives in `FinanceHubCards` row sections.

**Tech Stack:** CustomJS classes (vanilla JS) ; Obsidian APIs (`app.fileManager.processFrontMatter`, `app.workspace.openLinkText`, `app.vault.getAbstractFileByPath`) ; Dataview `dv.view` + customjs-guard wrapper ; Node test harness `platform/test/run-renderer.js` (jsdom-style stub) ; `node platform/test/run-install.js` for install green-check.

**Predecessors:** `Docs/plans/2026-05-05-v0.17.0-finance-widgets-design.md` (approved). v0.16.0 result writeup. `_handoff-protocol.md`.

**Bumps:** workshop 0.16.0 → 0.17.0 MINOR ; finance 0.1.4 → 0.2.0 MINOR ; no mechanism bumps.

**Reserved 0.17.x headroom:** finance 0.2.0 → 0.2.1 → 0.2.2 → ... per gotcha 9 + first-of-its-kind UI precedent (5-data-point: v0.5.0/v0.11.0/v0.12.0/v0.6.0/v0.16.0 → expect 2-6 inline CFs).

**Pacing:** per-task subagent dispatch on widget classes (T1.1-T1.3, T2.1-T2.4) ; master-driven on manifest/version-bump mechanics + harness extensions + template-body edits. Stage-close `AskUserQuestion` checkpoints. TDD-first on all renderer harness shell-DOM cases.
