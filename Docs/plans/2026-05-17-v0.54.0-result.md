## FIX-LATER notes

1. **FLN-FA2-1 — Templater `tp.file.creation_date` vs `{{now.X}}` in entity-create.** Templates use Templater syntax (`<% tp.file.creation_date("YYYY-MM-DDTHH:mm:ssZ") %>`), but entity-create uses `{{now.X}}`. The two substitution mechanisms produce equivalent ISO+TZ output but are evaluated at different times (template-render vs entity-create-render). Documented mismatch; harmless for users since both fire at note creation. Consider unifying in a future cycle.

2. **FLN-FA2-2 — `mtimeIsoWithTz` uses `Z` (UTC) not local TZ.** The migrate-frontmatter `created_at:` backfill from file mtime produces `2026-05-16T00:02:05Z` instead of local-TZ form. Audit walker accepts both shapes via the ISO_WITH_TZ_RE regex. Cosmetic inconsistency vs the entity-create + template-render paths which use local TZ. Could be normalized in a future patch.

3. **FLN-FA2-3 — accuris-sauce skipped.** Per user direction at Phase B. Will need a separate migration run before accuris ships canonical-vocab-extended blueprints (or it'll surface `legacy_key_used` warnings on its meetings/people files).

4. **FLN-FA2-4 — products + teams not subscribed at any consumer.** The blueprint changes ship in the catalogue but are dormant until first subscription. No consumer-side migration needed yet. Audit walker handles this via workshop-fallback for the canonical-vocab template path.

5. **FLN-FA2-5 — `tp.file.creation_date` returns blank string OUTSIDE Templater-managed file creation.** Templates with this token inside a `<% %>` block work at Templater-created notes; if a user opens an existing template-derived note and that token re-evaluates (it shouldn't, since Templater is single-pass at creation), the value would be empty. Not load-bearing under current usage.

6. **FLN-FA2-6 — Workshop-side `meetings.json` rule file regenerated by dogfood.** Workshop dogfood reinstall regenerated `ranch/rules/meetings.json` from the bumped manifest. Future test fixtures referencing `ranch/rules/meetings.json` directly would see the new shape; harnesses use the manifest source-of-truth instead, so no test breakage.
