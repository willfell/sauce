# v0.4.2 — runTemplaterTemplate split-field schema Implementation Plan

> **For the implementing agent:** REQUIRED SUB-SKILL: Use `de:executing-plans` to implement this plan task-by-task (or `de:subagent-driven-development` if running in the same session as the master).

**Goal:** Replace the single moment-format `folder` + `filename` fields on the `runTemplaterTemplate` nav-button action with a five-field split (`folder_prefix` + `folder_date_pattern` + `filename_prefix` + `filename_date_pattern` + `filename_suffix`). Bracket-escape gap (v0.4.0 FIX-1 + v0.3.2 FIX-2) becomes architecturally impossible. Migrate to-do + meetings in-cycle. Workshop bump 0.8.1 → 0.9.0 MINOR; annotated tag `v0.9.0` at close.

**Architecture:** Two stages. S1 lands the schema mechanics: nav-buttons mechanism 2.1.0 → 2.2.0 (renderer dispatch rewritten to compose folder/filename from five fields at click-time); `install.js` `validateAndResolve` rewritten to apply `substituteLenient` to the three literal fields and pass moment-format patterns through verbatim; `substituteForMomentFormat` helper deleted (no remaining callers); harness extended (R1 rewritten + R2/R3/R4 added) before any code change (TDD). S2 migrates the two consumer manifests (to-do + meetings), bumps subscriptions, runs install + manual smokes, writes the result, commits the cycle close + annotated tag `v0.9.0` + push. Hard cut on legacy fields — silently ignored if present after migration.

**Tech Stack:** Node.js (Templater desktop runtime); moment.js (already in renderer); CustomJS (SpaceNavButtons class only — no new classes); Bash + git.
