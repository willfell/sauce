## What shipped

NEW `activity-feed@0.1.0` mechanism — universal cross-blueprint activity feed renderer wrapping a time-windowed Dataview query over the canonical `type:` + `created_at:` vocab shipped FA-1..FA-7. Generalizes the BacklinkPanel reverse-query pattern from entity-keys (v0.61.0 FA-8) to time. Renders via existing BeaconCards.

`ActivityFeed.render(dv, opts)`:
- `scope`: `"today"` | `"week"` | `"month"` (default `"today"`)
- `blueprints`: canonical `type:` value list (default all 17 vocab types)
- `groupBy`: `"blueprint"` | `"hour"` | `"none"` (default `"blueprint"`)
- `limit`: number (default 50)
- `useStatusChangedAt`: boolean — when `true`, time-window comparison uses `status_changed_at` instead of `created_at` (FA-9a "Today's project status changes" section)
- `title`: optional H3 above the panel

Internals: `_DEFAULT_BLUEPRINTS` getter (17 canonical types); `_resolveTimeWindow` via `window.moment` with native-Date fallback (no isoWeek on fallback); `_query` with lexicographic ISO/YYYY-MM-DD compare for legacy notes; empty-state ("No activity in this <scope>"); 3 renderers (flat / group-by-blueprint / group-by-hour). Notice-on-error for unresolved window + missing BeaconCards.

## FIX-LATER-NOTEs (v0.62.0)

- **FLN-v62-1:** `_resolveTimeWindow` native-Date fallback doesn't handle isoWeek correctly when the OS locale starts week on Sunday. Affected when `window.moment` is somehow unavailable (rare in Obsidian). Acceptable degradation.
- **FLN-v62-2:** `useStatusChangedAt` opt is per-section, not per-page-blueprint. If a Today.md section mixes types where some carry `status_changed_at` and some don't, the non-status types drop out silently. Today.md project-section only filters `type: project` so this is fine in practice.
- **FLN-v62-3:** Today.md frontmatter `created_at` is static `2026-05-18T23:30:00-06:00`. After cycle ship, this becomes a stale-looking timestamp. Acceptable: all cowork hub files have static created_at timestamps (per FA-5 convention). Not a regression.
- **FLN-v62-4:** Workshop dogfood subscription does NOT include cowork — the cowork hub edits + Today.md + rule_fragment are validated via test harness JSON-parse + literal-grep only. Live install path tested at 4-vault consumer deploy time.
