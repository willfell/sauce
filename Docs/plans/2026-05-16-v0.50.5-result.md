# v0.50.5 — Project nav-buttons wrap on narrow widths (result)

**Date:** 2026-05-16
**Cycle:** PATCH v0.50.4 → v0.50.5
**Cadence:** controller-direct (single-file CSS change; sub-agents would be overkill)
**Sauce-pipeline round:** 4

## What shipped

A minimal CSS fix to `ProjectNavButtons.render()` so the project-specific button row wraps to multiple flex rows when it can't fit on one line (fixes mobile clutter on long project/task names without removing any buttons or labels).

Three CSS edits in [`platform/blueprints/project/helpers/project-nav-buttons.js`](../../platform/blueprints/project/helpers/project-nav-buttons.js) around lines 417–446:

1. Container: `flex-wrap: nowrap` → `flex-wrap: wrap`.
2. Button style: `flex: 1; min-width: 0;` → `flex: 0 1 auto;`.
3. Label span: added inline `style="white-space: nowrap;"` so the label stays on one line and the button wraps to a new flex row instead.

`project@1.10.3` → `1.10.4` PATCH; workshop `0.50.4` → `0.50.5`.

## Why this approach

User picked wrap-on-narrow-widths over three alternatives (mobile icon-only, conditional button trim by context, multi-row split via row-count math). The wrap approach is the minimum-blast-radius option: pure CSS, no JS branching, no `app.isMobile` check, no manifest schema changes. Wide desktop windows look identical to v0.50.4; narrow desktop windows and mobile wrap gracefully.

Full diagnosis + alternatives in [`2026-05-16-v0.50.5-project-buttons-mobile-wrap-design.md`](./2026-05-16-v0.50.5-project-buttons-mobile-wrap-design.md).
