# v0.55.0 — FA-3 Project migration (result)

Heaviest entity migration. project@1.12.1 → 1.13.0 MINOR. 5 template families + 3 rule_fragments + 2 entity-create entries + drive-by regex fix. Whole cycle (no FA-3a/FA-3b split needed). Consumer-vault migration ran across 3 vaults (headspace, barebones, ero; accuris skipped per round 10 user direction).

## What shipped

**Template family updates (canonical vocab; drop discriminator + temporal tags; preserve allowlisted functional tags):**
- `Project Map.md`: `created` → `created_at` ISO+TZ
- `Project Board.md`: drop `board` discriminator tag (kanban-plugin: board YAML key is separate); preserve `project/<slug>` nested tag; add `created_at`
- `Kanban Card.md`: drop temporal `YYYY/MM/DD` tag; preserve `kanban-card` + `project-card` (allowlisted); rename `created` → `created_at`
- `Task Note.md`: drop `{{DATE_TAG}}` temporal; ADD `type: task-note` (was missing); rename `created` → `created_at`
- `Task Board.md`: drop `{{DATE_TAG}}` temporal; preserve `task-board` (allowlisted); ADD `created_at`
- `Task Board Card.md`: **DRIVE-BY REGEX FIX** — `^beacon/projects/` → `^spice/projects/` (stale pre-v0.23.0 rebrand reference; was making `task_parent:` empty on every new card)
- `Docs Hub.md`: rename `created` → `created_at`; `docs-hub` tag retained

**Entity-create entries:**
- `project`: drop `tags: ["project"]` discriminator; add `created_at` canonical (ISO+TZ moment format)
- `doc-note`: add `created_at` canonical; tags retained (`doc-note` is functional)

**Rule fragments (3 rules: project, docs-hub, doc-note):**
- All declare `extends: "_canonical-vocab"` (FA-1 substrate)
- `project` rule: drop `required_frontmatter.created` + `required_tags: [{ tag: "project" }]`; preserve domain narrowings (description, workstreams, status enum, status_changed_at, teams, products)
- `docs-hub` rule: drop `required_frontmatter.created`; preserve type/slug/name + `required_tags: [{ tag: "docs-hub" }]` (functional marker)
- `doc-note` rule: drop `required_frontmatter.created`; preserve type/project/slug + `required_tags: [{ tag: "doc-note" }]` (functional marker)

**Mid-cycle softening of `cmd-migrate-frontmatter --apply`:**
Real-world consumer notes carry `workstreams:` as a list of OBJECTS (e.g. `- id: foo\n    name: Foo`) which is outside the minimal YAML parser's scope. Pre-fix `--apply` halted on the first parse error, blocking migration of the 96+ well-formed sibling files. Softened to skip-and-warn: applies to well-formed files, counts parse-error files separately, reports both in the summary. MF-AP-4 test updated; +2 new sub-asserts (MF-AP-4b reports count; MF-AP-4c sibling still migrates).

## FIX-LATER notes

1. **FLN-FA3-1 — `extends` doesn't propagate into `frontmatter_branch[]`.** When a rule_fragment uses `frontmatter_branch[]`, the branch's `required_frontmatter` is applied INSTEAD of the merged-with-extends fragment's. So FA-3's project rule has `extends: "_canonical-vocab"` declared but the branch (where the actual checks live) doesn't get the canonical type+created_at requirements. Forward-defense via templates + migration + audit walker. To fully wire validator-enforcement, FA-1's `_resolveExtends` could be extended to also merge into branches. Deferred.

2. **FLN-FA3-2 — Path-based `type` backfill is too aggressive for project.** `cmd-migrate-frontmatter._inferTypeFromPath("spice/projects/...")` returns `"project"` for ALL files under `spice/projects/` — but task notes, task boards, kanban cards, etc. under that path have their own canonical types. Result: some sub-files got `type: project` backfilled when they should be `task-note` / `task-board-card` / etc. Benign because `created_at` was also backfilled (no `missing_canonical_key` violations). Fix: drop `projects` from `PATH_TO_TYPE` or make inference more granular (only backfill for `<slug>/<slug>.md` or `<slug>/Project.md`). Affects ~20-30 sub-files across 3 vaults. Cleanup cycle candidate.

3. **FLN-FA3-3 — `--apply` softened to skip parse errors.** Was: halt-on-first-error. Now: skip-and-warn. 14 files in 3 vaults remain with legacy `created:` because their YAML couldn't be parsed (list-of-objects `workstreams:`). Either extend the YAML parser to handle list-of-objects, or write a smaller targeted migration script for those files. Cleanup cycle candidate.

4. **FLN-FA3-4 — accuris-sauce migration deferred** (carry-over from FA-2).

5. **FLN-FA3-5 — Task Note.md `type: task-note` added.** Was missing pre-FA-3 (a gap from v1.6.0 era when task-notes were created via Templater). New task notes will emit `type: task-note`; existing notes can be migrated via a future cycle's dedicated walker (path-based backfill is unsafe here per FLN-FA3-2).

6. **FLN-FA3-6 — Drive-by regex fix on Task Board Card.md** restores correct `task_parent:` population on new task-board-cards. Verified at template-source level; not exercised against a live new-card creation in this cycle.
