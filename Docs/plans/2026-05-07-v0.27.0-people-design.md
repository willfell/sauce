> [!info] API contract — quote literally per v0.6.0 lesson

```javascript
class PeopleRendering {
  /**
   * Render a single Person as an inline chip (name + hover tooltip).
   * @param {HTMLElement} parent
   * @param {string} personLink — wikilink "[[First Last]]" or path "spice/people/First Last.md".
   * @param {object} [opts]
   * @param {boolean} [opts.openOnClick=true]
   * @param {string}  [opts.tooltipFields="company,title"]
   * @returns {HTMLSpanElement}
   */
  renderChip(parent, personLink, opts) { /* ... */ }

  /**
   * Render a single Person as a BeaconCards row card.
   * Delegates to customJS.BeaconCards.render with a synthetic page list of one.
   * @param {object} dv — Dataview shim (must have container + createEl).
   * @param {string} personLink
   * @param {object} [opts] — passthrough to BeaconCards.
   * @returns {void}
   */
  renderCard(dv, personLink, opts) { /* ... */ }

  /**
   * Render a list of notes that mention a person, OR a list of people mentioned in a note.
   * @param {object} dv
   * @param {object} query
   * @param {"mentioning_person"|"mentioned_in_note"} query.mode
   * @param {string} [query.personLink]   — required when mode === "mentioning_person".
   * @param {string} [query.scopePath]    — Dataview source path (e.g., "spice/meetings"). Default: vault-wide.
   * @param {string} [query.notePath]     — required when mode === "mentioned_in_note".
   * @param {object} [opts]
   * @param {"chips"|"cards"|"list"} [opts.style="cards"]
   * @param {number} [opts.limit]
   * @param {string} [opts.sortBy="-file.cday"]
   * @returns {void}
   */
  renderMentionList(dv, query, opts) { /* ... */ }

  /**
   * Extract all [[Person]] wikilinks from a markdown body string.
   * Filters to links whose resolved path begins with `spice/people/`.
   * @param {string} markdownBody
   * @param {object} [opts]
   * @param {boolean} [opts.unique=true]
   * @returns {Array<{display: string, target: string}>}
   */
  extractMentions(markdownBody, opts) { /* ... */ }
}
```

**Behavioral contracts:**
- **Mobile-aware** per v0.4.2 gotcha 7 — chips collapse to color-tinted name initials; cards inherit BeaconCards' mobile behavior.
- **Path-prefix filter is `spice/people/`** — hardcoded in `extractMentions`. If `module_directory` ever changes, this is a one-line update; flagged as a v0.28.0 migration-prep concern.
- **`renderChip` failure mode** — unresolved `personLink` renders red-tinted span + `Notice("Unknown person: <link>")` on click.
- **`renderCard` synthetic-page composition** — uses BeaconCards' synthetic-page pattern (v0.12.0 lesson). Pre-fetches frontmatter via `app.metadataCache.getFileCache(file).frontmatter` (sync; no await needed).
- **`renderMentionList` Dataview composition** — `mentioning_person` mode emits `dv.pages(scopePath).where(p => contains(p.file.outlinks, dv.fileLink(personLink)))`; `mentioned_in_note` reads file via `app.vault.read` + calls `extractMentions`.
- **`extractMentions` strict matching** — `\[\[([^\]|#]+)(?:\|[^\]]+)?\]\]` regex; does NOT resolve aliases via metadataCache; does NOT handle escaped brackets or transclusions. Dedupes by canonical target.
