## 1. Architecture overview

```
sauce migrate --from /sync/accuris            # dry-run default → migration-plan.json + stdout summary
sauce migrate --from /sync/accuris --commit   # 5-phase atomic write
```

**Lib layout:**

```
platform/
├── cli/
│   └── cmd-migrate.js                   # NEW; 5th verb dispatcher
├── migrate/                              # NEW subsystem
│   ├── dispatcher.js                    # walks source, routes per migrator priority
│   ├── section-extract.js               # markdown heading/callout extraction helper (per v0.27.0 lesson g)
│   ├── wikilink-rewrite.js              # phase 4.5 cross-blueprint pass
│   ├── plan.js                          # plan output + migration-plan.json writer
│   ├── commit.js                        # 5-phase orchestrator
│   ├── verbatim.js                      # fallback migrator (priority 9999)
│   └── migrators/
│       ├── people.js                    # priority 10
│       ├── daily.js                     # priority 20
│       ├── meetings-note.js             # priority 30
│       ├── meetings-hub.js              # priority 40
│       └── to-do.js                     # priority 50
└── test/
    └── run-migrate.js                   # NEW harness; ~9 selectors
```

**Key invariants:**

- **Source vault is read-only.** Never written to. Codified as a new landmine candidate.
- **Failure-loud.** Any error mid-phase aborts + restores from backup.
- **Idempotent.** `--commit` re-run wipes + redoes; backup-sibling stack preserved.
- **Lowercase platform dirs** (#19) preserved at target.
- **Module-directory invariant** (#11) — every blueprint migrator lands content under `spice/<module_directory>/` only.

**Skip-list (logged as warnings, not errors):**

- `*.tmp` (e.g., `Timestamps/Meetings/*.md.tmp` Templater intermediates)
- `Timestamps/Invalid date/` (corrupted Templater output)
- `to-delete/`
- `.obsidian/`, `.git/`, `.DS_Store`, `node_modules/`
- Bootstrap-managed paths the migrator never writes (`pantry/`, `ranch/`, `.obsidian/<bootstrap-installed>`, `Scripts/sauce`, `Scripts/activate.sh`, `package.json`, `package-lock.json`)

**Filename collision policy:** scan-time detection across planEntries; if two source files map to the same target, abort failure-loud.

**Person-tag policy:** meeting note frontmatter `person/Jon-Levin` tags preserved verbatim. Body `[[Person Name]]` wikilinks resolve naturally to `spice/people/<Name>.md` post-migration via Obsidian short-form resolution.

## 3. Dispatcher + migrator interface

**Migrator interface contract** (every per-blueprint migrator exports):

```js
module.exports = {
  name: "people",                          // unique id
  priority: 10,                            // lower = checked first; verbatim is 9999
  canHandle(srcRelPath, srcStat) { ... },  // true if this migrator owns the file
  plan(srcRelPath, srcAbsPath, ctx) {      // returns { action, src, tgt, warnings, rewrite_summary }
    return {
      action: "rewrite_blueprint",
      src: srcRelPath,
      tgt: "spice/people/Alex Bennett.md",
      warnings: [],
      rewrite_summary: { rename: false, frontmatter_changes: 0, body_regenerated: true }
    };
  },
  migrate(planEntry, srcAbsPath, tgtRoot, ctx) { ... }  // performs the write atomically
};
```

**Dispatcher flow** (`platform/migrate/dispatcher.js`):

1. Resolve `--from <src>`; refuse if missing or not a directory.
2. Resolve target = `process.cwd()`; for `--commit`, target must be a vault dir (has `.obsidian/` OR is bootstrapped Sauce).
3. Load all migrators from `platform/migrate/migrators/` + sort by priority (verbatim last).
4. Walk source recursively per file:
   - Compute `srcRelPath`; skip skip-list matches.
   - For each migrator in priority order: if `canHandle()` → assign owner; break.
   - If no migrator claims → assign verbatim fallback.
   - Call owner.plan() → push entry into `planEntries[]`.
5. Aggregate warnings + collisions:
   - Detect tgt-path collisions across planEntries; failure-loud abort if any.
   - Categorize warnings (skipped files, malformed frontmatter, ambiguous dates).
6. Emit plan:
   - `--plan` (default): print formatted table to stdout + write `migration-plan.json` next to invocation cwd; EXIT 0.
   - `--commit`: invoke 5-phase write.

**Action vocabulary:** `copy_verbatim` | `rewrite_blueprint` | `skip` | `bootstrap_managed`.

## 4. 5-phase --commit flow with error handling

| Phase | Action | Failure behavior |
|---|---|---|
| **0. precheck** | Verify `--from` exists + is dir; verify cwd is target vault root; compute backup-sibling path; refuse if backup-sibling exists. Re-run dispatcher. | abort; no writes; print error + remedy. |
| **1. backup** | `cp -R <vault>/* <vault>.pre-migration-<ts>/`; `cp -R <vault>/.obsidian/ <backup>/`; `rm -rf <vault>/<everything except dir entry>`. Vault dir entry preserved (Obsidian Sync's anchor). | abort + leave-backup-intact: backup is the rollback path. |
| **2. bootstrap** | Invoke shared bootstrap orchestrator phases (`phaseFirstRunWizard` + `phaseFetchPlugins` + `phaseRunInstaller`) using `--non-interactive --mechanisms=all --blueprints=all`. | abort + restore: copy backup contents back; remove partial bootstrap. |
| **3. carry-verbatim** | Walk planEntries with `action: "copy_verbatim"`; cp source→target preserving relative path + mtime; mkdir target dirs as needed. | abort + restore (same as phase 2). |
| **4. rewrite-blueprints** | Walk `action: "rewrite_blueprint"`; per migrator, call `migrate(planEntry, srcAbs, tgtRoot, ctx)`; each writes atomically (temp file + rename). | abort + restore. |
| **4.5. wikilink-rewrite** | Walk every migrated `.md` file (NOT verbatim carries); apply WIKILINK_REWRITE_RULES (daily prefix→suffix, meetings-hub prefix→suffix, to-do prefix→suffix, Extras/People→spice/people fully-qualified path). Idempotent. | log + warn on parse failure; never abort migration (rewrites are best-effort safety net). |
| **5. finalize** | Write `migration.log` (every action with timestamp + result); write `migration-plan.json` (final actual plan); emit summary (counts per migrator + skipped + warnings); exit 0. | log-write failures degrade to warning. |

**Backup sibling is OUTSIDE the vault dir** (`/sync/workshop/accuris-beacon-poc.pre-migration-<ts>/`) so Obsidian doesn't try to index it.

**Re-run posture:** `--commit` is idempotent — second run creates a new timestamped backup, wipes, redoes. Iteration-friendly.

**Phone Sync expectations** documented in `Docs/migrate.md`:

- First post-migration sync push will be large (~125 notes + verbatim attachments + Cowork/ + Files/ + ranch/ + pantry/). May take minutes.
- Backup sibling won't sync (Obsidian only indexes the vault dir).

## 5. Per-blueprint user-section vs platform-managed mapping

For each migrator, a declarative section mapping at `platform/migrate/migrators/<name>.js` exporting `SOURCE_SECTIONS` (regex/heading-match) + `TARGET_TEMPLATE` (Sauce template with `{{user_section_<name>}}` slots).

| Blueprint | Frontmatter handling | Platform-regenerate | User-preserve | Notes |
|---|---|---|---|---|
| **people** | Identity copy. Drop empty `aliases:` and `phone:` if absent. Add `tags: [accuris]` if missing. | `# [[Name]]` heading; `## Meetings` block (Sauce dataview/people-rendering) | `## Notes` content (verbatim, default empty) | Already shape-matched per v0.27.0; thinnest migrator. |
| **daily** | Preserve `created`, `tags`, `cssclasses`. Drop legacy fields. | Top dataviewjs blocks (SpaceNavButtons, SpaceDailyDashboard) | `> [!summary]- Morning Briefing` callout (full block); `## Today's Tasks` mirror via wikilink to to-do; everything below dashboard markers up to next platform marker | Filename prefix→suffix. |
| **meetings-note** | Preserve `date`, `type`, `company`, `summary`, `daily_note` (rewrite wikilink target), `tags` (incl `person/X`). | `# [[<title>]]` heading; `## Attendees` chip block prepended (v0.27.0 meetings@0.3.0); legacy `tags: [[🗣 Meetings MOC]]` line dropped; `Date: [[...]]` line dropped | `**Attendees**:` bullet list (the `[[Name]]` wikilinks); `**Clients**` block if non-empty; `## Agenda/Questions`, `## Notes`, `## Action Items` verbatim | Filename rewrite + space-preserved title. |
| **meetings-hub** | Preserve `created`, `daily_note` (rewrite), `tags`, `cssclasses`. | 100% regenerate body from Sauce hub template | (none — hubs are pure platform) | Filename rewrite. |
| **to-do** | Preserve `created`, `daily_note` (rewrite), `tags`, `cssclasses`. | "Back to Daily Note" dataviewjs button regenerated via Sauce to-do template | `## Today's Tasks` checkbox list verbatim | Filename rewrite. |

**Section-extraction algorithm** (shared helper at `platform/migrate/section-extract.js`):

```js
extractSection(body, headingMatcher, opts = { stopAt: nextHeadingOrEof })
  → { found: bool, content: string, raw: string }
```

Matches markdown headings (`# / ## / ### / >`) with regex; returns substring from heading start to next-heading-of-equal-or-higher-level (or EOF). Works for callouts (`> [!type]-`) too. Reuses v0.27.0 lesson (g) — never extract from full body.

**Wikilink rewrite registry** (phase 4.5):

```js
WIKILINK_REWRITE_RULES = [
  // Daily: 2025-04-29-Tuesday → Tuesday-2025-04-29
  { match: /^(\d{4}-\d{2}-\d{2})-((?:Mon|Tues|Wednes|Thurs|Fri|Satur|Sun)day)$/, replace: "$2-$1" },
  // MeetingHub: 2026-01-06-Meetings → Meetings-2026-01-06
  { match: /^(\d{4}-\d{2}-\d{2})-Meetings$/, replace: "Meetings-$1" },
  // To-do: 2026-01-05-ToDo → ToDo-2026-01-05
  { match: /^(\d{4}-\d{2}-\d{2})-ToDo$/, replace: "ToDo-$1" },
  // Fully-qualified path: Extras/People/X → spice/people/X
  { match: /^Extras\/People\/(.+)$/, replace: "spice/people/$1" },
];
```

Walks bodies + frontmatter (`daily_note: "[[...]]"`); rewrite is idempotent. Counts emitted to `migration.log`.

### Risk register

| Risk | Likelihood | Mitigation |
|---|---|---|
| Migration ships latent shape miss for one migrator (e.g., a Daily edge case from 2025) | High | Dry-run against full real Accuris in S3 before --commit; review migration-plan.json line by line |
| --commit abort mid-phase leaves vault in partial state | Medium | Backup-first phase 1 + abort+restore in every phase failure path |
| Phone Sync rejects giant changeset | Low-Med | Document in `Docs/migrate.md`; user can pre-disable Sync during migration + re-enable after |
| Cross-vault wikilink rewrite missing a regex case | Medium | Section-scoped TDD harness + dry-run shows before/after wikilinks per file |
| Person-tag `person/Jon-Levin` frontmatter format ages poorly | Low | Preserved verbatim per Section 5; future cycle can codify |
| `Invalid date/Invalid date-Invalid date.md` corrupted file in source breaks scan | Low | Skip-list at dispatcher level; logged as warning |
| Existing accuris-beacon-poc has untracked changes user wants to keep | Med | Phase 1 backup is full + lossless; user can restore from backup any file post-migration |
